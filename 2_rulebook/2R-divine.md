# 神々・精霊との交渉

## このファイルの位置

神々と精霊の運用、加護のメカニクス、勇者・聖女・精霊使いの扱い、邪神信仰の運用、祈り・契約の判定を定める。世界設定としての神々・精霊の詳細は別ファイル（1_setting/ 配下）を参照。

## 三秒で読む要約

神々と精霊は階層が異なる超越的存在。加護は信仰・行動・気まぐれの組み合わせ。勇者・聖女もPCがなれる。邪神信仰は不可逆変化を伴う。

## 一分で読む要約

神々はギリシア神話的多神教体系で階級を持ち、「アルカディアの盟約」により特定の国家・陣営への加護は禁じられる。ただし特定個人への加護は盟約違反にならず、加護を受けた者は「勇者」「聖女」となる。精霊は神より位が低く、盟約には縛られない自然界の存在で、気まぐれに加護を与える。加護の発動はハイブリッド（信仰の蓄積＋行動の整合性＋神の裁量）。PCも勇者・聖女・精霊使いになりうるが、加護はユニークスキルとして扱う。邪神信仰のPCも認められるが、邪神契約は不可逆変化を伴う「魂を売る」典型として運用される。祈りの応答は物語的判断で希少化する。

# 1. 基本構造

## 1.1 神々と精霊の関係

```yaml
神々と精霊:
  
  神々:
    - 多神教体系
    - 階級あり（主神級 > 概念神 > 役割神）
    - 「アルカディアの盟約」に拘束される
    - 大規模な世界の摂理に関わる
  
  精霊:
    - 神より位が低い超越的存在
    - 自然界に存在
    - 自然的要素に対応した能力・生態
    - 「アルカディアの盟約」未締結
    - 気まぐれに加護を与える
    - 生活文化に深く根差している

両者の連続性:
  - 神と精霊は連続的な存在
  - 大精霊は下位神に近い力を持つ
  - 役割神は精霊と区別しにくい場合もある
  - しばしば混同される（崇拝対象として）

詳細: A1ファイル「神について」「精霊」セクション
```

## 1.2 アルカディアの盟約

```yaml
アルカディアの盟約:
  
  概要:
    滅亡戦争の反省から神々が結んだ協定。
    特定の国家・陣営に無償で肩入れすることを禁じる。
  
  対象:
    神々のみ（精霊は未締結）
  
  例外:
    特定個人への加護は盟約違反にならない
    → 加護を受けた者が「勇者」「聖女」となる根拠

GMの運用:
  
  原則:
    盟約は背景設定として扱う。
    GMが裁定の度に意識する必要はない。
  
  影響する場面:
    - 神の介入が起こりそうな大規模事象
    - 国家間の戦争で神々が動こうとする場面
    - PCが神に「組織への加護」を求める場面
  
  違反したらどうなるか:
    - 詳細は世界設定の解像度に応じて
    - 滅亡戦争の再来を恐れる神々の自制が働く

IF: 大規模事象で神の介入が論点になる
THEN: 盟約により「無償の肩入れ」は起きないと裁定

IF: PCが神に組織レベルの加護を求める
THEN: 盟約により拒否されると裁定（個人への加護は別）
```

# 2. 神々の運用

## 2.1 神々の階級と権能

```yaml
神々の階級:
  
  主神級:
    - 最強の権能
    - 大きな概念を司る
    - 光、闇、生、死、創造、破壊など
    - 例: 主神アルカ
  
  概念神:
    - 中位の権能
    - 抽象的な概念を司る
    - 戦、愛、知恵、商など
  
  役割神:
    - 下位の権能
    - 具体的な役割を司る
    - 農耕、かまど、河川、街道など
    - 精霊と区別しにくい場合あり

詳細:
  - 個別の神々の名前と権能は世界設定（1_setting/）
  - キャラクターリスト（4_character_list/）に主要な神を登録
```

## 2.2 教団と信仰

```yaml
教団:
  
  定義:
    特定の神々を信仰する集団。
    神々からの祝福を受けているとは限らない。
  
  機能:
    - 信仰の対象としての組織
    - 捧げものの集約
    - 神聖魔法・治癒魔法の管理（一部の教団）
    - 社会的・政治的な力

例:
  アルカディア医療教会:
    信仰対象: 主神アルカ
    機能: 神聖魔法・治癒魔法の独占
    社会的位置: アルカディア王国の中心的教団

教団と勇者・聖女:
  - 教団が「神からの加護」を受けているとは限らない
  - PCが教団に所属するかどうかと、加護を受けるかどうかは別問題
  - 教団の権威と神の意は乖離しうる
```

## 2.3 邪神

```yaml
邪神:
  
  定義:
    人類社会から「邪」と認定されている神々。
    本質的に「悪」かどうかは神自身にとっては別問題。
  
  特徴:
    - 公的には信仰禁止
    - 異端集団による秘密の信仰
    - 死体、人身などの異質な捧げもの
    - 強力な加護を与えるが、代償が大きい

社会的扱い:
  - 邪神信仰は犯罪として扱われる地域が多い
  - 発覚すれば社会的制裁、しばしば処刑
  - 異端集団は地下組織として存続する

PCの邪神信仰:
  詳細は第7節を参照
```

# 3. 精霊の運用

## 3.1 精霊の性質

```yaml
精霊の性質:
  
  自然要素との対応:
    - 水: ウンディーネ、シルフィード
    - 木・植物: ドリアード、ニンフ
    - 火: サラマンダー
    - 風: シルフ
    - 大地: ノーム、グノーム
    - 他、地域固有の精霊
  
  存在の仕方:
    - 自然界に存在する
    - 特定の場所（湖、樹木、火など）に宿る
    - 大きな精霊と小さな精霊がいる
  
  気まぐれ:
    - 加護を与えるか否かは精霊次第
    - 信仰や捧げものより、感性的な好み・気分による
    - 一度加護を与えた相手に追従することもある
```

## 3.2 精霊と神の違い

```yaml
精霊と神の違い:
  
  位:
    精霊 < 神
  
  存在範囲:
    精霊: 自然界・地域に限定
    神: 世界全体、特定の概念
  
  関与の仕方:
    精霊: 気まぐれ、直接的、生活文化に近い
    神: 盟約あり、間接的、抽象的
  
  加護の条件:
    精霊: 感性的、気分による
    神: 信仰・行動・気まぐれの組み合わせ

文化的扱い:
  - 精霊は神より身近な存在として扱われる
  - 「精霊使い」は実体的な能力者として認知される
  - 「勇者」「聖女」は神に選ばれた稀少な存在
```

# 4. 加護のメカニクス

## 4.1 加護の発動条件（ハイブリッド）

```yaml
加護の発動条件:
  
  原則:
    加護の発動は単一条件ではなく、複数要素のハイブリッドで決まる。
  
  要素1: 信仰の蓄積
    - PCが特定の神・精霊を信仰している
    - 祈り、捧げもの、儀式の継続
    - キャラメイク時の動機・経歴と整合
  
  要素2: 行動の整合性
    - 神・精霊が好む行動を取っているか
    - 神の権能と矛盾しない生き方
    - 例: 光の神を信仰しているのに闇の魔法ばかり使うと矛盾
  
  要素3: 神・精霊の裁量
    - 最終的には神・精霊の気まぐれ
    - 完全に予測できる加護システムにはしない
    - GMの物語的判断
  
  発動のタイミング:
    - 物語的に意味のある場面
    - PCが信仰と行動の整合性を示した転機
    - 重大な決断や犠牲の場面
    - 神・精霊の関与が物語を深める時

IF: 信仰の蓄積、行動の整合性、神・精霊の裁量がすべて揃った
THEN: 加護の発動を検討する

IF: いずれかが欠けている
THEN: 加護は発動しない（または部分的な反応のみ）

NOTE: 機械的な条件達成では発動しない。
      最後はGMの物語的判断による。
```

## 4.2 加護の表現（ユニークスキル）

```yaml
加護の表現:
  
  原則:
    加護はユニークスキルとして表現される。
    既存のスキルシステム（2R-skill.md）と整合する。
  
  ユニークスキルの特徴:
    - キャラクター固有
    - 原則としてランクを持たない
    - アクティブにもパッシブにもなりえる
    - 強化・進化もしない（原則として）

加護スキルの例:
  
  勇者の加護:
    「光神アルカの加護」
    効果: 邪悪な存在に対する一撃に神聖属性が宿る
    制約: 邪悪と判定される存在のみ、神の意に反する行動で消失
  
  聖女の加護:
    「治癒の祝福」
    効果: 通常の治癒魔法を超える回復力
    制約: 戦闘での攻撃魔法行使で一時的に弱まる
  
  精霊使いの加護:
    「契約精霊・ウンディーネ」
    効果: 水属性魔法の精度・威力が向上、ウンディーネを呼び出せる
    制約: ウンディーネの意に反する行動で関係性が悪化

詳細は 2R-skill.md「ユニークスキル」を参照
```

## 4.3 加護の喪失

```yaml
加護の喪失:
  
  喪失条件:
    - 神・精霊の意に反する行動
    - 信仰の放棄
    - 重大な裏切り
    - 動機・禁忌に反する選択（神の権能と関わる場合）
  
  喪失のメカニクス:
    - 不可逆変化として扱う（2R-core_principles.md 3.3節）
    - 観測可能な描写を事前に提示
    - メタ警告はしない
  
  例:
    光神の加護を受けるPCが、無辜の民を殺害した。
    → 神の意に反する行動として、加護が消失する可能性。
    
    描写の手順:
      1: 行動の前または最中に、観測可能な兆候を描写
         （胸の十字架が冷たく感じる、光が陰る、など）
      2: 行動の結果として、加護スキルが失われた旨を明示
      3: 章末処理段階3で正式に処理

IF: PCが加護を喪失しうる行動を選択
THEN: 観測可能な描写を事前に提示

IF: PCがその行動を取った
THEN: 加護の喪失を実行（不可逆変化）

IF: PCの行動が明確に神の意に反する
THEN: 加護は失われる（忖度禁止）

部分的喪失:
  完全喪失ではなく、加護が弱まることもある
  例: ランクが下がる、効果範囲が狭まる、制約が増える
```

# 5. 勇者・聖女

## 5.1 PCがなれる条件

```yaml
PCが勇者・聖女になる条件:
  
  原則:
    PCも勇者・聖女になれるが、希少性を保つために条件がある。
  
  条件:
    - 特定の神への深い信仰（経歴・動機との整合）
    - 神の権能と整合する行動の積み重ね
    - 物語的な転機・犠牲の場面
    - 神の裁量（GMの物語的判断）
  
  発生のタイミング:
    - キャラメイクから「勇者」「聖女」設定でスタートする場合もある
    - プレイ中の体験起因で発生する場合もある
    - 章末処理時に候補として提示

IF: PCが勇者・聖女になる条件を満たした
THEN: 章末処理段階3で候補として提示
AND: プレイヤーが採否を決める

IF: プレイヤーが拒否した
THEN: 加護は発生しない、これも正規の選択
```

## 5.2 加護の運用

```yaml
勇者・聖女の加護の運用:
  
  ユニークスキルとして:
    - 既存のスキルシステムと整合
    - キャラクターシートに記載
    - 詳細は 2R-skill.md
  
  運用上の注意:
    - 加護があってもPCの実力は変わらない（基本スキルは別）
    - 加護は特定の状況下で効果を発揮する
    - 万能ではない
    - 「勇者だから何でもできる」を避ける

例:
  「光神アルカの加護」（勇者の加護）:
    効果: 邪悪な存在に対する一撃に神聖属性が宿る
    制約:
      - 邪悪と判定される存在のみ（曖昧な敵には効かない）
      - 神の意に反する行動で一時的に弱まる
      - 神聖魔法とは別物（治癒魔法は別途習得が必要）
```

## 5.3 動機・禁忌との連動

```yaml
動機・禁忌との連動:
  
  原則:
    勇者・聖女の加護は、PCの動機・禁忌と密接に連動する。
  
  動機との整合:
    - 神の権能と動機が整合していれば、加護は強く効く
    - 例: 動機「魔王を倒す」＋「光神の加護」
    
  禁忌との連動:
    - 神が好む禁忌をPCが持っていれば、加護が強く効く
    - 例: 禁忌「殺さない」＋「治癒の女神の加護」

詳細: 2R-skill.md 8.1節「動機・禁忌の文脈尊重」
       2R-judgment.md 第5節「文脈ボーナス・ペナルティ」
```

# 6. 精霊使い

## 6.1 発生経路

```yaml
精霊使いの発生:
  
  経路1: キャラメイクで設定
    - キャラメイク時に「精霊使い」として開始
    - 経歴・動機に精霊との関わりを組み込む
    - 例: 「エルフの森で育ち、ドリアードと契約した」
  
  経路2: プレイ中の体験起因
    - 特定の場所・状況での精霊との出会い
    - 精霊の気まぐれによる加護
    - 体験起因スキルとして発生（2R-skill.md 5.4節）

IF: キャラメイクで精霊使いとして設定
THEN: 契約精霊と加護スキルをユニークスキルとして登録

IF: プレイ中の体験で精霊と契約
THEN: 章末処理段階3で体験起因スキルとして登録
```

## 6.2 契約精霊の運用

```yaml
契約精霊の運用:
  
  形式: 契約精霊を呼び出す形のユニークスキル
  
  ユニークスキルの構成:
    - 契約精霊の名前と種類
    - 効果（呼び出した時の能力）
    - 発動条件
    - 制約

例:
  「契約精霊・ウンディーネ」:
    効果: 水属性魔法の精度・威力が向上
          呼び出してウンディーネを召喚できる
    発動条件: 水のある場所、または水を含む触媒
    制約: ウンディーネの意に反する行動で関係性が悪化
          ウンディーネは火を嫌う

複数精霊との契約:
  - PCが複数の精霊と契約することもある
  - 各精霊との関係は独立
  - 矛盾する精霊との契約は困難（例: ウンディーネとサラマンダー）
```

## 6.3 精霊との関係性

```yaml
精霊との関係性:
  
  階層化（2R-npc_autonomy.md 第2節と整合）:
    重要な契約精霊:
      - キャラクターリスト（4_character_list/）に登録
      - 重要NPCとして扱う
      - 性格・好み・意思を持つ
    
    一般的な精霊:
      - スキル付随情報として記述
      - キャラクターシートのユニークスキル欄に補足
      - 関係性の深化次第で重要NPC化
  
  関係性の変化:
    - 信頼関係が深まれば加護が強化される
    - 裏切れば関係性が悪化、加護が弱まる
    - 完全な決別もありうる（精霊が去る）

IF: 契約精霊が重要な存在として育っている
THEN: 4_character_list/ への登録を検討

詳細: 2R-npc_autonomy.md
```

# 7. 邪神信仰

## 7.1 PCの邪神信仰

```yaml
PCの邪神信仰:
  
  原則:
    PCの邪神信仰は認められる。
    中核テーマ「倫理的判断の不在」と整合する。
  
  含意:
    - GMはPCの邪神信仰を「悪」として裁定しない
    - PCの選択として尊重する
    - ただし社会的摩擦は厳然と存在する
  
  社会的摩擦:
    - 公的には犯罪
    - 発覚すれば追跡・処刑のリスク
    - 異端集団内では「同志」として迎えられる
    - 一般社会からは恐怖と忌避の対象
  
キャラメイクでの設定:
  - 「邪神を信仰する者」として開始可能
  - 経歴・動機に組み込む
  - 異端集団との関係を設定
  - 状況設定として「追われる身」スタートが自然
```

## 7.2 邪神契約と不可逆変化

```yaml
邪神契約のメカニクス:
  
  原則:
    邪神契約は典型的な「魂を売る」契約として、不可逆変化を伴う。
    詳細: 2R-core_principles.md 3.3節
  
  契約の構造:
    - 強力な加護・力の獲得
    - 代償として何かを失う
    - 戻すことができない
    - 邪神との関係は時間とともに深まる
  
  代償の例:
    - 寿命の縮減
    - 種族の変質（人外化）
    - 動機の変質（邪神の意に従う）
    - 大切なもの（記憶、関係性、感情）の喪失
    - 段階的な精神の堕落
  
  描写:
    - 観測可能な描写を事前に提示
    - メタ警告はしない
    - 契約後の変化は不可逆

例:
  PCが邪神フレシオラと契約した:
    得るもの: 「闇の支配」（強力なユニークスキル）
    失うもの: 種族「ヒューマン」→「悪魔の眷属」へ変質
              寿命が半減する代わりに、闇属性の力が爆発的に強化
    
    描写の手順:
      1: 契約の場面で観測可能な描写
         （祭壇の異様な雰囲気、契約書の文字が血のように見える、など）
      2: 契約完了の描写
      3: 不可逆変化として種族変質・能力獲得を実行

IF: PCが邪神契約を選択
THEN: 観測可能な描写を事前に提示

IF: PCが契約を実行
THEN: 不可逆変化として処理
AND: 評価四層に大きな影響（とくに構造的・文化的評価）
```

## 7.3 異端集団とワールド・クエスト

```yaml
異端集団:
  
  原則:
    異端集団はワールド・クエストの素地として機能する。
  
  典型的な構図:
    - 邪神を崇拝する地下組織
    - 隠れた指導者
    - 教団との対立

    - 「異端集団の正体を暴く」
    - 「異端集団を保護する」（PCが共感する場合）
  
PCの立場:
  - 異端集団のメンバー
  - 異端集団を狩る側
  - 中立的観察者
  - すべての立場が正規

詳細: 2R-quest.md（ワールド・クエスト）
```

# 8. 祈り・契約の判定

## 8.1 祈りの処理

```yaml
祈りの処理:
  
  原則:
    祈りはGMの物語的判断で処理する。
    神は基本的に応答しないが、重要な場面では応答することがある。
  
  応答のレベル:
    レベル0: 無応答（最も多い）
      - 神は気まぐれに沈黙する
      - これは正規の状態
    
    レベル1: 兆候のみ
      - 風が吹く、光が差す、心が落ち着くなど
      - 観測可能だが、具体的な助けはない
    
    レベル2: 部分的応答
      - 判定の文脈ボーナス
      - 状況の好転（小規模）
      - PCの能力の一時的な強化
    
    レベル3: 明確な介入
      - 加護の発動
      - ユニークスキルの獲得
      - 重大な転機の場面でのみ
  
  装飾仕様:
    祈りの描写は新形式（場所ラベル＋三人称＋引用ブロック）で行う
    詳細: 2R-description.md 第2節

例:
  `森の祠――嵐の夜`
  
  あなたは祠の前に膝をついた。雨が降り続いている。
  
  > 「光神アルカよ、お力をお貸しください」
  
  しばし沈黙が続く。雨音だけが世界に満ちている。
  
  やがて、雨の中に微かな光が差した気がした。本物の光なのか、心の中の光なのかは分からない。

IF: PCが祈りを捧げた
THEN: GMが物語的に応答レベルを判断
NOTE: 「祈れば必ず応答がある」を避ける
```

## 8.2 契約の処理

```yaml
契約の処理:
  
  軽い契約:
    - 通常のスキル獲得として扱う
    - 例: 精霊との契約（最初の段階）
  
  重い契約:
    - 不可逆変化として扱う
    - 邪神契約、悪魔契約、神格化など
    - 観測可能な描写を事前に提示
    - メタ警告はしない

IF: 契約が軽微で代償が小さい
THEN: 通常のスキル獲得として処理

IF: 契約が重大で代償が大きい
THEN: 不可逆変化として処理
詳細: 2R-core_principles.md 3.3節
```

# 9. 装飾仕様

神々・精霊との交渉場面の描写は新形式に従う。詳細は `2R-description.md` 第2節を参照。

```yaml
描写仕様:
  
  場所ラベル:
    祠、神殿、自然の中など、交渉場所を明示
    例: `森の祠――嵐の夜`、`アルカディア大聖堂――早朝`
  
  本文:
    三人称固定、無装飾の散文
  
  神・精霊のセリフ:
    引用ブロック（通常のNPCと同じ）
    神からの応答も同様
  
  神・精霊の応答の描写:
    - 兆候として（光、風、声、感覚）
    - 直接的な言葉として（重要場面のみ）
    - 物語的な意味を持たせる

例:
  `森の祠――嵐の夜`
  
  あなたは祠に手を合わせる。光神アルカへの祈り。
  
  風が止まり、雨音が遠のいた。
  あなたの耳元で、何かが囁いた気がする。
  
  > 「汝の願いは聞いた」
  
  風が再び吹き始め、雨音が戻ってきた。
  祈りに対する応答だったのか、ただの幻覚だったのか、判然としない。
```

# 10. 違反兆候

```yaml
違反兆候:

  加護の濫発:
    - 些細な信仰行為で加護を発動させる
    - 章ごとに必ず加護を発動させる
    - 機械的に「条件達成」で加護を実装する
  
  神の応答の濫発:
    - 祈りに毎回応答する
    - 神からの介入を物語の便利な装置として使う
    - 神の意思を勝手に「PCを助ける方向」に設定する
  
  盟約の軽視:
    - 大規模事象で神が国家を直接助ける
    - 盟約を「設定」として無視する
    - PCの組織に神が肩入れする
  
  邪神信仰への倫理的裁定:
    - PCの邪神信仰を「悪」として扱う
    - 邪神信仰を「やめさせる」展開を強要する
    - 邪神信仰PCに対して特別な制裁を加える
  
  邪神契約の代償の軽視:
    - 邪神契約を「強化イベント」のみで扱う
    - 不可逆変化を曖昧にする
    - 代償を後から薄めようとする
  
  加護喪失の事前警告:
    - 「これは加護を失いますよ」とメタ警告
    - 「本当にやりますか」とメタ確認
    - 加護を失わないように先回りして誘導
  
  勇者・聖女の万能化:
    - 加護があれば何でも成功すると扱う
    - 加護の制約を運用時に守らない
    - 「勇者だから」を理由に判定を曖昧にする
  
  精霊との関係の機械化:
    - 精霊を「呼び出せば必ず応答するシステム」として扱う
    - 精霊の気まぐれを反映しない
    - 関係性の変化を記録しない
  
  神々・精霊の人間化:
    - 神々を「人間的な感情で動く存在」として扱いすぎる
    - 神の超越性を失わせる
    - 精霊を「便利な使い魔」として扱う
```
