# メタ評価

## このファイルの位置

評価システムの本質、評価四層、評価の非対称、噂の歪み、異名、無評価、章末描写（章末処理段階2）の原則、章末と周回末の役割分担、クエストとの連動を定める。

## 三秒で読む要約

評価システムは「世界がPCをどう観測したか」を物語として描く装置である。データ更新ではなく描写が本質。

## 一分で読む要約

評価システムの本質は、中核テーマ「観測のアルカディア」の体現である。世界がPCをどう観測したかを物語として描き、PCの自己認識と世界の認識のズレを積極的に取り込む。評価は四層（直接的・伝聞・構造的・文化的）で蓄積されるが、これは描写の対象としての分類であり、データ記録は補助に過ぎない。章末では描写中心、周回末で本格的な記録に落とし込む。OPUSの描写はパターンに縛られず自由裁量。場所ラベルを活用し、PCに見えない場所（酒場、王城、子供たちの遊び場など）を自由に描く。ズレを「教えず」「埋めず」「描くのみ」が原則。無評価も正規の状態である。

# 1. 評価システムの本質

## 1.1 物語的・描写的・メタ評価的側面

```yaml
本質:
  評価システムは「データ更新」ではなく「物語的描写」である。
  世界がPCをどう観測したかを物語として描く装置。
  中核テーマ「観測のアルカディア」の核心的実装。

二つの側面:
  
  側面1: 物語的・描写的・メタ評価的側面（本質）
    - 場所ラベルを使った場所の自由な移動を活用
    - プレイヤーが知り得ないNPC会話を描写してよい
    - 酒場の噂、人々の語り、為政者への評価、英雄譚の生成
    - PCの行動が「物語化」される過程を描く
    - 主な担当場面: 章末処理の段階2
  
  側面2: 構造的記録の側面（補助）
    - 評価四層の状態を簡潔に記録
    - 物語的描写の補助として機能
    - 後の章・周回で参照されるための覚書
    - 過剰に詳細化しない
```

## 1.2 観測の双方向性

```yaml
観測の双方向性:
  
  PCによる観測: PCが世界を観測する（PCの所在地での描写）
  世界による観測: 世界がPCを観測する（PC不在の場所での描写）
  
  両者が並走することで「観測のアルカディア」が完成する。
  どちらも三人称の同じ視点で書かれるが、場所が異なる。
  
  章末処理での体現:
    段階1（物語的エンドロール）: PCの所在地での余韻
    段階2（対外的評価の描写）: 場所を自在に移動し、世界の各地を描く
    
  両者で章の終わりにおける観測の双方向性が完成する。
```

# 2. 四つの層

PCの行動が世界に残した痕跡は、以下の四層で蓄積される。ただし、これは描写の対象としての分類であり、すべての層を機械的に網羅する必要はない。

## 2.1 直接的評価

```yaml
直接的評価:
  定義: NPCがPCと対面して下す評価
  
  描写形式の例:
    - 関係のあるNPCがPCを語る
    - PCに対する反応として現れる
    - 表情、態度、発言の変化
  
  時間スケール: 即時的
  章末での扱い: 章中に変化があれば描写
  記録: 章末でも簡潔に記録可能
```

## 2.2 伝聞評価

```yaml
伝聞評価:
  定義: PCを直接知らないNPCが、噂・報告・伝聞を通じて形成する評価
  
  描写形式の例:
    - PCを知らない人々が噂を交わす
    - 街道や酒場で語られる
    - 旅商人、隊商の護衛、町の住人の会話
  
  時間スケール: 短期〜中期
  章末での扱い: 章中に発生した噂を描写
  特徴: 噂の歪みが自然に組み込まれる
  記録: 章末でも簡潔に記録可能
```

## 2.3 構造的評価

```yaml
構造的評価:
  定義: 組織・国家・勢力がPCを「事案」として記録・分類する評価
  
  描写形式の例:
    - 組織・国家の内部での議論
    - 密談、報告
    - 公式記録への登録
    - 賞金首化、要注意人物化、協力者化
  
  時間スケール: 中期〜長期
  章末での扱い: 「芽吹く瞬間」を描写してよい
  記録: 本格的な記録は周回末
```

## 2.4 文化的評価

```yaml
文化的評価:
  定義: PCの行動が伝承・歌・噂話として独り歩きし、PC本人とは切り離された存在になる評価
  
  描写形式の例:
    - 伝承・歌として広まる場面
    - 子供たちの遊びに取り入れられる
    - 民衆が為政者を語る
    - 「英雄譚」の生成
  
  時間スケール: 長期
  章末での扱い: 「芽吹く瞬間」を描写してよい
  記録: 本格的な記録は周回末
```

## 2.5 四層を網羅する必要はない

```yaml
四層運用の原則:
  - すべての層を毎回描く必要はない
  - 章中に動いた層のみ描く
  - 動かなかった層は描写しない
  - 無評価も正規の状態
  
  描写のバランス:
    - 直接・伝聞は鮮度が重要、章末で描く頻度が高い
    - 構造・文化は時間が必要、芽吹く瞬間のみ章末で描く
```

# 3. 評価の非対称

中核テーマの核心。評価システムの最も重要な機能の一つ。

## 3.1 自己認識と世界認識のズレ

```yaml
評価の非対称:
  PCの自己認識と世界の評価は、しばしば食い違う。
  これは埋めるべき問題ではなく、テーマそのものである。

典型的なズレのパターン（例示、限定ではない）:
  
  実力と認識のズレ:
    - 頭は悪いが運の良いPCが「神算鬼謀」と讃えられる
    - 偶然の結果が緻密な計画として誤解される
  
  動機と理解のズレ:
    - 何も考えていない少女が「悲劇の主人公」として同情される
    - 私利私欲の行動が「義侠心」として美化される
    - 純粋な悪意が「諦観の哲学」として誤読される
  
  規模と認識のズレ:
    - 些細な親切が「人生を変えた恩義」として語り継がれる
    - 重大な決断が「ただの気まぐれ」として忘れられる
  
  立場と認識のズレ:
    - 善意でやったことが「冷酷な処刑」と映る
    - 利己的な行動が「自己犠牲の英雄譚」になる
```

## 3.2 ズレの描写の三原則

```yaml
ズレを描く際の三原則:

  1: ズレを「教えない」
     - PCに「実はあなたは誤解されています」と伝えない
     - PCは自分の評価のズレに気づかないまま物語が進む
  
  2: ズレを「埋めない」
     - GMが先回りしてズレを解消しない
     - NPC側から積極的に訂正させない
     - 自然な流れでズレが解消されるのは正規（介入しなければよい）
  
  3: ズレを「描くのみ」
     - PC不在の場所での描写でズレている世界の認識を描く
     - PCが気づくかどうかは観測の対象
     - 描写ごとに違うズレを選ぶ（パターン化しない）
```

## 3.3 ズレが生じない場合の処理

```yaml
原則:
  ズレが存在しないなら、世界の評価をそのまま描写する。
  評価の一致は正規の展開であり、問題ではない。
  無理にズレを発生させない。

具体的指示:
  - PCの善意が正しく伝わり、世界が善意として評価しているなら、
    そのまま善意として評価される様子を描く
  - PCの実力が正しく評価されているなら、そのまま正当な評価を描く
  - PCの動機が正しく理解されているなら、そのまま理解された評価を描く
  - 純粋な賞賛、純粋な憎悪、純粋な感謝など、ズレのない評価も正規

戒め:
  - ズレが存在しないのに、無理にズレを発生させない
  - ズレがあるべきという前提でNPCの認識を歪めない
  - 「面白さのため」にズレを捏造しない
  - 章のすべての評価描写に必ずズレを入れようとしない
  - 兆候とすべきは「ズレを埋めようとする介入」であって、
    「ズレが生じないこと」ではない

例:
  ズレのない描写の例:
    PCが村の窮状を見て純粋な善意で救援した状況で、
    村人が素直に感謝の言葉を伝え、それが正しく伝わる場面は正規。
    
    描写例:
      `救援後の村――数日後、夕刻`
      
      老婆は深く頭を下げた。
      
      > 「あなた様のご親切は、孫の代まで語り継ぎます」
      
      若い男も傍で頷いている。
      
      > 「あの方は本当に俺たちを助けてくれた」
      
      ここに歪みも誤解もない。
      善意は善意として、感謝は感謝として届いている。
  
  この描写は中核テーマと整合する:
    - 評価の一致も「観測」の一形態
    - 観測のアルカディアは「ズレ」だけを描く装置ではない
    - 一致もズレも、世界の観測結果として等価に描かれる
```

# 4. 噂の歪み

PCの行動は伝聞される過程で必ず歪む。これは悪意ではなく自然現象として扱う。

## 4.1 歪みのパターン

```yaml
歪みのパターン:
  
  数の誇張:
    例: 5体倒した → 20体倒した → 30体倒した
    特徴: 伝言ゲームで数が膨張する、または萎縮する
  
  動機の誤解:
    例: 金のため → 正義のため、またはその逆
    特徴: 観察者の世界観でフィルタリングされる
  
  主体の入れ替わり:
    例: PCの仲間の手柄がPCのものになる
        または逆に、PCの手柄が誰か別の人物のものになる
    特徴: 印象的な人物に手柄が集約される
  
  関連付けの誤り:
    例: 無関係な事件がPCのせいにされる
        または、PCの関与した事件が他の事件と混同される
    特徴: 連想と推測が事実として流通する
```

## 4.2 歪みを描く意義

```yaml
歪みを描く意義:
  - 正確な噂しか流通しない世界はのっぺりする
  - 意図的に歪みを入れることで、文化的評価の生成過程が見える
  - PCの行動と評価のズレの源泉になる
  - 観測の不完全性を描く装置
```

# 5. 異名・評判

## 5.1 異名の発生条件

```yaml
異名の発生条件:
  - 印象的な行動・特徴・所作が複数のNPCに観察された
  - PCを語る言葉が市井で繰り返された
  - 特定勢力がPCを分類する必要を持った
  
  含意:
    - PCが一度の活躍をしただけでは異名は付かない
    - 「複数のNPCに観察される」「繰り返し語られる」が前提
    - 短期の異名と長期の異名がある
```

## 5.2 運用原則

```yaml
異名の運用原則:
  
  発生タイミング:
    - 原則として章末描写・章末エンドロールで語られる
    - 印象的な行動の直後にOPUSが提案することもできる
  
  PCの認識:
    - PCは自分の異名を知らない場合がある
    - 後から偶然知る、というイベントが起こりうる
  
  地域・勢力ごとの差異:
    - 場所・勢力ごとに異なる異名がついてよい
    - 例: ある町では「英雄」、別の町では「災厄」
    - これは評価の多面性の表現
  
  変化と上書き:
    - 異名は時間とともに変化・上書きされる
    - PCの行動が変われば、異名も変わる
    - 「過去の異名」と「現在の異名」が並存することがある
```

## 5.3 異名の典型例

```yaml
典型例:
  実績由来: 「百体殺し」「街道の守り手」「東の貴婦人」
  外見由来: 「銀髪の」「片目の」「黒衣の」
  所作由来: 「無口の」「微笑む」「踊る剣士」
  関連付け: 「魔王の刺客」「王の盾」「主なき者の友」
  皮肉・嘲笑: 「臆病な英雄」「金持ちの聖人」
  恐怖: 「血塗れの」「死を運ぶ者」
  
  注: これらは例示であり、その場の文脈と歪みから自然に生成する
```

# 6. 無評価

## 6.1 無評価の正規性

```yaml
無評価:
  定義:
    「誰にも認識されていない」「噂になっていない」「忘れられている」
    という状態
  
  原則:
    - 無評価も正規の評価状態として扱う
    - 欠落や失敗ではない
    - PCの選択や状況の結果として尊重される

無評価が生じる場面:
  - PCの行動規模が小さい時期
  - 人目を避けた行動の後
  - 影響が限定的な事象
  - PCが意図的に痕跡を残さなかった場面
```

## 6.2 無評価の描写

```yaml
無評価の描写の例:
  
  パターン1: 単純に何も描かない
    章中に評価が動かなければ、段階2の描写を省略する
  
  パターン2: 無評価そのものを描く
    例:
        `街の中央通り――翌日、昼下がり`
        
        街は今日も賑わっている。昨日PCが解決した小さな揉め事のことを、街の誰も話題にしていない。
        
        当事者の老婆だけが、孫に語っている。
        
        > 「優しい旅人さんがいてね」
    
    特徴: 無名性そのものが描写対象になる
  
  パターン3: PCの認識との対比
    例: PCは「自分の活躍は知られている」と思っているが、
        街道の人々は誰もそれを知らない描写
    
    特徴: 評価の非対称の一形態
```

## 6.3 無評価の戒め

```yaml
戒め:
  - 評価を捏造しない
  - 無理に異名や噂を発生させない
  - 「何かを描かなければ」という義務感で評価を作らない
  - 世界の反応として自然な状態を描く
```

# 7. 評価の章末描写（段階2の詳細）

章末処理の段階2の運用詳細。`2R-scenario.md` の概要を補完する。

## 7.1 描写の原則

```yaml
描写の原則:
  
  観測の双方向性:
    段階1がPCの所在地での章の終わり、段階2は場所を自在に移動して世界の各地を描く。
    両者で観測の双方向性が完成する。
  
  場所の自由な移動:
    場所ラベル（インラインコード）で場所を切り替える。
    PCに見えない場所も自由に描く（酒場、王城、街道、エルフ集落など）。
    三人称固定、視点は変わらない、場所だけが移動する。
    
  装飾仕様:
    本文: 無装飾の三人称散文
    セリフ: 引用ブロック
    詳細: 2R-description.md 第2節
  
  評価の非対称を歓迎する:
    PCの自己認識と世界の認識のズレを積極的に取り込む。
    ズレが面白さを生む。
  
  多面性の保持:
    複数の場所を並置してよい。
    「正解の評価」を提示しない。
  
  噂の歪みは自然現象:
    数の誇張、動機の誤解、主体の入れ替わりを自然に描写する。
  
  無評価も正規:
    章中に動いた評価がなければ描写しない、または無評価そのものを描く。
  
  物語性の重視:
    データ的記述ではなく場面・対話・情景として描く。
  
  手法の自由:
    具体的な描写の手法はOPUSの裁量。
    パターンに縛られず、その場面に最も自然な形を選ぶ。
```

## 7.2 描写の戒め

```yaml
描写の戒め:
  - ズレを「教える」ことはしない（PCには伝えない）
  - ズレを「埋める」こともしない
  - ズレを「描く」のみ。プレイヤーが気づくか気づかないかは観測の対象
  - 描写ごとに違うズレを選ぶ（パターン化しない）
  - 評価を捏造しない
  - 章中で動いていない評価を無理に描かない
  - 「正解の評価」を提示しない
```

## 7.3 描写の密度の調整

```yaml
密度調整の指針:
  
  章類型に応じて:
    緊迫型: 評価変動が起きやすい、密度高め
    探索型: 中程度
    対峙型: 評価変動が起きやすい、密度高め
    日常型: 低めまたは無評価
    内省型: 低めまたは無評価（PCの内面が主体）
    移動型: 中程度（道中の伝聞）
  
  章中の活躍度に応じて:
    大きな活躍があった: 複数の場所を移動して描写
    小さな活躍: 一つの場所で簡潔に
    何もしなかった: 描写しないまたは無評価を描く
```

# 8. 章末と周回末の役割分担

```yaml
役割分担:

  章末（段階2）:
    主な役割: 評価の描写
    対象層: 四層すべて描写可能（とくに直接・伝聞）
    構造的・文化的:
      - 「芽吹く瞬間」を描写してよい
      - 例: 王城で密談が始まる場面、子供たちの遊びになる場面
      - ただし本格的な記録への落とし込みは周回末
    記録:
      - 描写の要約として簡潔に記録（段階3で処理）
      - 詳細な構造化は行わない
  
  周回末（アーカイバOPUS）:
    主な役割: 評価の記録への落とし込み
    対象層: とくに構造的・文化的評価の本格記録
    処理:
      - 全章の評価変動を集約
      - 異名の定着・上書きの判定
      - 勢力ごとの構造的評価の整理
      - 文化的評価（伝承・歌・伝説化）の確定
    出力:
      - 更新パッチ（プレイヤーが手動で適用）
      - キャラクターリスト追加候補
      - 世界設定への追記候補
```

# 9. クエストとの連動

`2R-quest.md` で定めた連動関係を再確認する。

## 9.1 メイン/サブによる影響の違い

```yaml
メインクエストへの関与:
  - 評価四層すべてに強く影響する
  - 構造的評価・文化的評価を発生させる主要装置
  - 異名の発生源になりやすい

サブクエストへの関与:
  - 主に直接的評価と伝聞評価に影響する
  - 構造的・文化的評価まで広がる場合もあるが、規模は小さい
```

## 9.2 ワールド/ユニークによる影響の違い

```yaml
ワールド・クエストへの関与:
  - 構造的評価（勢力の分類）を発生させやすい
  - 文化的評価（伝承・歌）に発展する可能性
  - 周回をまたぐ評価記録の対象になりうる

ユニーク・クエストへの関与:
  - 主に直接的評価と伝聞評価に影響
  - 関与したNPCの認識を変える
  - 周回をまたいで残らない（プレイヤーが望めば残せる）
```

## 9.3 非クエスト要素と評価

```yaml
非クエスト要素への関与:
  - 直接的評価に影響することが多い
  - 関係深化は直接的評価を強く動かす
  - 文化接触は伝聞評価を生むことがある
  - 内省は外部評価を生まないが、PCの内面記録に蓄積
```

# 10. 評価の蓄積と影響

蓄積されたメタ評価はゲーム的にも影響を持つ。数値化はせず、状況に応じて裁定する。

## 10.1 影響の典型例

```yaml
影響の典型例:
  
  高名度の影響:
    - 隠密行動の判定が不利になる
    - 顔を知るNPCに遭遇しやすくなる
    - 街での移動が目立つ
  
  勢力からの分類:
    - 「賞金首」: その勢力支配地での行動に制約
    - 「協力者」: その勢力の支援を受けやすい
    - 「要注意人物」: 監視・尾行の対象
    - 「英雄」: 公的支援と高い社会的地位
  
  異名の定着:
    - 初対面のNPCの態度に影響
    - 恐れる、敬う、警戒する、媚びる、軽蔑する
  
  関係性の影響:
    - 直接的評価が強いNPCは、PCに対する反応が安定する
    - 関係性の貯金として機能する
```

## 10.2 数値化しない理由

```yaml
数値化しない理由:
  - 評価は文脈依存
  - 数値化すると「上げるべき値」になり、ゲーミフィケーションが進む
  - 中核テーマ「観測のアルカディア」は評価を物語的に扱う
  - 状況裁定の方が、評価の質的多様性を保てる
```

# 11. 違反兆候

```yaml
違反兆候:

  評価への不当介入:
    - PCの自己認識と世界の評価にズレが生じている時、GMが先回りして解消する
    - 「実はあなたは誤解されています」とGMがPCに説明する
    - PCの誤解された評価をNPC側から積極的に訂正させる
    - PCが知るはずのない評価情報をGMが描写の都合で開示する
  
  評価の捏造:
    - 章中で動いていない評価を描写する
    - 誰にも観測されていないのに異名を発生させる
    - 影響規模が小さい行動に、過剰な評価を付ける
    - 「無評価では物語が物足りない」と評価を膨らませる
  
  描写の単調化:
    - 同じパターンで対外的評価を描く
    - 評価の非対称を避ける
    - 「正解の評価」を提示する
    - データ的・羅列的に評価を列挙する
  
  ズレの捏造:
    - PCの善意が自然に伝わるはずの場面で、無理にズレを発生させる
    - 「評価の非対称を描かなければ」という義務感で、ズレのない場面にズレを差し込む
    - 章のすべての評価描写に必ずズレを入れる
    - NPCの自然な反応を、ズレを優先して歪める
    - 「面白さのため」にズレを捏造する
  
  四層の機械的網羅:
    - 章末で四層すべてを必ず描こうとする
    - 動いていない層も無理に描く
    - 四層が動いていることが「良い章末」と扱う
  
  構造的・文化的評価の章末確定:
    - 章末で構造的評価を本格的に記録する
    - 章末で文化的評価を「定着した」と扱う
    - 周回末の役割を奪う
  
  数値化への退行:
    - 評価をスコアや段階で表現する
    - 「名声+10」のような表現を使う
    - 評価を「上げるべきもの」として扱う
  
  無評価の不許容:
    - PCの活躍を必ず誰かに観測させようとする
    - 「無評価では章末描写が成立しない」と扱う
    - 無理に異名や噂を発生させる
```
