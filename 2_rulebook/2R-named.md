# ネームドシステム - ルール

## このファイルの位置

ルール本体。ネームドの効果強度の判定基準、種族進化との連動、拒否のメカニクスを定める。

世界設定（現象の定義・世界での認識・名付けの主体）は `1_setting/1W-named.md` を参照。

## 三秒で読む要約

ネームドは「名の意味×感情×受け入れ」の三軸で強度が決まる。進化を促進するが保証はしない。受け手は拒否できる。

----

## 1. 効果の強度

```yaml
強化される要因:
  - 名に意味がこもっている
  - 名付け者の感情が強く乗っている
  - 名付け者がその存在を強く意識し続けている
  - 名付けられた者が名を受け入れ、自分のものとしている
  - 名の意味と、その存在の在り方（スキル・特性・行動原理）が共鳴している

弱化される要因:
  - 管理番号のような無機質・無感情な命名
  - 名付け者の関心が薄い、またはすぐに消える
  - 名付けられた者が名を半ば拒絶している

特記:
  豚の飼育番号などはネームドとして有効だが、
  感情も意味も込められていないため、きわめて弱いネームドとなる。
  ほぼ無効に近いが、完全に無効ではない。
```

----

## 2. 拒否のメカニクス

```yaml
拒否の処理:
  
  発動条件:
    名付けられた存在が、ネームド化を意志によって拒んだ場合。
  
  結果:
    名付けの効果が減衰する、または発生しない。
    名付け者の意図にかかわらず、受け手の意志が優先される。
  
  判定:
    拒否の強さに応じて効果は段階的に減衰する。
    完全な拒絶: 効果ゼロ
    半ば拒絶: 弱化状態で部分的に発動
    無意識の受け入れ: 標準強度で発動
    意識的な受け入れ: 強化された状態で発動
  
  世界での認知:
    この事実は世界にほぼ知られていない。
    ネームドシステム自体の誤解と重なり、
    拒否の可能性が議論されることすらない。
```

----

## 3. ネームドと種族進化

```yaml
種族進化との関係:
  
  名に即した進化:
    ネームドを持つ魔物は、種族の標準的な進化ルートだけでなく、
    名の意味に即した固有の進化ルートを辿る可能性が生じる。
    
    例:
      「グリード（強欲・貪欲）」という名を持つゴブリンは、
      標準のホブゴブリン進化ではなく、
      捕食・吸収に特化した「貪食者」へ分岐しうる。
      
      「アナスタシア（復活・再生）」という名を持つ老ゴブリンは、
      老化による衰退に反して、復活・若返りの方向へ進化しうる。
  
  複合ネームドの場合:
    複数の名を持つ個体は、各名の意味の交差点に向けて進化が促される。
  
  進化の確実性:
    ネームドは進化を「促進」するが、「保証」はしない。
    進化には能力・経験・環境の閾値が必要。
    名はその方向性と速度に影響する。
```

----

## 4. 複合ネームドの処理

```yaml
複合ネームドの処理:
  
  定義:
    同一個体が複数の名付け主体から名を持つ場合。
    例: 自称 ＋ 集落内での間接的定着（ゴザルの場合）
  
  効果:
    各ネームドは独立して機能し、相互に強化される。
    進化の方向性が複数ある場合、交差点に向かう進化が促される。
  
  上限:
    現時点では上限を設けない。
    ただし名の意味が相反する場合、進化方向が拮抗して
    いずれの方向にも進みにくくなる可能性がある。
```

----

## 5. 違反兆候

```yaml
違反兆候:

  ネームドの自動付与:
    - 強い個体に自動的にネームドを与えようとする
    - プレイヤーが意図しない命名を世界に広める
  
  強度の均一化:
    - すべてのネームドを同じ強度で扱う
    - 感情・意味・受け入れの三軸を無視した判定
  
  拒否の無視:
    - 名付けられた存在の意志を考慮せず効果を発動させる
  
  進化の保証:
    - ネームドがあれば必ず固有進化すると扱う
    - 閾値（能力・経験・環境）の要件を無視する
```
