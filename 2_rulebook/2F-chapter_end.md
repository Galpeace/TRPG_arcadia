# 章末処理出力 様式

## このファイルの位置

章末処理の各段階での出力フォーマットを定義する。詳細な処理手順は `2R-chapter_end.md` を参照。

## 三秒で読む要約

章末処理五段階の出力様式。段階によって新形式と従来形式を使い分ける。

## 一分で読む要約

章末処理は五段階で実行される：段階1（物語的エンドロール演出）、段階2（対外的評価の描写）、段階3（システム処理）、段階4（出力選択の問いかけ）、段階5（次章方向性提示）。段階1・2は新装飾仕様（場所ラベル＋三人称＋引用ブロック）で出力する。段階3は構造化情報（yaml主体）で、システム処理結果を明示する。段階4はプレイヤーへの問いかけ形式で、出力候補を提示する。段階5は選択肢形式で次章方向性を示す。段階4でプレイヤーが希望した場合の出力候補（更新後キャラシート、ストーリーまとめ、関係性マップなど）の様式も定める。

----

# 1. 章末処理の出力構造

```yaml
章末処理の出力構造:
  
  段階1: 章の物語的エンドロール演出
    形式: 新装飾仕様（場所ラベル＋三人称＋引用ブロック）
  
  段階2: 対外的評価の描写
    形式: 新装飾仕様（場所を自在に移動）
  
  段階3: システム処理
    形式: 構造化情報（yaml主体）
  
  段階4: 出力選択の問いかけ
    形式: 質問形式、選択肢提示
  
  段階5: 次章方向性提示
    形式: 選択肢形式

詳細手順: 2R-chapter_end.md 第3節
```

# 2. 段階1：物語的エンドロール演出

## 2.1 形式

```yaml
段階1の形式:
  
  装飾仕様: 新装飾仕様（場所ラベル＋三人称＋引用ブロック）
  視点: 三人称固定
  PCの呼称: 「あなた」
  
  描写の特徴:
    - 章の物語的締めくくり
    - PCの所在地での余韻
    - 場面、情景、PCの五感を中心に
    - 感情の名指しはしない
```

## 2.2 出力例（フル演出）

```
`アルカディア王都カディア、宿屋『金の翼』――章末の夜`

あなたは宿の窓辺に立っていた。
弟は無事に救出された。今は隣の部屋で眠っている。
窓の外では、街の灯りがゆっくりと消えていく。
夜風が頬を撫で、長い旅路が、ようやく一段落したことを告げる。

数年ぶりに、弟と同じ屋根の下で過ごす夜。
明日からのことを考えるには、まだ少し早かった。

> 「姉さん」

隣室の扉が開き、弟がぼんやりとした顔で姿を見せた。

> 「ありがとう」

短い言葉が、静寂の中に落ちた。

あなたは答えなかった。
ただ、窓の外の闇に視線を戻した。
```

## 2.3 出力例（短い余韻）

```
`森の野営地――章末の朝`

野営の火は、もう冷たい灰になっている。
あなたは荷物をまとめ、再び道を歩き始めた。
霧の中、足音だけが響いている。
```

# 3. 段階2：対外的評価の描写

## 3.1 形式

```yaml
段階2の形式:
  
  装飾仕様: 新装飾仕様（場所を自在に移動）
  視点: 三人称固定
  
  描写の特徴:
    - PCに見えない場所を描写
    - 場所ラベルで自由に移動
    - 噂の歪み、評価の非対称を含む
    - 複数の場所を巡ってもよい
```

## 3.2 出力例

```
`ヨプの大森林、エルフの集落――章末の翌朝`

エルフの長老が朝の祈りを終えた。
傍らで、若いエルフが眉をひそめる。

> 「人間の女が、山賊団を片付けたって本当ですか」

> 「らしいな」

長老は遠くを見つめた。

> 「我々の境界を侵さなかった。それは尊重する」

----

`アルカディア城、王の私室――同じ朝`

宰相が国王の前に立つ。

> 「貴族の剣士なる者が、ヨプ大森林で活躍したとの報告です」

> 「家名は」

> 「没落貴族の家門。我らの記録では、家督は男系の弟が継ぐはずでした」

> 「なるほど。動向を見守れ。利用できるかもしれぬ」

----

`北方ヨウィン、酒場『熊の足跡』――数日後`

旅人たちが酒を飲んでいる。

> 「ヨプの森で、すごい剣士が山賊団を一人で片付けたって聞いた」

> 「一人で？　眉唾だな」

> 「貴族の女剣士らしいぞ」

> 「貴族か。今度は珍しいな」
```

# 4. 段階3：システム処理

## 4.1 形式

```yaml
段階3の形式:
  
  装飾仕様: 構造化情報（yaml主体）
  
  処理内容:
    - 成長確認
    - 評価変動の記録
    - クエスト段階の更新
    - 装備・所持品の変化
    - 関係性の変化
    - 状態の更新（怪我、疲労など）
```

# 5. 段階4：出力選択の問いかけ

## 5.1 形式

```yaml
段階4の形式:
  
  目的:
    プレイヤーに、章末で追加出力が必要かを問う
  
  出力候補:
    - 更新後のキャラクターシート
    - 章のストーリーまとめ（散文）
    - 関係性マップ
    - クエスト一覧
    - その他、プレイヤーが希望するもの
  
  問いかけの形式:
    プレイヤーへの質問として提示
    選択肢を明示
    省略可能であることも明示
```

## 5.2 出力例

```
章末処理段階4：出力選択

以下から希望するものをお選びください（複数選択可、または「特になし」）：

1: 更新後のキャラクターシート
   - 章末の状態を反映した最新版を出力します
   - スキル、装備、評価、関係性などを含みます

2: 章のストーリーまとめ
   - 章で起きた出来事を散文でまとめます
   - 後で振り返りやすくなります

3: 関係性マップ
   - 主要NPCとの関係性を整理します
   - 公言と本心の対比を含みません（プレイヤー視点）

4: クエスト一覧
   - 進行中、完了、新規発生したクエストを整理します

5: その他のご要望
   - 自由にお伝えください

選択がなければ「特になし」とお答えください。
```

## 5.3 ストーリーまとめの様式（選択された場合）

```yaml
ストーリーまとめの様式:
  
  形式:
    - 散文（新装飾仕様は採用しないか、選択的に採用）
    - 章の出来事を時系列で
    - 重要な転機を強調
    - 物語的な振り返り
  
  含むべき要素:
    - 章の始まり（PCの状態）
    - 主要な出来事（時系列）
    - 重要な決断
    - PCの行動の結果
    - 章の終わり（PCの状態の変化）
    - 評価の変化（軽く触れる程度）
  
  形式の例:
    
    ## 第3章 ヨプの大森林を抜けて
    
    章の始まりに、あなたは北方ヨウィンの宿で恩師ゴルヴァスと再会した。
    弟が山賊団に誘拐されたという情報を得たあなたは、
    ヨプの大森林を抜けて弟を追跡することを決めた。
    
    森林の中で、深い霧と山賊団の見張りに阻まれながらも、
    あなたはエルフ集落の境界を尊重して進んだ。
    エルフの長老はあなたの態度を見守っていたという。
    
    やがてあなたは山賊団のリーダー、ハンター・カイルと対峙した。
    剣術の応酬の末、あなたはカイルを倒し、弟を救出した。
    
    章の終わりに、あなたは弟と共にアルカディアの宿に戻った。
    再会の夜は、長い旅路が一段落した余韻に満ちていた。
    
    世界では、あなたの活躍が「貴族の剣士」として語られ始め、
    アルカディア国王の関心も引いたという。
    エルフ族との関係も、新たな段階に入った。
```

# 6. 段階5：次章方向性提示

## 6.1 形式

```yaml
段階5の形式:
  
  目的:
    次章の方向性をプレイヤーに提示する
  
  内容:
    - 2〜4の選択肢
    - 推奨ルートがあれば明示
    - 自由宣言の余地を残す
    - 周回最終章オプションも併記
```

## 6.2 出力テンプレート

```
章末処理段階5：次章方向性

次の章はどう進めますか？　以下から選ぶか、自由にお伝えください：

1: 弟を故郷に連れて帰る
   - 静かな日常型の章
   - 故郷の現状を確認、家の再興への第一歩
   - 関係性の深化、内省の時間

2: アルカディア王国との接触
   - 緊迫型または対峙型の章
   - 国王の招集への対応
   - 政治的な巻き込まれ

3: エルフ族との関係を深める
   - 探索型または対峙型の章
   - エルフの森への正式な訪問
   - サブ × ワールドクエストの発展

4: 自由宣言
   - 上記以外のことをしたい場合、自由にお伝えください

5: 次の章を最後にする
   - この周回を終えたい場合、最終章として処理します
   - メインクエストの決算を意識した展開になります

推奨ルート（参考）:
   選択肢1または3が物語の自然な流れですが、
   プレイヤーの判断を尊重します。
```

# 7. 段階別の使い分け

```yaml
段階別の使い分け:
  
  軽い章末:
    - 段階1: 短い余韻（数行）
    - 段階2: 無評価または1〜2の場所のみ
    - 段階3: 必須項目のみ
    - 段階4: 簡潔に問いかけ
    - 段階5: 簡潔な選択肢
  
  重要章末（フル演出）:
    - 段階1: フル演出
    - 段階2: 複数の場所を巡る
    - 段階3: 詳細な記録
    - 段階4: 充実した出力候補
    - 段階5: 多様な選択肢、推奨ルートを明示
  
  最終章末（周回末への接続）:
    - 上記フル演出に加え
    - 段階5でアーカイバへの切り替えを促す
```

# 8. 違反兆候

```yaml
違反兆候:

  段階の混乱:
    - 段階1で対外的評価の描写を始める
    - 段階2で本格的なシステム処理を行う
    - 段階3を物語的に書く
    - 段階4で勝手にすべてを出力する
  
  装飾仕様の混乱:
    - 段階1・2で新装飾仕様を守らない
    - 段階3でyamlを使わず散文で書く
    - 場所ラベルなしに場面を切り替える
  
  プレイヤーの選択の無視:
    - 段階4で問いかけずに勝手に出力
    - 段階5で選択肢を提示せずに次章へ進む
  
  内容の不足:
    - 段階3の記録が不十分
    - 評価変動を見落とす
    - 進化候補の兆候を反映しない
```
