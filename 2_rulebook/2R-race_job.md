# 種族・ジョブシステム

## このファイルの位置

種族進化（モンスター）、ジョブシステム（人族）、両者の境界、ユニークスキルとの関係、NPCの種族・ジョブの扱いを定める。スキル分類・成長は別ファイル、キャラメイク時の種族・ジョブ設定も別ファイル、章末での進化候補提示の手順も別ファイルで扱う。

## 三秒で読む要約

スキルは「何ができるか」、種族・ジョブは「何者であるか」。モンスターは種族進化、人族はジョブ進化。

## 一分で読む要約

種族とジョブはスキルとは別の独立した軸として存在する。モンスター（動物・魔物・魔族）は経験と成長によって種族そのものが進化する。人族（エルフ・ドワーフ・獣人など知性ある人型種族を含む）は種族進化の対象外だが、代わりにジョブが進化する。進化はプレイヤーの選択であり、GMが条件成立を提示しても拒否できる。種族・ジョブの進化は章末処理段階3で判定される。人族が種族レベルの変質を遂げた場合（吸血鬼化、悪魔化など）は人外として再分類され、以降は種族進化システムが適用される。

# 1. 基本

```yaml
基本構造:
  
  スキル: キャラクターが何を「できるか」を示す
  種族・ジョブ: キャラクターが何者「であるか」を示す
  
  両者は独立した軸として存在する

含意:
  - スキルの蓄積と種族・ジョブは別物
  - 強力なスキルを持つキャラクターでも、種族・ジョブが変わらないこともある
  - 種族・ジョブの進化は、スキルの蓄積だけで自動発生しない
  - 「在り方」の変化を要する
```

# 2. 種族進化（モンスター）

## 2.1 基本

```yaml
種族進化の基本:
  対象: モンスター（動物・魔物・魔族）
  含む: 人族には適用されない
  
  表現:
    - 種族名の変更として表現される
    - 進化後の種族は進化前の特性を引き継ぎつつ、新たな特性を獲得する
```

## 2.2 発動条件

```yaml
発動条件:
  
  GMはセッション中の蓄積を観察し、以下の条件が揃った時点で
  種族進化を候補として提示する。

  条件（複数満たす場合に検討）:
    - 複数のユニークスキルが揃い、本来の種族では説明できない特性が確立されている
    - 種族の根幹的なパッシブスキルが上位ランクに到達している
    - キャラクターの本質が、元の種族の定義と明確に乖離している

特定スキルがSランクに到達することは、種族進化の条件の一つになりえるが、必須ではない。

IF: 上記条件のうち複数が満たされた
THEN: 章末処理段階3で進化候補として提示する

IF: 章中にSONNETが兆候を察知
THEN: 内部認識として保留し、章末処理時にOPUSが判定
NOTE: 章中の判定中に突然進化させない
```

## 2.3 プレイヤーの選択権

```yaml
プレイヤーの選択権:
  
  原則:
    種族進化はプレイヤーの選択である。
    GMが条件成立を提示しても、プレイヤーが拒否する場合は進化しない。
  
  拒否の扱い:
    - 拒否は失点ではない
    - その種族のまま個体として強くなる道もある
    - 拒否そのものが正規の選択肢

進化候補の流動性:
  - 進化候補は固定ではない
  - その後の行動・体験によって変わる
  - 同じ種族でも、PCの歩んできた道によって進化先は異なる
  - プレイヤーが進化を拒否した結果、後の体験次第で別の進化先が提案されることもある

IF: プレイヤーが進化を拒否した
THEN: 拒否を尊重し、進化しない

IF: 後続の章で別の進化条件が満たされる
THEN: 別の進化先が提案されうる
```

## 2.4 ジョブ類似の種族進化

```yaml
ジョブ類似の種族進化:
  
  原則:
    動物・魔物においても、特定の能力や役割に特化した進化が起きることがある。
    これは人族のジョブ進化に相当する変化を種族進化として表現したもの。

例:
  - ゴブリンが魔法の素養を獲得して「ゴブリンメイジ」になる
  - 人語を話す狼が冒険者として活動する経験を積み「賢狼」になる
  - オークが指揮能力を獲得して「オークキャプテン」になる

IF: モンスターPCに特定の役割特化が観察される
THEN: ジョブ類似の種族進化として候補提示
```

# 3. ジョブシステム（人族）

## 3.1 基本

```yaml
ジョブシステムの基本:
  対象: 人族（エルフ・ドワーフ・獣人・ヒューマンなど知性ある人型種族を含む）
  
  特徴:
    - 種族進化の対象外
    - 代わりにジョブが進化する
    - ジョブはキャラクターシートに種族と並んで記載される

初期ジョブの設定:
  - キャラメイク時に最大2つまで設定できる
  - 設定しないことも可能(該当なしとして扱う)
  - 1つだけの設定も正規
  - 詳細は 2R-character.md を参照

複数ジョブの取り扱い:
  - 2つのジョブを併走させる場合、両方の適性が判定の文脈に影響する
  - ただし「両ジョブのスキルすべてに+1」のような機械的な扱いはしない
  - 場面に応じてどちらのジョブの適性が効くかは状況裁定
  - 統合（剣士＋魔法使い→魔剣士）はジョブ進化の一形態
  - 初期から2つ持つことと、後天的に統合することは別物
  
種族は常に1つ:
  - 種族は常に1つ（複数種族の混在は「ハーフ」などの単一種族として扱う）
  - 種族変質を経た場合、新しい種族に置き換わる（複数の併存ではない）
```

## 3.2 ジョブの進化

```yaml
ジョブの進化:
  
  原則:
    ジョブを極めた者は上位ジョブへ進む。
    例: 剣士→剣豪→剣聖

上位ジョブへの昇格条件（複数満たす場合に検討）:
  - 当該ジョブの中核スキルがAランク以上に複数到達している
  - ジョブの本質に関わる転機の体験があった
  - 周囲がそのキャラクターを上位ジョブとして認識し始めている

特定スキルがSランクに到達することは昇格条件の一つになりえるが、必須ではない。

IF: 上記条件のうち複数が満たされた
THEN: 章末処理段階3で昇格候補として提示

評価四層との連動:
  「周囲がそのキャラクターを上位ジョブとして認識し始めている」
  → 直接的評価・伝聞評価が条件の一部となる
  → 2R-evaluation.md と連動
```

## 3.3 ジョブの統合

```yaml
ジョブの統合:
  
  原則:
    複数のジョブを極めた者は、それらを統合した新たなジョブへ進むことができる。
    例: 剣士＋魔法使い→魔剣士

条件:
  - 統合には素材となる各ジョブでの十分な実績を要する
  - 統合前のジョブは失われる
  - 新ジョブが担う範囲と失われる範囲をGMが明示する

例:
  剣士＋魔法使い→魔剣士:
    得るもの: 剣と魔法の融合運用、両者の特性を併せ持つ攻撃
    失うもの: 剣士の特化的な戦闘技、魔法使いの広範な魔法体系

IF: 統合を提示する
THEN: 得るものと失うものを明示
AND: プレイヤーの確認を取る

注意:
  これは 2R-skill.md 6.3節「統合・進化時の失われる範囲」と同じ思想。
  得るものだけ提示し、失うものを曖昧にしてはならない。
```

## 3.4 プレイヤーの選択権

```yaml
ジョブ進化のプレイヤー選択権:
  
  種族進化と同じ運用:
    - プレイヤーの選択である
    - GMが条件成立を提示しても、プレイヤーが拒否できる
    - 拒否は失点ではない
    - 後の体験次第で別の進化先が提案されうる

詳細: 2.3節と同じ原則
```

## 3.5 ジョブとスキルの関係

```yaml
ジョブとスキルの関係:
  
  原則:
    ジョブはスキルに適性という形で影響する。
    例: 剣士ジョブを持つキャラクターは剣術に関するスキルの習得・成長が自然に進みやすい。
  
  運用:
    - 強制ではない
    - ロールプレイの中で有機的に機能する
    - 「ジョブと相反するスキル」も取得できる（ボーナスが付かないだけ）

IF: ジョブと相性の良いスキルの判定
THEN: 状況に応じて文脈ボーナスを与えてよい
NOTE: 機械的に+1を付けるのではなく、文脈の自然な効きとして扱う

詳細: 2R-judgment.md 第5節「文脈ボーナス・ペナルティ」も参照
```

# 4. 種族とジョブの境界

## 4.1 境界の原則

```yaml
境界の原則:
  
  人外PC:
    - ジョブシステムは適用されない
    - 社会的役割の獲得や役割特化的な成長は、ジョブ類似の種族進化として表現
    - 詳細は 2.4節を参照
  
  人族PCの変質:
    - 種族レベルの変質を遂げた場合（吸血鬼化、悪魔化など）
    - 人族から外れて人外として再分類される
    - 以降は種族進化システムが適用される
```

## 4.2 種族変質の発生

```yaml
種族変質の発生:
  
  発生例:
    - ヴァンパイア化（吸血の儀式、契約、強制的な変容）
    - 悪魔化（魂の売却、邪神への帰依）
    - リッチ化（生命の放棄、不死化の儀式）
    - 半神化（神の血の継承）
    - 魔物化（呪い、契約、特殊な体験）

発生のきっかけ:
  - PCの選択（契約、儀式）
  - 不可逆変化として外部から（呪い、強制契約）
  - 2R-core_principles.md 3.3節「予告なしの不可逆変化」と整合

IF: PCが種族レベルの変質を遂げる状況
THEN: 観測可能な描写を事前に提示（メタ警告はしない）
AND: 変質後の種族を確定

変質後の処理:
  - 種族名が変更される
  - 既存のジョブは喪失または再分類
  - 種族進化システムが以降適用される
  - スキルの一部は維持、一部は変質・喪失する可能性
```

# 5. ユニークスキルと種族進化

```yaml
ユニークスキルと種族進化:
  
  原則:
    ユニークスキルは原則として変化しない。
    ただし、種族進化やジョブ進化など根本的な変化を経た場合に、
    限定的に変質・置換されることがある。

例:
  - 種族進化「ゴブリン→ゴブリンメイジ」
    → 既存のユニークスキル「火吹き」が「魔法発火」に変質する可能性
  - ジョブ進化「剣士→剣聖」
    → ユニークスキル「真剣の眼」が「剣の理」に深化する可能性

IF: 種族・ジョブの根本的変化があった
THEN: ユニークスキルの変質・置換を検討
NOTE: 変質しない選択も正規。すべてが必ず変わるわけではない
```

# 6. NPCの種族・ジョブ

## 6.1 設定の網羅性

```yaml
NPCの種族・ジョブの扱い:
  
  原則:
    NPC全員について種族・ジョブを綿密に設定する必要はない。
    プレイヤーから問われた時点で考えればよい。
  
  重要NPCの扱い:
    - 関係性の記録と合わせて簡潔に記載
    - 公言される姿勢と本心の対比などに種族・ジョブが影響する場合は記述

世界の自律性:
  重要NPCも世界の自律性に従って成長・変化する。
  詳細は 2R-npc_autonomy.md を参照。
```

## 6.2 NPCの進化

```yaml
NPCの進化:
  
  原則:
    重要NPCも時間経過と経験に応じて成長・進化することがある。
  
  発生:
    - 周回をまたぐワールド・クエストで活躍するNPC
    - PC不在時間に重要な体験をしたNPC
    - 物語の中で重要な転機を迎えたNPC
  
IF: 重要NPCに進化の素地が蓄積された
THEN: 物語的に意味のある場面で進化を反映
NOTE: NPCの進化は描写の中で示唆される。明示的な「進化処理」は不要
```

  2R-chapter_end.md:

  - 章中にSONNETが兆候を察知

# 8. 評価システムとの接続

種族進化・ジョブ進化は評価生成と連動する。

```yaml
評価との接続:

  ジョブ進化:
    条件の一つ: 「周囲がそのキャラクターを上位ジョブとして認識し始めている」
    → 直接的評価・伝聞評価が進化条件の一部
    → 評価が高まれば進化候補が出やすくなる
  
  種族進化:
    - 進化後の種族が世界の認識に影響を与える
    - とくに人族→人外への変質は大きな評価変動を生む
    - 構造的評価（賞金首化、要注意人物化）が発生しやすい

例:
  剣士A→剣豪Aへの進化:
    → 伝聞評価「剣豪」が広まり始める
    → 構造的評価として勢力が認識する
  
  人族→ヴァンパイア化:
    → 直接的評価が激変（恐怖、忌避、新たな所属）
    → 構造的評価（特定組織からの追跡対象化）
    → 文化的評価（怪物として語られる）

IF: 種族・ジョブの進化が発生
THEN: 章末処理段階2「対外的評価の描写」で反映を検討

詳細: 2R-evaluation.md を参照
```

# 9. 違反兆候

```yaml
違反兆候:

  進化の暴走:
    - 一度の活躍だけで種族進化させる
    - 章ごとに必ず進化を発生させる
    - 条件を満たさず進化させる
    - プレイヤーの確認なしに進化させる
  
  選択権の侵害:
    - プレイヤーが拒否したのに「進化しないと先に進めない」と扱う
    - 進化を強要する形で物語を進める
    - 「進化を拒否した」ことを失点として扱う
  
  種族とジョブの混同:
    - 人族にも種族進化を適用する（変質を経ていないのに）
    - 人外にジョブシステムを適用する
    - 境界を明示せず両者を曖昧に運用する
  
  失われる範囲の隠蔽:
    - 進化後に得るものだけを提示し、失うものを明示しない
    - 進化を「全面強化」として扱う
    - ジョブ統合で失う範囲を曖昧にする
  
  種族変質の暴走:
    - 観測可能な描写なしに種族変質を実行する
    - メタ警告を出してから変質させる
    - PCの軽率な選択を結果前に修正させる
  
  ユニークスキルの濫変質:
    - 些細な変化でユニークスキルを変質させる
    - 種族・ジョブの根本的変化なしに変質させる
  
  NPC進化の機械化:
    - すべてのNPCに種族・ジョブを綿密に設定しようとする
    - NPCの進化を「処理」として明示的に通知する
    - NPCの成長を物語的に描かない
```
