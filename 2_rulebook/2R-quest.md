# クエスト体系

## このファイルの位置

クエストの分類、発生、段階、可視性、周回時の扱いを定める。シナリオ仕様書での扱い、章末処理、章プロットとの関係は別ファイルで扱う。

## 三秒で読む要約

クエストは「持続性」と「重要度」の二軸で分類される。クエストは原則としてPCの行動から発生する。クエスト化されない章の構成要素を「非クエスト要素」として独立に置く。

## 一分で読む要約

クエストはワールド／ユニーク（持続性軸）とメイン／サブ（重要度軸）で分類され、四象限を成す。これと別に、関係深化・文化接触・内省などの「非クエスト要素」が章を構成する素材として独立に存在する。クエストはキャラメイク時に確定する自動発生以外、PCの探索・会話・遭遇などの行動を契機に発生する。非クエスト要素の中からクエストが浮上することもある。段階は認識・理解・介入・決算の四段階だが、すべてを経る必要はない。ワールド・クエストは原則として周回をまたいで持続し、ユニーク・クエストは周回終了で消失する。

# 1. クエストの二軸分類

クエストは二つの独立した軸で分類される。

## 1.1 持続性軸（ワールド／ユニーク）

```yaml
ワールド・クエスト:
  定義: 世界の側に存在し、複数のシリーズ・複数の周回をまたいで持続するクエスト
  特徴:
    - 特定のPCに紐づかない
    - 世界の構造的問題
    - PCが関与しなくても世界の側で自律的に進行しうる
    - 周回が変わってもクエストは残る（状態はリセット）
  例:
    - 「主なき土地ヴェレンの主は誰か」
    - 「アルカディア王国の森林伐採事業の帰結」
    - 「魔王国は再侵略するか」
    - 「霊峰オリアスは聖イシュタル皇国に返還されるか」
    - 「ノルトハイム帝国の皇帝継承は誰の手に」

ユニーク・クエスト:
  定義: 当該PCのプレイ中のみ存在し、周回が変われば消失するクエスト
  特徴:
    - PC固有
    - PCの動機・経歴・関係性・状況に紐づく
    - PCのプレイ終了で消失（プレイヤーが望めば痕跡を残す）
  例:
    - 「弟を探す」（動機由来）
    - 「故郷の襲撃犯を追う」（経歴由来）
    - 「育てた使い魔の失踪を探す」（関係性由来）
    - 「酒場で出会った老人からの個人的な依頼」（プレイ中発生）
```

## 1.2 重要度軸（メイン／サブ）

```yaml
メインクエスト:
  定義: PCにとって重要なクエスト
  特徴:
    - PCの中軸的目標
    - 動機由来、経歴由来、状況由来のものが多い
    - 通常は1〜複数を持つ
    - シリーズ・周回を通じて遷移しうる（動機の変質、状況の変化により）
  
サブクエスト:
  定義: PCにとって些細または周辺的なクエスト
  特徴:
    - 軽い依頼、距離を置いている世界問題
    - メインクエストの周辺で発生することが多い
    - 複数が同時並行することがある
```

「章の中心」との関係：

```yaml
メイン／サブと「章の中心」は独立した軸である:
  - メインクエストが章の中心になることが多い
  - ただし、サブクエストが章の中心になることもある（日常回など）
  - 「章の中心は何か」はシナリオ仕様書の設計判断（プロット設計）
  - メイン／サブの分類は、章の中心とは独立した、PCにとっての重要度の軸

例:
  - 戦闘回・決算回・主軸進展回: メインクエストが章の中心
  - 日常回（友人の酒屋のネズミ退治で終わる章など）: サブクエストが章の中心
  - 探索回・内省回: 非クエスト要素が章の中心
```

章の中心をどう決めるかの運用詳細は `2R-scenario.md` で扱う。

## 1.3 二軸の組み合わせ（四象限）

```yaml
四象限:

  メイン × ユニーク:
    定義: PCにとって重要な個別クエスト
    頻度: 多い
    章の中心になりやすさ: 高
    例:
      - 「弟を探す」（動機由来）
      - 「脱獄する」（状況由来）
      - 「故郷の襲撃犯を追う」（経歴由来）
  
  メイン × ワールド:
    定義: PCが世界の構造的問題を主軸としている場合
    頻度: 限定的だが重要
    章の中心になりやすさ: 高
    例:
      - PCの動機が「ヴェレンを救う」
      - PCの動機が「魔王を倒す」
    注意: ワールド・クエストであっても、PCがそれをメインとする場合は
          章の中心に据えられやすい
  
  サブ × ユニーク:
    定義: PCにとって些細な個別クエスト
    頻度: 多い
    章の中心になりやすさ: 通常は低いが、日常回では中心になりうる
    例:
      - 「酒屋のネズミ退治」
      - 「街で迷子を見つけて送る」
      - 「酒場の店主の悩みを解決する」
      - 道中で受けた些細な依頼
    日常回での扱い:
      - メインクエストが章中に動かない時間帯
      - サブクエストが章の中心に据えられる
      - 例: 友人の酒屋のネズミ退治だけで終わる章は十分に正規
  
  サブ × ワールド:
    定義: PCが距離を置いている世界の問題
    頻度: 最も多い
    章の中心になりやすさ: 低（背景として並走するのが基本）
    例:
      - 道中で耳にする「ヴェレンの主は誰か」の噂
      - 自律進行する魔王国の動向
      - PCが認識しているが介入していない世界問題
    補足: 多くのワールド・クエストはこの位置にある
```

# 2. 非クエスト要素

クエストとは別に、章を構成する素材として「非クエスト要素」が存在する。これはクエスト化されない章の構成要素を指す。

```yaml
非クエスト要素:

  定義:
    章の構成要素として存在するが、クエストとして登録されないもの
  
  種類:
    関係深化:
      - NPCとの関係性の変化
      - 信頼の構築、対立の発生、和解、距離
    
    文化接触:
      - 土地・風物・祭礼への触れ合い
      - 食事、衣装、慣習、伝承への接触
    
    内省:
      - PCの自己理解の深化
      - 過去との対面、価値観の問い直し
    
    副次的調査:
      - 探索の中で得られる雑多な情報
      - クエストに直結しない知識・噂
    
    日常体験:
      - 買い物、食事、休息
      - 街の風物を味わう、何気ない会話
    
    純粋な空間:
      - 移動、休息、何も起きない時間
  
  特徴:
    - クエストの段階を持たない
    - 評価には影響しうる（とくに直接的評価・伝聞評価）
    - 章末処理での明示的な「進展」記録は不要
    - これらの中からクエストが浮上することがある（後述）
```

## 2.1 非クエスト要素とサブクエストの違い

```yaml
サブクエストとの違い:

  サブクエスト:
    - 「達成すべき目標」がある
    - 段階を持つ（認識・理解・介入・決算）
    - クエスト一覧に登録される（ユニークの場合は文脈管理）
  
  非クエスト要素:
    - 「達成すべき目標」がない
    - 段階を持たない
    - 登録されない
    - 章の中での体験として完結する

例:
  サブクエスト: 「酒場の店主の悩みを解決する」
    → 店主との会話で問題を把握 → 解決行動 → 達成
  
  非クエスト要素: 「酒場で店主と雑談する」
    → 関係深化、文化接触として記録される
    → ただし、雑談の中から「店主の悩み」が浮上した場合、
      その時点でサブクエストに転化する
```

# 3. クエストの発生

クエストの発生には三つの類型がある。

## 3.1 自動発生（限定的）

```yaml
自動発生:
  対象: キャラメイク時点で自明なもの
  発生条件: キャラメイク時の動機・経歴・状況設定
  状態: プレイ開始時から第一段階（認識）にある
  
  例:
    - 動機「弟を探す」→ 同名のユニーク・クエストが開始時に存在
    - 動機「故郷の再建」→ 同様
    - 状況「囚人としてスタート」→ ユニーク・クエスト「脱獄」が
      キャラメイク時に既に発生している
    - 動機「魔王を倒す」→ ワールド・クエストへの主軸コミット
  
  特徴:
    - キャラメイク時に確定する
    - 通常はメインクエストとなる
    - プレイ開始前から段階を持っている

IF: キャラメイクで動機・経歴・状況が確定した
THEN: 動機・経歴・状況に対応するクエストを自動発生として登録する
```

## 3.2 行動駆動発生（標準）

```yaml
行動駆動発生:
  対象: 自動発生以外のすべてのクエスト
  発生条件:
    - PCが探索した（地理的・情報的）
    - PCが会話した（NPCとの対話）
    - PCが特定の場所・物・人物と遭遇した
    - PCが特定の行動を取った
  
  特徴:
    - プレイヤーの能動性が前提
    - PCが動かなければクエストは現れない
    - これは正規の状態である（クエストが降って湧かない）

IF: PCが探索・会話・遭遇・行動を取った
THEN: その結果として手掛かりや依頼が観察可能な範囲に現れる
ELSE IF: PCが何もしなかった
THEN: クエストは現れない（正規の状態）

違反兆候:
  - PCが何もしていないのにNPCが「実は問題があって…」と話しかけてくる
  - 街に入っただけでクエスト一覧が開示される
  - PCの行動と無関係に「降って湧いた」依頼が現れる
```

## 3.3 浮上型発生

非クエスト要素の中からクエストが立ち上がる形態。最も自然な発生形態である。

```yaml
浮上型発生:
  対象: 非クエスト要素から立ち上がるクエスト
  発生条件:
    - 関係深化の途中で重要な依頼が出てきた
    - 文化接触の途中で謎が見つかった
    - 内省の中でPCの動機が変質し、新たな目標が立った
    - 日常の会話から事件の手掛かりが浮かんだ
  
  浮上の前提条件:
    - PCがその非クエスト要素に十分関与した
    - PCがそれに対して関心を示した（プレイヤーの能動的選択）
    - その要素がクエストとして展開する素地を持っている

浮上の典型例:

  関係深化 → ユニーク・クエスト:
    例: 老ドルイドとの会話を重ねるうち、彼の失踪した弟子の話が出る
        → 「老ドルイドの弟子を探す」ユニーク・クエストが浮上
  
  文化接触 → ワールド・クエスト:
    例: 街の祭礼に参加するうち、祭礼の起源が滅亡戦争と関わると知る
        → 「滅亡戦争の真実を追う」ワールド・クエストが浮上
  
  内省 → ユニーク・クエスト:
    例: PCが自分の出自と向き合う中で、自分が魔族の混血である可能性に気づく
        → 「自分の血の正体を確かめる」ユニーク・クエストが浮上
  
  日常体験 → ユニーク・クエスト:
    例: 何気なく寄った酒場で、馴染みの店主が悩んでいる
        → 「店主の悩みを解決する」ユニーク・クエストが浮上

IF: PCが非クエスト要素に関与し、その要素にクエスト浮上の素地がある
THEN: 浮上の機会を提示する（手掛かり、暗示、暗黙の依頼）
IF AND: PCが関心を示した
THEN: クエストとして発生する
ELSE: 浮上せず、非クエスト要素として完結する
```

# 4. クエストの段階

クエストは以下の四段階を持つ。

```yaml
標準的な段階:
  
  第一段階：認識
    定義: PCがクエストの存在を知る段階
    特徴: 手掛かりを通じて世界に未解決問題があると認識する
  
  第二段階：理解
    定義: クエストの構造を把握する段階
    特徴: 関与する勢力、利害、選択肢が見えてくる
  
  第三段階：介入
    定義: PCが具体的な行動を取る段階
    特徴: 特定の勢力に味方する、自ら独自の解を作る、観察に徹するなど
  
  第四段階：決算
    定義: クエストの状態が確定する段階
    特徴: 「決まらない」「持続する」も正規の決算結果
```

## 4.1 段階の柔軟性

```yaml
段階を経る柔軟性:

  全段階を経る必要はない:
    - 認識のみで終わる: PCが手掛かりに触れただけで、関与しない
    - 理解までで終わる: 状況を把握したが、介入しない
    - 介入したが決算に至らない: 途中で離脱する、状況が変化する、別のクエストに移る

  すべて正規の進行である:
    - クエストを途中で放棄しても「失敗」ではない
    - 段階の停滞は正規の状態
```

## 4.2 周回での段階の引き継ぎ

```yaml
ワールド・クエストの場合:
  - 周回ごとに状態がリセットされる
  - 前周回で第三段階まで進めても、次の周回では第一段階から
  - ただし「世界の解像度の進化」として、OPUSは前周回の進展を踏まえて
    新周回のシナリオ仕様書を書ける

ユニーク・クエストの場合:
  - 周回終了で消失するため、引き継がれない
  - プレイヤーが明示的に望めば、痕跡（NPC、場所、影響）として残せる
```

# 5. クエストの可視性

クエストはプレイヤーに自動開示されない。世界内の手掛かりを通じて発見される。

## 5.1 自動開示しない原則

```yaml
自動開示しない原則:
  - ルールブックは「クエスト一覧」のような形でPCにクエストを提示しない
  - 中核テーマ「観測のアルカディア」と整合する
  - 世界を観測する個が、観測を通じてクエストの存在を発見する

例外:
  - 自動発生のクエスト（キャラメイク時に確定）はPCの認識として存在する
```

## 5.2 手掛かりの種類

```yaml
ワールド・クエストの手掛かり:
  - NPCの対話（噂、嘆き、依頼）
  - 街道や酒場で耳にする情報
  - 訪れた地域で観察される異常
  - 新聞・告示・布告
  - 過去の出来事の痕跡（廃墟、戦場跡、墓所）
  - PC不在時間の描写（プレイヤーには見えるが、PCには見えない状況）

ユニーク・クエストの手掛かり:
  - 特定NPCとの個別接触
  - PCの経歴に関わる場所や人物との遭遇
  - PCの動機が刺激される状況
  - 関係性のあるNPCの変化
```

## 5.3 段階的な開示

```yaml
段階的開示:
  - 最初は「何かがある」という感触のみ
  - 次に「具体的な対象」が見える
  - さらに進むと「複数の関係者と立場」が見える
  - 深まるごとに解像度が上がる

これはクエストの段階（認識→理解→介入）に対応する。
```

# 6. 受諾の自由

クエストはPCに提示されても、受諾するかどうかはプレイヤーの自由である。

## 6.1 拒否も成立する

```yaml
拒否の自由:
  
  ワールド・クエスト:
    - 認識しても、関わらない選択ができる
    - 「ヴェレンの主は誰か」を知っていても、PCがヴェレンに行かなければ
      関与は発生しない
  
  ユニーク・クエスト:
    - 動機由来のクエストでさえ、プレイヤーが「今は追わない」と選べる
    - 動機は判定の文脈ボーナスに影響するが、絶対的拘束ではない

IF: PCがクエストを拒否した
THEN: 拒否を尊重する。別ルートでの誘導はしない
```

## 6.2 部分的受諾

```yaml
部分的受諾:
  - クエストを完全に受諾も拒否もせず、部分的に関わる選択
  - 特定の勢力には味方せず観察に徹する
  - 特定の局面でのみ介入する
  - これも正規の選択肢
```

# 7. 周回時の扱いと自律的進行

## 7.1 ワールド・クエストの原則：完全リセット

```yaml
原則:
  ワールド・クエストの状態は、周回ごとに完全にリセットされる。
  ある周回でPCがヴェレンの主となっても、次の周回では
  「ヴェレンの主は誰か」が再び未解決の状態で存在する。

理由:
  - 中核テーマの「世界は固定的にそこに在り、プレイヤーは不確定な因子」と整合
  - 各周回はパラレルワールドとして扱われる
```

## 7.2 例外：プレイヤーからの反映提案

```yaml
反映の例外:
  発生条件: プレイヤーが特定の周回の結末を「世界に反映させたい」と提案
  処理:
    - アーカイバOPUS（周回末ロール）が更新パッチを生成
    - プレイヤーが手動で適用
  
  詳細: 2O-opus_archivist.md および 2F-cycle_patch.md を参照
```

## 7.3 ユニーク・クエストの扱い

```yaml
ユニーク・クエストの周回末:
  - 周回終了で消失するのが原則
  - 痕跡（重要なNPCの運命、特定の場所への影響）はプレイヤーの提案ベースで残せる
```

## 7.4 自律的進行

```yaml
PC不在下のワールド・クエスト:
  - PCが関与を拒否した、または認識すらしていないワールド・クエストは
    世界の側で自律的に進行する
  - 勢力間の力関係や時間経過により状況が変化する
  - 中核テーマの「世界はプレイヤーが行動しなくても動いている」と整合

具体的運用:
  - OPUSはシナリオ仕様書作成時に、PCが関わっていないワールド・クエストの
    状態を更新する
  - PCがある期間を別の場所で過ごしている間に、別地域で進行が起きうる
  - PCが何年か後にその地を訪れた時、状況が様変わりしている可能性

詳細: 2R-npc_autonomy.md を参照
```

# 8. 評価との連動

クエストへの関与は評価システム（`2R-evaluation.md`）と連動する。

## 8.1 メイン／サブによる影響の違い

```yaml
メインクエストへの関与:
  - 評価四層すべてに強く影響する
  - 構造的評価・文化的評価を発生させる主要装置
  - 異名の発生源になりやすい

サブクエストへの関与:
  - 主に直接的評価と伝聞評価に影響する
  - 構造的・文化的評価まで広がる場合もあるが、規模は小さい
```

## 8.2 ワールド／ユニークによる影響の違い

```yaml
ワールド・クエストへの関与:
  - 構造的評価（勢力の分類）を発生させやすい
  - 文化的評価（伝承・歌）に発展する可能性
  - 周回をまたぐ評価記録の対象になりうる

ユニーク・クエストへの関与:
  - 主に直接的評価と伝聞評価に影響
  - 関与したNPCの認識を変える
  - 周回をまたいで残らない（プレイヤーが望めば残せる）
```

## 8.3 四象限別の評価への影響

```yaml
四象限と評価:

  メイン × ワールド:
    - 評価への影響: 最大
    - 異名発生: 高確率
    - 例: 「ヴェレンの解放者」「主なき土地の王」
  
  メイン × ユニーク:
    - 評価への影響: 大
    - 異名発生: 関与した範囲で
    - 例: 「弟を探す者」（限定的だが特定地域では強い認識）
  
  サブ × ワールド:
    - 評価への影響: 中
    - 直接関与したNPCの認識を変える程度
  
  サブ × ユニーク:
    - 評価への影響: 小
    - 個別NPCの直接的評価のみ
```

# 9. 違反兆候

クエスト体系の運用において、LLMがやりがちな違反パターン。

```yaml
違反兆候:

  自動開示の暴走:
    - PCが街に入っただけでクエスト一覧を提示した
    - 「現在の依頼」をシステムメッセージとして開示した
    - NPCが頼まれてもいないのに依頼を切り出してきた
  
  自動発生の暴走:
    - PCが何もしていないのに新しいクエストが降ってきた
    - 章ごとに「今回のクエスト」を強制的に発生させている
    - PCの行動と無関係にクエストが発生する
  
  メイン／サブの混同:
    - サブクエストを章の中心に据えた
    - メインクエストを背景に追いやり、サブクエストの解決に章を費やした
    - 重要度の判定をせず、最初に出てきたクエストを章の中心に据えた
  
  拒否の不尊重:
    - PCが拒否したクエストを別ルートで再提示した
    - 「やっぱりお願いします」と同じNPCが何度も依頼してきた
    - 拒否したクエストを別NPC経由で再度提示した
  
  浮上の濫用:
    - 非クエスト要素を毎回クエスト化してしまう
    - 雑談から必ず依頼が浮上する設計になっている
    - 関係深化が常にクエストの素地として扱われている
  
  非クエスト要素の軽視:
    - 関係深化・文化接触・内省を「クエストになる前段」としか扱わない
    - これらを単独で章の主体とすることを許さない
    - 章末処理で非クエスト要素の蓄積を記録しない
  
  自律進行の忘却:
    - PCが関与しないワールド・クエストの状態が章をまたいでも変わらない
    - 周回をまたいでもワールド・クエストが固定されている
    - 自律進行の結果がシナリオ仕様書に反映されていない
```
