# 装備・所持品・魔具

## このファイルの位置

携行品の管理、消耗品、通貨と経済、マジックアイテム、無形の所持物を定める。装備の判定への影響、評判・異名の運用は別ファイルで扱う。

## 三秒で読む要約

装備・所持品は社会的摩擦の表現。物理的な品だけでなく、情報・関係性・拠点・評判も「所持物」として扱う。

## 一分で読む要約

PCの装備・所持品は章をまたいで継続する。厳密な重量管理は行わないが、無理な携行は判定に影響する。消耗品は長期の旅や危険な環境で意味を持つ。通貨は社会的摩擦の表現として機能する。マジックアイテムは効果・発動条件・制約を明記する。無形の所持物（情報・関係性・拠点・評判）も継続管理の対象。装備・所持品の変化は章末処理段階3で扱われる。

# 1. 携行品の意識

## 1.1 携行品の把握

```yaml
携行品の意識:
  
  原則:
    PCが今何を身につけ、何を背負っているかを把握する。
  
  厳密な重量管理:
    - 行わない
    - ただし無理な携行は判定に影響する

無理な携行の例:
  - 重装備で水中活動
  - 両手を塞ぐ品で隠密行動
  - 大荷物で素早い移動
  - 目立つ装備で潜入

IF: PCが無理な携行で判定する
THEN: 判定にペナルティを与える
NOTE: 機械的な数値ではなく、状況裁定として
```

## 1.2 装備の判定への影響

装備が判定に影響する場合、`2R-judgment.md` 第5節「文脈ボーナス・ペナルティ」と連動する。

```yaml
装備の判定への影響:
  
  有利な装備:
    - 該当判定で文脈ボーナス
    - 例: ミスリル剣を持つ判定での切断
    - 例: ドワーフ製武具での戦闘
  
  不利な装備:
    - 該当判定で文脈ペナルティ
    - 例: 両手を塞ぐ品を持つ判定での隠密
    - 例: 重装備での水中活動
  
  状況の影響:
    - 装備の特性に応じて自動成功・自動失敗の裁定もありうる
    - 例: 「魔法を吸収する盾」を持つPCが魔法攻撃を受けた

IF: 装備が判定の文脈に関わる
THEN: 判断の透明化（2R-judgment.md 1.2節）で影響を明示
```

# 2. 消耗品

```yaml
消耗品:
  
  種類:
    - 食料
    - 水
    - 油（ランプ、火種）
    - 薬
    - 矢
    - 触媒（魔法用）
  
  管理の場面:
    意味を持つ場面:
      - 長期の旅
      - 辺境への遠征
      - ダンジョン探索
      - 戦時下
      - 飢饉や物資不足の状況
    
    管理を省略する場面:
      - 日常的な街中
      - 補給が容易な状況

IF: 物資補給が困難な状況
THEN: 消耗を意識する

IF: 日常的な街中
THEN: 消耗管理を省略してよい
```

# 3. 通貨と経済

## 3.1 通貨の機能

```yaml
通貨の機能:
  
  社会的摩擦の表現:
    通貨は社会的摩擦の表現として機能する。
  
  典型的な用途:
    - 宿代
    - 食事代
    - 装備修理費
    - 情報料
    - 賄賂
    - 通行料
    - 治療費

PCの経済状況:
  - 章をまたいで継続する
  - キャラクターシートに記録
```

## 3.2 通貨単位と物価

```yaml
通貨単位:
  
  詳細は世界設定ファイル（1_setting/）を参照
  
  基本構造:
    - 銅貨、銀貨、金貨、白金貨
    - 地域によって流通する通貨が異なる
    - 一部地域では物々交換が主流

物価の感覚:
  - 詳細は 1W-economy.md（仮）または世界設定を参照
  - シナリオ運用時はGMの裁量で物価を設定

IF: 物価が論点になる場面
THEN: 世界設定を参照し、地域・時代に応じて裁定
ELSE: 厳密化しない
```

# 4. マジックアイテム

## 4.1 取得時の明記事項

```yaml
マジックアイテムの取得:
  
  取得した際は以下を明記する:
    - 効果の概要
    - 発動条件（常時／任意／条件付き）
    - 制約やコスト
    - 入手経緯・由来（後の物語に響く可能性）

例:
  「炎の指輪」
    効果: 炎魔法の威力を増幅し、低位の炎魔法を任意発動できる
    発動条件: 装着時に任意発動
    制約: 一日に三回まで、装着時に手の温度が上がる
    由来: メジプト砂炎国の砂漠商人から購入

IF: マジックアイテムを獲得した
THEN: 上記項目を明記して記録する
```

## 4.2 強力な魔具の裁定

```yaml
強力な魔具の裁定:
  
  原則:
    強力な魔具にはパッシブスキルと同様の裁定を適用し、相応の制約を設ける。
  
  装備効果の判定への影響:
    - GMが状況に応じて修正値の付与や自動成功などを裁定
    - 機械的な計算は避ける

強力な魔具の例:
  「真理の眼鏡」
    効果: 嘘を見抜く
    制約: 一日に三度まで、使用後は一時的に視力低下
  
  「不死者の腕輪」
    効果: 致命傷を一度だけ無効化
    制約: 使用後は腕輪が砕け、再使用不可

IF: 強力な魔具が判定に関わる
THEN: パッシブスキル同様の裁定で扱う
AND: 制約・コストも反映する

違反兆候:
  - 強力な魔具に制約を設けない
  - 「魔具を持っているから自動成功」と短絡する
  - マジックアイテムの濫発
```

# 5. 無形の所持物

物理的な所持品とは別に、無形のものも「所持物」として扱う。これらは中核テーマ「観測のアルカディア」と密接に連動する。

## 5.1 無形の所持物の種類

```yaml
無形の所持物:

  情報・噂・地図:
    - 入手した情報、聞いた噂、入手した地図
    - 後の章で参照可能
    - 記録の対象
  
  NPCからの貸し借り:
    - 後から回収できる関係性の貯金
    - 「あの時の借りを返す」「貸しを作った」
    - 評価四層の直接的評価と連動
  
  場所への所有権・拠点:
    - 拠点、隠れ家、所有物件
    - 章をまたいで継続
    - 自律進行の影響を受ける可能性
  
  認識の対象としての評判:
    - 異名、評判、名声、悪名
    - 評価四層と直接連動
    - 詳細は 2R-evaluation.md を参照
```

## 5.2 無形所持物の継続管理

```yaml
継続管理:
  
  原則:
    無形の所持物はシナリオ仕様書の「所持物」欄に記録され、章をまたいで継続する。
  
  記録の手順:
    - 章中に新規発生したものは章末処理段階3で記録
    - 章をまたぐ参照はシナリオ仕様書のPCの現状に含める

詳細: 2R-chapter_end.md 3.4節「段階3：システム処理」を参照

IF: 章中に無形所持物の変化があった
THEN: 章末処理段階3で記録する

含む例:
  - 情報の獲得
  - 関係性の貯金の発生・回収
  - 拠点の獲得・喪失
  - 評判の変化（段階2と連動）
```

## 5.3 評判との連動

```yaml
評判と無形所持物の連動:
  
  原則:
    評判（異名、名声、悪名）は無形所持物の一種だが、
    詳細な運用は評価システムで扱う。
  
  本ファイルの担当範囲:
    - 「所持物」として継続管理対象であること
    - キャラクターシートへの記載方法
  
  2R-evaluation.md の担当範囲:
    - 評判の発生メカニクス
    - 異名の運用原則
    - 評価四層との関係
    - 描写の手法

参照: 2R-evaluation.md 第5節「異名・評判」
```

装備・所持品の変化は章末処理段階3で扱われる。

```yaml

  2R-chapter_end.md:
    - 章末処理段階3での装備変化の処理手順
    - 確認すべき情報

```

# 7. 評価システムとの接続

特殊な装備の所持は、評価生成と連動することがある。

```yaml
評価との接続:

  特徴的な装備:
    - 異名の発生源になりうる
    - 例: 「銀剣の」「炎の指輪の使い手」
  
  伝説的な品:
    - 構造的評価を生むことがある
    - 例: 国宝級の品の所持により、勢力からの監視対象になる
  
  魔具の派手な使用:
    - 文化的評価の素材になる
    - 例: 「不死者の腕輪を持つ者」として伝承化

IF: 特徴的な装備が章中に発動した
THEN: 章末処理段階2「対外的評価の描写」で反映を検討

詳細: 2R-evaluation.md を参照
```

# 8. 違反兆候

```yaml
違反兆候:

  携行の不自然な無視:
    - 重装備で水中を泳がせる
    - 両手を塞ぐ品で隠密行動を成功させる
    - 大荷物で素早い移動を許す

  消耗品の機械的管理:
    - 日常的な街中で食料・水を細かく管理する
    - すべての場面で消耗品の残量を確認する

  通貨の社会的摩擦の消失:
    - 強いPCには宿代・食事代が無料になる
    - 賄賂・情報料が不要になる
    - 経済状況が章をまたいで継続しない

  マジックアイテムの裁定甘さ:
    - 強力な魔具に制約を設けない
    - 「魔具を持っているから自動成功」と短絡する
    - 制約を設けても運用時に守らない
    - マジックアイテムを濫発する

  無形所持物の忘却:
    - 情報・噂・地図を記録しない
    - 関係性の貯金を反映しない
    - 章をまたいで継続させない
    - 拠点の存在を忘れる

  評判の機械化:
    - 異名を数値スコアで管理する
    - 評判を「上げるべきパラメータ」として扱う
    - 評価システムとの連動を無視する
```
