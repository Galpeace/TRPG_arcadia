# スキルシステム

## このファイルの位置

スキルの分類、ランク、Sランクの特殊効果、不利な性質型スキル、成長（強化・変質・統合・新規獲得）、体験起因スキル、発動タイミングと提案権、動機・禁忌の文脈尊重を定める。種族進化・ジョブ進化は別ファイル、判定処理の方法も別ファイルで扱う。

## 三秒で読む要約

スキルは三分類（アクティブ・パッシブ・ユニーク）、五ランク（S・A・B・C・D）。成長は四形態（強化・変質・統合・新規獲得）。Sランクは質的変容。

## 一分で読む要約

スキルは三分類（アクティブ・パッシブ・ユニーク）と五ランク（S・A・B・C・D）で構成される。ランクはGMの判定の参照軸であり、機械的な数値計算には使わない。Sランクは数値的優位だけでなく質的変容を持ち、到達には章を超えた中心的役割と転機の体験が必要。不利な性質型スキルはランクを持たず、取得時にBランクスキルの追加枠を得る。成長は四形態（強化・変質・統合・新規獲得）。体験起因スキルはセッション中の具体的体験を直接の契機として発生する。動機・禁忌は判定の文脈として尊重される。進化・統合は章末処理で提示・判定される。

# 1. スキルの分類

## 1.1 アクティブスキル

```yaml
アクティブスキル:
  定義: 意図的に発動する能力
  含む: 戦闘行動、魔法、技術など
  ランク: 原則として持つ

例:
  - 剣術
  - 火魔法
  - 鍵開け
  - 交渉
  - 軽業
```

## 1.2 パッシブスキル

```yaml
パッシブスキル:
  定義: 常時発動するスキル
  含む: 種族特性、体質、耐性など
  
  二種類のサブ分類:
    
    有利な効果を持つもの:
      ランク: 持つ
      例: 夜目、毒耐性、魔力感知、頑健、暗視
    
    不利な性質を表すもの:
      ランク: 持たない
      発動: 該当状況で自動発動
      例: 炎弱点、水中行動不可、日光弱点、寒さに弱い
```

## 1.3 ユニークスキル

```yaml
ユニークスキル:
  定義: キャラクター固有の世界に一つ〜少数のスキル
  由来: 経歴・種族・獲得経緯に基づく
  
  特徴:
    - 原則としてランクを持たない
    - 強化・進化もしない
    - アクティブにもパッシブにもなりえる
  
例:
  - 「真実の眼」（嘘を見抜く生まれつきの能力）
  - 「過去視」（触れた物の過去を見る希少能力）
  - 「神の祝福」（特定の神から授かった保護）
  - 「禁忌の血」（魔王の血筋による特殊属性）
```

# 2. ランクと修正値の参照

## 2.1 ランクの種類

```yaml
ランク:
  S（極意）:
    意味: 到達者の領域。常人を超えた事象を成し得る
    初期取得: 不可
    特徴: 質的変容を伴う
  
  A（得意）:
    意味: 核となる能力。困難な状況でも安定して成果を出せる
    初期取得: 可（2〜3個程度が目安）
  
  B（習熟）:
    意味: 補助的に扱える能力。日常的な状況なら問題なく行える
    初期取得: 可（3〜4個程度が目安）
  
  C（一般）:
    意味: 誰でも試みられる行動
    記載: 不要
    特記: 「C相当の能力」として裏で参照される
  
  D（苦手）:
    意味: 明確な不得手。日常的な状況でも失敗しやすい
    初期取得: 任意で1〜2個設定可
    扱い: 該当判定でペナルティ
```

## 2.2 ランクの位置付け

```yaml
ランクの位置付け:
  - GMが結果を判断する際の参照軸として機能する
  - 機械的な数値計算には用いない
  - 詳細な運用は 2R-judgment.md「判断の透明化」を参照

IF: 判定でスキルが関与する
THEN: ランクを明示してから結果を判断する
NOTE: 「Sだから自動成功」「Dだから自動失敗」のような短絡をしない
```

## 2.3 初期取得の目安

```yaml
初期取得の目安:
  - Aランク: 2〜3個
  - Bランク: 3〜4個
  - Dランク: 任意で1〜2個
  - Sランク: 取得不可（成長で到達）
  
  歴戦の達人などの設定がある場合は調整する。
  キャラメイクの詳細は 2R-character.md を参照。
```

# 3. Sランクの特殊効果

## 3.1 質的変容

```yaml
Sランクの質的変容:
  Sランクは数値的な優位だけでなく、質的な特殊効果を獲得する。
  これは「常人を超えた領域」というロールプレイ的価値を実装に反映させるため。

特殊効果の類型（Sランク到達時にプレイヤーとGMで具体化）:

  自動成功条件:
    定義: 特定の状況下では判定不要で成功する
    例: 「剣術S」は通常の打ち合いで判定不要
  
  対抗の優位:
    定義: 同等以下のランクの相手との対抗判定で優位
    例: 「気配察知S」はB以下の隠密を一方的に見破る
  
  派生応用:
    定義: そのスキルを基にした周辺領域の行動でも、ランクの恩恵を受ける
    例: 「火魔法S」は熱を扱うあらゆる現象に応用できる
  
  質的変容:
    定義: 常人にはできない規模・精度・速度の事象を成し得る
    例: 「弓術S」は通常の射程の数倍の距離を狙える
  
  他者への影響:
    定義: そのスキルが他者の認識や行動に影響する
    例: 「演説S」は群衆の心理を実際に動かす

IF: Sランクに到達した
THEN: 上記類型から特殊効果を一つ以上具体化する
NOTE: 複数を組み合わせてもよい
```

## 3.2 Sランク到達の認定

```yaml
Sランク到達の認定:
  
  認定条件（複数満たす場合に検討）:
    - 当該スキルが章を超えて中心的役割を果たした
    - スキルの本質に関わる転機の体験があった
    - 既存のAランク運用では説明できない事象を成し遂げた
  
  判定の場:
    - 原則として章末処理の段階3「システム処理」で判定
    - 章中にSONNETが兆候を察知したらOPUSの章末処理で扱う
    - 詳細は 2R-chapter_end.md 2.1節「確認すべき情報」参照

IF: A→Sの昇格を検討する
THEN: 認定条件を複数満たすかを確認
ELSE: A→Sは慎重に判断する（昇格は希少）

違反兆候:
  - 一度の活躍だけでSランクに到達させる
  - 認定条件を満たさずSランクに上げる
  - Sランクへの昇格を頻繁に発生させる
```

# 4. 不利な性質型スキルの取得とボーナス

## 4.1 取得時のボーナス

```yaml
不利な性質型スキルのボーナス:
  
  原則:
    キャラメイク時に不利な性質型スキル（ランクのないパッシブスキル）を取得すると、
    初期スキル枠が拡張される。
  
  ボーナス内容:
    - 不利な性質型スキル1つにつき、Bランクスキルを1つ追加可能
  
  例:
    - 「日光弱点」を取得 → Bランクスキル1個追加
    - 「水中行動不可」を取得 → Bランクスキル1個追加
    - 両方取得 → Bランクスキル2個追加

ボーナスの非強制:
  - このボーナスはマストではない
  - ロールプレイ上の動機で性質型スキルを取得する場合、
    ボーナスを受け取らない選択も可能

IF: キャラメイクで不利な性質型スキルを取得した
THEN: Bランクスキル追加枠の希望を確認

IF: プレイヤーがロールプレイ上の動機のみで取得する
THEN: ボーナスを受け取らない選択を尊重
```

# 5. スキルの成長

スキルの成長形態は以下の四つに整理される。

## 5.1 強化

```yaml
強化:
  定義: ランクが1段階上昇する
  
  発動条件:
    - セッション中の活躍
    - 訓練
    - 経験の蓄積
  
  特殊条件:
    A→Sの昇格: 「Sランク到達の認定」条件を満たす（3.2節）
  
  判定タイミング:
    - 章末処理の段階3「システム処理」
    - 章中にSONNETが兆候を察知 → OPUSの章末処理で判定

IF: 章中にスキルの活躍があった
THEN: 章末処理で強化候補として検討
```

## 5.2 変質

```yaml
変質:
  定義: ランクを維持したまま、スキルの性質・解釈・効果が変化する
  
  発生のきっかけ:
    - ロールプレイ上の体験
    - Sランク到達
    - プレイヤー・GMの合意

呼称:
  明らかな強化を伴う変質は「進化」と呼んでよい。
  これは呼称の選択肢であり、システム上は変質に統合される。

例:
  - 「死霊術」→「蘇生魔法」（性質の変化）
  - 「火魔法」→「蒼炎魔法」（進化）
  - 「闇耐性S」→「影纏いS」（Sランク到達による再定義）

IF: スキルの性質を変えるロールプレイ上の体験があった
THEN: 章末処理で変質候補として検討
```

## 5.3 統合

```yaml
統合:
  定義: 複数のスキルが合わさり、新たなスキルになる
  
  処理:
    - 統合前のスキルは失われる
    - 新スキルのランクは素材スキルの最高ランクを基準にGMが決定
  
  目的:
    強化のためだけでなく、スキル整理のためにも行ってよい。
    スキルが増えすぎて煩雑になった場合、関連するスキルを上位概念にまとめる。

例:
  強化的統合:
    - 「剣術」＋「身体強化」→「強斬り」
  
  独自の相乗的統合:
    - 「ダンス」＋「剣術」→「剣の舞」
  
  整理的統合:
    - 「弓術」＋「投擲」＋「狙撃」→「遠距離戦闘」

統合の対象外:
  - ユニークスキルとランクのない性質型スキルは原則として統合の対象にならない

IF: 関連スキルが複数蓄積し、統合の素地がある
THEN: 章末処理で統合候補として検討
```

## 5.4 新規獲得

```yaml
新規獲得:
  二種類:
    
    通常獲得:
      定義: 訓練、教授、研究などにより新たなスキルを得る
      発動: 章末処理時または明示的な訓練の場面後
    
    体験起因スキル:
      定義: セッション中の具体的な体験・行動・発見を直接の契機としてスキルが獲得される
      特徴: 訓練や蓄積ではなく、一つの体験が核になる

体験起因スキルの発生条件:
  以下のいずれかを満たす場合に発生する。
    - キャラクターが初めて行った行動であり、その行動がキャラクターの定義や方向性に関わる
    - 行動の結果として、キャラクターが新たな知識・技術・認識を得た
    - 行動がキャラクターのアイデンティティや価値観と強く結びついている

体験起因スキルの初期ランク:
  - 原則: Bランクで獲得
  - 体験の核心性が高い場合: Aランクで獲得（裁量）
  - 体験が浅い場合: Dランクで獲得（裁量）

体験起因スキルの命名:
  - ロールプレイ上の意味を反映してよい
  - 機能的説明より体験・価値観を反映した名称を優先

例:
  体験: 自分と同種の存在を複数体処理し、その構造を観察・理解した
  スキル: 「ジェノサイド B」
  効果: 同種の構造・弱点・行動原理を熟知している

IF: 上記の条件を満たす体験が章中にあった
THEN: 体験起因スキルの獲得候補として、章末処理で提案

タイミングの注意:
  - 体験起因スキルはセッション中に即座に発生してよい
  - ただし、判定の最中に突然発動させるのは避ける
  - 判定結果が確定した後、または場面の区切りで提案する
```

# 6. 成長・進化・統合の発動タイミングと提案権

## 6.1 タイミング

```yaml
発動タイミング:
  
  原則:
    進化と統合は、章末処理を待たずにセッション中に発動条件が満たされた時点で
    GMが候補として提示してよく、プレイヤーが採否を決める。
  
  避けるべき場面:
    判定の最中に突然新たな能力として発動させる
    → 必ず一度プレイヤーの確認を取る

IF: 章中に進化・統合の条件が満たされた
THEN: 判定の区切りでプレイヤーに候補として提示する
AND: プレイヤーの採否を待ってから適用する

判定中の発動禁止:
  IF: 判定処理の途中
  THEN: 新スキルを発動させない
  WAIT: 判定結果確定または場面の区切りまで
```

## 6.2 提案権

```yaml
提案権:
  
  GM側からの提案:
    - 条件が満たされた時点で候補を提示
    - プレイヤーが採否を決める
  
  プレイヤー側からの提案:
    - プレイヤーが「○○というスキルを得たい」と提案できる
    - GMは条件（累積実績、きっかけ、代償）を提示する
    - 条件が満たされていれば許可する
    - 条件が満たされていなければ、何が足りないかを伝える

IF: プレイヤーから進化・統合の提案があった
THEN: 条件を提示して合意形成する

IF: 条件が満たされている
THEN: 許可する

IF: 条件が満たされていない
THEN: 何が足りないかを伝える
```

## 6.3 統合・進化時の失われる範囲

```yaml
失われる範囲の明示:
  
  原則:
    統合や進化によって得られる新スキルには、失われる運用範囲が必ず明記される。
    GMは提示時に、得られるものと失われるものを明示し、プレイヤーに確認を取る。

例:
  「剣術」＋「身体強化」→「強斬り」
    得るもの: 一撃の威力増大、装甲貫通
    失うもの: 細かい受け流しの柔軟性、長期戦の持久力

IF: 統合・進化を実行する
THEN: 得るものと失うものを明示

違反兆候:
  - 得るものだけを提示し、失うものを明示しない
  - 進化を「全面強化」として扱う
  - 失う範囲を曖昧にする
```

# 7. スキル枠とスキルの喪失

```yaml
スキル枠:
  原則: 上限はない
  注意: かさばらないように、統合などの選択は緩く行う

スキルの喪失:
  原則: 原則として喪失しない
  整理: 整理・削除を意図する場合は、「統合」の枠組みで処理する

例外的な喪失:
  - 物語的に重要な剥奪（呪い、神罰、契約違反など）
  - PCの大きな変化に伴う必然的喪失
  これらは不可逆変化の一種として扱う（2R-core_principles.md 3.3節参照）

IF: スキルを失わせる必要がある状況
THEN: 観測可能な描写を事前に提示（メタ警告はしない）
NOTE: 2R-core_principles.md 3.3節「予告なしの不可逆変化」と整合
```

# 8. 動機・禁忌の文脈尊重

`2R-judgment.md` 第5節「文脈ボーナス・ペナルティ」と連携する詳細を扱う。

## 8.1 文脈の影響

```yaml
動機・禁忌の文脈尊重:
  
  PCの動機・禁忌に関わる判定では、GMはそれを文脈として意識する。
  
  動機への影響:
    動機に直接沿った行動の判定:
      - 状況に応じて+1相当の文脈ボーナスを与えてよい
      - 例: 動機「弟を探す」のPCが弟の手掛かりを追う判定
  
  禁忌への影響:
    禁忌に反する状況での判定:
      - 状況に応じてペナルティを与えてよい
      - 例: 禁忌「殺さない」を持つPCが殺害判定をする
    
    禁忌を貫く選択をした場合:
      - 別の判定で文脈ボーナスを与えてよい
      - 例: 殺さずに無力化する判定が容易になる
  
  運用原則:
    機械的な発動ではなく、シミュレーションの文脈で自然に効く形にとどめる。
```

## 8.2 動機・禁忌と体験起因スキル

```yaml
動機・禁忌と体験起因スキル:
  
  動機の達成・禁忌の貫徹は、章末処理時に体験起因スキルの契機として
  優先的に検討される。

例:
  動機「故郷の再建」を貫いた章末:
    → 体験起因スキル「故郷の意志 B」など
  
  禁忌「殺さない」を貫いた章末:
    → 体験起因スキル「無傷の制圧 B」など

IF: 章中に動機の達成または禁忌の貫徹があった
THEN: 章末処理で体験起因スキルの候補として優先検討
```

# 9. 章末処理との接続

スキルの成長は章末処理段階3で扱われる。本ファイルと `2R-chapter_end.md` の責任分担を明示する。

```yaml
責任分担:

  本ファイル（2R-skill.md）:
    - スキルの分類、ランク、Sランク特殊効果
    - 成長の四形態の定義
    - 体験起因スキルの発生条件
    - 動機・禁忌の文脈尊重

  2R-chapter_end.md:
    - 章末処理段階3でのシステム処理の手順
    - 確認すべき情報（進化候補の兆候など）

  接続:
    - 章中にSONNETが兆候を察知
      → 章末処理時にOPUSが本ファイルを参照して判定
      → 結果を段階3で実行

IF: 章末処理段階3でスキルの成長を扱う
THEN: 本ファイルの該当節を参照する
```

# 10. 評価システムとの接続

スキルの活躍は評価の生成と連動する。

```yaml
評価との接続:

  Sランクスキルの発動:
    - 直接的評価を強く動かす
    - 伝聞評価・文化的評価の源泉になりやすい
    - 異名の生成源
  
  ユニークスキルの発動:
    - 構造的評価を生むことがある
    - 「あの能力を持つ者」として勢力に分類される
  
  体験起因スキルの命名:
    - 異名として外部に現れることがある
    - 「ジェノサイド B」を持つPCが「殲滅者」として知られる、など

詳細: 2R-evaluation.md を参照

IF: Sランクや特殊なスキルが章中に発動した
THEN: 章末処理段階2「対外的評価の描写」で反映を検討
```

# 11. 違反兆候

```yaml
違反兆候:

  ランクの誤運用:
    - 「Sだから自動成功」のような短絡判定
    - 「Dだから自動失敗」のような短絡判定
    - ランクを明示せず判定する
    - 数値計算にランクを使う
  
  Sランク認定の濫用:
    - 一度の活躍だけでSランクに到達させる
    - 認定条件を満たさずSランクに上げる
    - Sランクへの昇格を頻繁に発生させる
    - Sランクの特殊効果を具体化しない
  
  不利な性質型スキルの軽視:
    - 該当状況で自動発動させない
    - ボーナス枠を機械的に強制する（プレイヤーの選択を奪う）
    - 性質型スキルを「ランクで弱体化」させる
  
  成長の暴走:
    - 章ごとに必ず成長を発生させる
    - 「活躍がない」のに進化させる
    - プレイヤーの確認なしに新スキルを発動させる
    - 判定の最中に新スキルを発動させる
  
  体験起因スキルの濫発:
    - 些細な体験から体験起因スキルを生成する
    - 章ごとに必ず体験起因スキルを発生させる
    - 機能的な説明のみで命名する（ロールプレイ要素を欠く）
  
  進化・統合の不透明:
    - 得るものだけ提示し、失うものを明示しない
    - 進化を「全面強化」として扱う
    - 失う範囲を曖昧にする
  
  動機・禁忌の機械化:
    - 動機への+1ボーナスを毎回機械的に適用する
    - 禁忌のペナルティを「呪いのように」効かせる
    - 動機・禁忌を「効率的なボーナス源」として運用する
  
  スキル喪失の暴走:
    - 些細な失敗でスキルを喪失させる
    - 観測可能な描写なしにスキルを奪う
    - スキル喪失を「ペナルティ」として濫用する
```
