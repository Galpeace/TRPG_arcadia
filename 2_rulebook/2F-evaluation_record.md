# 評価記録 様式

## このファイルの位置

PCの評価四層を記録するフォーマットを定義する。章末は軽い記録、周回末で本格的な記録を作る。詳細な評価ルールは `2R-evaluation.md` を参照。

## 三秒で読む要約

評価四層を記録する。章末は要点のみ、周回末で本格化。異名は独立欄、内的・外的呼称を分ける。

## 一分で読む要約

評価記録は四層（直接的・伝聞・構造的・文化的）で構成される。章末では「変動の有無」と「変動内容の要点」のみ簡潔に記録し、周回末で本格的な記録を作成する。異名・評判は独立欄で管理し、「内的呼称（PCが自認）」と「外的呼称（世界の呼称）」を分けて記録することで、中核テーマ「PCの自己認識と世界の評価のズレ」を実装する。yaml主体で構造化し、必要箇所で散文を使う。アーカイバが周回末に本格的な記録を作成する。

----

# 1. 評価記録の基本

```yaml
評価記録の基本:
  
  作成主体:
    - 章末（要点記録）: OPUS設計
    - 周回末（本格記録）: OPUSアーカイバ
  
  形式:
    - yaml主体＋必要箇所で散文
    - 章末記録: 100〜200行程度
    - 周回末記録: 300〜500行程度
  
  位置付け:
    - キャラクターシートの「評価四層の要点」を補完
    - 周回末に本格化する
```

----

# 2. 評価四層の記録構造

```yaml
評価四層の記録構造:
  
  直接的評価（direct）:
    - 個別NPC・組織がPCをどう認識しているか
    - 章中の交流による変化
    - 周回末で確定
  
  伝聞評価（hearsay）:
    - 噂、伝聞、第三者の語り
    - 噂の歪み（数の誇張、動機の誤解、主体の入れ替わり）を含む
  
  構造的評価（structural）:
    - 勢力・組織レベルの認識
    - 賞金、指名手配、要観察などの公的評価
    - 公式記録への登録
  
  文化的評価（cultural）:
    - 伝承、歌、遊び、子供たちの語り
    - 一般民衆の認識
    - 長期的に蓄積される
```

----

# 3. 章末の軽い記録

## 3.1 形式

```yaml
章末記録の形式:
  
  記録内容:
    - 変動があった層のみ記録
    - 変動の要点（詳細は周回末で本格化）
    - 「変動なし」も明示
  
  記録の粒度:
    - 個別NPCの細かな心情変化はキャラシートで管理
    - 本記録は「構造的・文化的に意味のある変化」を中心に
```

## 3.2 テンプレート（章末）

```yaml
chapter_evaluation_record:
  
  chapter: 3
  chapter_title: "ヨプの大森林を抜けて"
  
  direct:
    change: "あり"
    summary: |
      - エルフの長老: 未認識 → 中立的認識
      - アルカディア国王: 未認識 → 観察対象
      - 弟ヴァレリオ: 信頼の深化
    detail_to_summary: "周回末に詳細化"
  
  hearsay:
    change: "あり"
    summary: |
      - 異名「貴族の剣士」が北方ヨウィン周辺で広まる
      - 山賊団撃破の活躍が誇張されて伝わる
        実際: 数人を相手にした → 噂: 三十人を一人で
    bynames_appeared:
      - "貴族の剣士"
  
  structural:
    change: "あり"
    summary: |
      - アルカディア王国: 観察対象として認識
      - その他勢力: 未認識のまま
  
  cultural:
    change: "なし"
    notes: "文化的評価が芽吹くには時間が必要"
  
  notes: |
    本章は評価変動の起点となる重要な章。
    周回末で本格的な記録が必要。
```

# 4. 周回末の本格記録

## 4.1 形式

```yaml
周回末記録の形式:
  
  記録内容:
    - 周回全体での評価の確定
    - 四層すべての詳細
    - 章ごとの変動の集約
  
  作成タイミング:
    - 周回末処理時、OPUSアーカイバが作成
    - キャラクターリスト・世界設定への更新パッチと連動
```

## 4.2 テンプレート（周回末）

```yaml
cycle_evaluation_record:
  
  cycle: 1
  pc_name: "ヴァランディーネ"
  chapters_covered: 8
  
  direct:
    
    individual_npcs:
      - npc: "弟ヴァレリオ"
        relationship: "家族、信頼"
        evolution: |
          周回開始時: 離別状態、再会への思いのみ
          周回中盤: 再会、共に旅をする
          周回末: 深い信頼関係、家族として確立
      
      - npc: "恩師ゴルヴァス"
        relationship: "師、信頼"
        evolution: |
          周回開始時: 数年来の知己
          周回中盤: 数度の助言を受ける
          周回末: 弟子としての地位を確立
      
      - npc: "エルフの長老"
        relationship: "中立的認識"
        evolution: |
          周回開始時: 未認識
          周回中盤: ヨプ大森林の境界を尊重する人物として認識
          周回末: 中立的に友好、限定的な情報交換が可能
    
    notes: |
      直接的評価は、相手NPCの数だけ存在する。
      本記録には主要なものを記載し、その他はキャラシートまたは
      4_character_list/ に分散して記録される。
  
  hearsay:
    
    bynames:
      - byname: "貴族の剣士"
        spread: "北方ヨウィン、ヨプ大森林周辺、アルカディア南部"
        accuracy: "比較的正確"
        notes: "出自を尊重する人々から肯定的に語られる"
      
      - byname: "三十人斬りの女剣士"
        spread: "酒場、街道"
        accuracy: "誇張"
        notes: |
          実際の戦闘は数人だが、噂は誇張されている。
          これがPCの自己認識とのズレを生んでいる。
    
    rumors:
      - story: "ヨプの大森林で山賊団を一人で壊滅させた"
        accuracy: "誇張あり、エルフの協力は伏せられている"
        spread_area: "北方ヨウィン、隣接諸国"
      
      - story: "アルカディア王国の没落貴族の末裔が、剣で世を渡っている"
        accuracy: "比較的正確"
        spread_area: "アルカディア王国南部"
  
  structural:
    
    powers:
      - power: "アルカディア王国"
        classification: "観察対象、潜在的な利用可能性"
        notes: |
          国王が動向を見守っている。
          家門の再興を許す可能性を検討中。
      
      - power: "エルフ族（ヨプの大森林）"
        classification: "中立的、警戒は緩和"
        notes: "境界を尊重した実績が評価されている"
      
      - power: "山賊団残党"
        classification: "敵対、報復の対象"
        notes: "リーダーを失った残党が散発的に動く"
      
      - power: "魔王国"
        classification: "未認識"
        notes: "現時点では魔王国の視野には入っていない"
  
  cultural:
    
    folklore_emerging:
      - story: "貴族の剣士の物語"
        spread: "北方ヨウィン地方の村々"
        stage: "芽吹き段階"
        notes: |
          まだ確立した伝承ではないが、
          一部の村では「正しき道を歩む剣士」として語られ始めている。
      
      - story: "山賊団を片付けた女"
        spread: "酒場の戯れ歌"
        stage: "芽吹き段階"
        notes: "誇張された武勇伝として、酒場で歌われ始める"
    
    children_play:
      - game: "貴族の剣士ごっこ"
        location: "ヨウィン地方の一部の村"
        notes: "子供たちが棒切れを剣に見立てて遊ぶ姿が見られる"
    
    notes: |
      文化的評価は最も時間がかかる。
      周回1の段階では芽吹き程度。
      周回を重ねるごとに確立されていく可能性。
```

# 5. 内的呼称と外的呼称

## 5.1 概念

```yaml
内的呼称と外的呼称:
  
  内的呼称:
    PCが自分自身をどう認識しているか
    - PCの名前
    - PCが自認する役割（「剣士」「貴族の娘」など）
    - PCの動機・禁忌から導かれる自己像
  
  外的呼称:
    世界がPCをどう呼んでいるか
    - 異名
    - 評判
    - 噂で広まる呼称
    - 公的な肩書（指名手配など）
  
  両者のズレ:
    中核テーマ「観測のアルカディア」の核心
    PCの自己認識と世界の評価のズレを明確化
```

## 5.2 記録テンプレート

```yaml
naming_record:
  
  internal:
    - "ヴァランディーネ"  # 本名
    - "貴族の娘"  # 自認
    - "弟を探す者"  # 動機由来
    - "正道を歩む剣士"  # 自認
  
  external:
    
    formal:  # 公的な呼称
      - "観察対象"  # アルカディア王国の記録上
      - "貴族の剣士"  # 一般的な呼称
    
    informal:  # 噂・俗称
      - "三十人斬りの女剣士"  # 誇張された武勇伝
      - "森を抜けた女"  # 一部の旅人の語り
    
    cultural:  # 伝承化された呼称
      - "正しき道を歩む剣士"  # 一部の村での呼称（芽吹き段階）
  
  asymmetry:
    notes: |
      PCの自己認識と世界の評価のズレ:
      - PCは自分を「動機を持つ一人の剣士」と認識
      - 世界は「三十人斬り」「観察対象」「正しき道を歩む者」など
        多様な呼称で認識
      
      とくに「三十人斬り」の誇張は、PC自身は不本意な可能性。
      しかし、これが世界の観測結果である。
```

# 6. テンプレートの運用

## 6.1 厳密性

```yaml
フォーマットの厳密性:
  
  必須項目（章末記録）:
    - chapter, chapter_title
    - 四層それぞれの change（あり/なし）
    - change が「あり」の場合の summary
  
  必須項目（周回末記録）:
    - cycle, pc_name, chapters_covered
    - 四層それぞれの記録（変動があった項目）
    - 内的呼称・外的呼称
  
  任意項目:
    - 各層の詳細な分類
    - 「その他」欄
```

## 6.2 散文の使い方

```yaml
散文を使う箇所:
  
  各 evolution, notes, summary 欄:
    - 散文で記述
    - |（パイプ）を使った複数行yaml
    - 物語的な変化を表現
  
  構造化情報のままの箇所:
    - 名前、勢力名、所属
    - 各種フラグ（change の あり/なし）
    - 段階分類
```

## 6.3 章末と周回末の差異

```yaml
章末と周回末の差異:
  
  章末:
    - 変動の有無のみ確認
    - 要点をsummaryに記述
    - 詳細は周回末まで保留
    - 章末処理段階3で作成
  
  周回末:
    - 周回全体での確定
    - 四層すべての詳細
    - 内的・外的呼称の整理
    - 周回末処理（アーカイバ）で作成
    - キャラクターリスト・世界設定への更新パッチと連動
```

# 7. 違反兆候

```yaml
違反兆候:

  記録の不徹底:
    - 章末で評価変動を確認せず、記録を残さない
    - 周回末で本格記録を作らない
    - 変動なしの場合でも「なし」を明示しない
  
  四層の混同:
    - 直接的評価と伝聞評価を混在させる
    - 構造的評価と文化的評価を区別しない
    - 章末で構造的・文化的評価を本格記録する
      （周回末の役割を奪う）
  
  ズレの解消:
    - 内的呼称と外的呼称のズレを「埋める」記述
    - 「実は誤解されていた」と訂正する
    - 評価の非対称を見落とす
  
  異名の機械化:
    - 異名を「数値スコア」として扱う
    - 異名を「上げるべきパラメータ」として扱う
    - 物語性を欠いた異名生成
  
  キャラシートとの責任の混乱:
    - キャラシートに本格的な評価記録を書く
    - 本ファイルにキャラシートの情報を重複記載
```
