# 周回末更新パッチ 様式

## このファイルの位置

OPUSアーカイバが周回末に生成する「更新パッチ」のフォーマットを定義する。プレイヤーが手動で適用することで、世界設定・キャラクターリスト・クエストリストが更新される。

## 三秒で読む要約

周回末にアーカイバが生成する世界への反映パッチ。プレイヤーが手動適用。新規記述形式、対象別に分けて提示。

## 一分で読む要約

更新パッチは、周回末にOPUSアーカイバが生成する世界への反映候補集である。LLM側がファイルを直接編集する権限を持たないため、プレイヤーが手動で適用する形式を取る。対象別（キャラクターリスト、世界設定、クエストリスト）に分けて提示し、プレイヤーが部分的に採用できる粒度で記述する。差分形式ではなく新規記述形式を採用し、適用しやすさを優先する。アーカイブ（物語的振り返り）とは別物で、パッチは構造化された反映候補、アーカイブは物語的散文。

----

# 1. 更新パッチの基本

```yaml
更新パッチの基本:
  
  作成主体: OPUSアーカイバ
  
  発生タイミング: 周回末処理時
  
  形式:
    - 新規記述形式（差分形式は使わない）
    - 対象別に分けて提示
    - yaml主体＋必要箇所で散文
  
  位置付け:
    - LLM側のファイル編集権限の制約を回避する装置
    - プレイヤーが手動適用する反映候補
    - アーカイブ（物語的振り返り）とは別物
  
  プレイヤーの操作:
    - 全件採用、部分採用、不採用を選べる
    - 手動でファイルにコピー＆ペースト
    - 周回をまたぐセッションで利用
```

----

# 2. パッチの対象

```yaml
パッチの対象:
  
  1: キャラクターリスト（4_character_list/）
     - 新規重要NPCの追加
     - 既存NPCの状態変化
     - 死亡・失踪したNPCの整理
  
  2: 世界設定（1_setting/、3_geography/）
     - 周回中に明らかになった世界の詳細
     - 勢力関係の変化
     - 地名・固有名詞の追加
  
  3: クエストリスト（6_quest_list/）
     - 新規ワールド・クエストの追加
     - 既存クエストの段階更新
     - 完了・失敗したクエストの記録

  対象外:
    - PCのキャラクターシート（同一チャットの文脈で管理）
    - シナリオ仕様書（章単位、周回末では対象外）
    - 評価記録（2F-evaluation_record.md として独立）
```

# 3. テンプレート

## 3.1 全体構造

```
# 周回末更新パッチ

## 周回情報
- PC名: ヴァランディーネ
- 周回番号: 1
- 周回期間: 第1章〜第8章
- 生成日時: 2026-XX-XX

## 1. キャラクターリストへの追記

### 新規重要NPC
（個別NPCのパッチ）

### 既存NPCの状態変化
（個別NPCの変化パッチ）

## 2. 世界設定の更新

### 新規追加項目
（個別項目のパッチ）

### 解像度向上分
（既存項目への追記パッチ）

## 3. クエストリストへの追記

### 新規ワールド・クエスト
（個別クエストのパッチ）

（個別クエストの変化パッチ）

```

## 3.2 キャラクターリストへの追記（新規）

```yaml
new_important_npc:
  
  file: "4_character_list/N-007_賢者ゴルヴァス.md"
  
  content: |
    # 賢者ゴルヴァス
    
    ## 基本情報
    
    - 種族: ヒューマン
    - ジョブ: 賢者、学者
    - 外見: 老年の男性、白い顎髭、フラニエの学者衣装
    - 所属: フラニエ学術連合の出身、現在は自由旅人
    
    ## 役割と関係性
    
    - PCの恩師、剣術と学問を教えた
    - PCの動機「弟を探す」の重要な助言者
    - PCの最大の信頼関係の一人
    
    ## 公言している姿勢
    
    「PCを支援する旅人」「学者として世界を巡る」
    
    ## 本心
    
    PCの正体（または血筋）について何かを知っている。
    完全な真実を伝えるべきタイミングを見計らっている。
    
    ## 自律進行の可能性
    
    - 周回をまたいで存続する可能性が高い
    - 老齢のため、いずれ世を去る運命
    - PCに重要な情報を残す役割を担う可能性

apply_target: "4_character_list/"
apply_method: "新規ファイルとして追加"
```

## 3.3 既存NPCの状態変化

```yaml
existing_npc_update:
  
  file: "4_character_list/N-EXIST-XXX_アルカディア国王.md"
  
  changes: |
    ## 変更点
    
    ### 認識の変化
    
    PCを「観察対象」として認識するようになった。
    家門の再興を許す可能性を内部で検討中。
    
    ### 関係性の変化
    
    - 直接的接触はなし
    - 宰相を通じてPCの動向を監視
  
  add_section: |
    ## PCとの関係（周回1で確立）
    
    - 認識: 観察対象
    - 評価: 潜在的な利用可能性
    - 接触: 未接触、宰相経由で監視
    - 想定される動き: 周回2以降で接触の可能性

apply_target: "4_character_list/N-EXIST-XXX_アルカディア国王.md"
apply_method: "セクション「## PCとの関係」を新規追加"
```

## 3.4 世界設定の新規追加

```yaml
new_world_element:
  
  file: "1_setting/W-007_山賊団とその後.md"  # または既存ファイルへの追記
  
  content: |
    # ヨプの大森林の山賊団
    
    ## 概要
    
    ヨプの大森林の東部を拠点としていた山賊団。
    周回1の第3章で、PCにより主要リーダーが討伐された。
    
    ## 主要メンバー（討伐前）
    
    - リーダー: ハンター・カイル（死亡）
    - 副官: 数名（散在）
    
    ## 魔王国との関係
    
    - 周回1で疑惑が浮上
    - 完全な証拠はないが、密接な関係が示唆される
    - 周回2以降の重要なワールド・クエスト素地
    
    ## 現状（周回1末）
    
    リーダーを失った残党が森林周辺で散発的に活動。
    完全な再編には時間がかかる。
    
    ## 周回をまたぐ可能性
    
    - 残党が再編される可能性
    - 魔王国からの新たな指導者の派遣の可能性
    - PCの動向次第で再び大きな勢力になる可能性

apply_target: "1_setting/"
apply_method: "新規ファイル W-007_山賊団とその後.md として追加"
```

## 3.5 世界設定の解像度向上

```yaml
world_resolution_update:
  
  file: "3_geography/G-005_ヨプの大森林.md"  # 既存ファイル
  
  add_section: |
    ## エルフ集落の境界（周回1で明らかに）
    
    周回1の第3章で、PCがエルフ集落の境界を尊重する行動を取った。
    これによりエルフ族の警戒が一部緩和。
    
    ### 境界の位置
    
    ヨプの大森林東部、北方ヨウィン側からアクセス可能な領域。
    具体的な境界線は、PCの観察に基づいて以下のように確認された:
    
    - 巨大な岩石の遺構が並ぶライン
    - 一定の樹齢を超えた古木が分布する区域
    - 鳥獣の生態が明らかに変化する境目

apply_target: "3_geography/G-005_ヨプの大森林.md"
apply_method: "セクション「## エルフ集落の境界（周回1で明らかに）」を追加"
```

## 3.6 クエストリストへの追記

```yaml
new_world_quest:
  
  file: "6_quest_list/Q-W-005_アルカディア王国とエルフ族の関係.md"
  
  content: |
    # アルカディア王国とエルフ族の関係
    
    ## 分類
    
    サブ × ワールド
    
    ## 概要
    
    アルカディア王国の森林伐採事業がエルフ族との緊張を生んでいる。
    周回1で、PCの介入により一部緩和したが、根本的解決には至っていない。
    
    ## 関与履歴
    
    ### 周回1
    
    - 第3章: PCがエルフ集落の境界を尊重
    - 第4章間: アルカディア使節の訪問（PCの介入なしに自律進行）
    - 第6章間: 森林伐採の縮小決定
    
    ## 現状
    
    - 緊張は緩和したが、完全な信頼関係には至らない
    - PCの今後の関与次第で、関係性が大きく動く可能性
    
    ## 関係する勢力・NPC
    
    - アルカディア王国（国王、宰相）
    - エルフ族（長老）
    - PCヴァランディーネ（周回1で重要な関与）

apply_target: "6_quest_list/"
apply_method: "新規ファイル Q-W-005_xxx.md として追加"
```

## 3.7 適用ガイド

```
## 適用ガイド

以下の手順でパッチを適用してください。

### 1. パッチの確認
全体を一度読み、採用したいものと採用しないものを判断してください。

### 2. 採用するパッチの選別
- 全件採用したい場合: すべてを順次適用
- 部分採用したい場合: 採用したいパッチのみを選択
- 不採用にしたい場合: そのパッチを無視

### 3. ファイル操作
- 「新規ファイルとして追加」: 指定されたファイル名で新規作成
- 「セクション追加」: 既存ファイルに該当セクションを追加
- 「内容更新」: 既存ファイルの該当箇所を更新

### 4. 整合性確認
適用後、既存ファイルとの整合性を確認してください。
矛盾がある場合は、プレイヤーの判断で調整してください。

### 5. 不採用の選択も尊重
すべてのパッチを採用する必要はありません。
プレイヤーの世界観に合わないものは不採用でも構いません。
```

# 4. テンプレートの運用

## 4.1 厳密性

```yaml
フォーマットの厳密性:
  
  必須項目:
    - 周回情報（PC名、周回番号）
    - 各パッチの apply_target
    - 各パッチの apply_method
  
  任意項目:
    - 個別パッチの詳細度
    - 任意の追加情報
  
  変形:
    - 対象がない場合、そのセクションは省略可
    - 適用しにくい場合、注釈を追加してよい
```

## 4.2 アーカイブとの区別

```yaml
アーカイブとの区別:
  
  更新パッチ（本ファイルの様式）:
    対象: ファイル更新用
    形式: 構造化情報、新規記述形式
    目的: 世界設定・キャラリスト・クエストリストへの反映
    読み手: プレイヤー（適用作業のため）
  
  アーカイブ:
    対象: プレイヤーへの物語的提示
    形式: 散文、新装飾仕様も採用可
    目的: 周回の物語的振り返り、祝祭
    読み手: プレイヤー（読み物として）
  
  両者の提示順序:
    1: 更新パッチを提示（プレイヤーが適用）
    2: プレイヤーが確認、適用完了
    3: アーカイブを提示（プレイヤーが読む）
    4: 周回終了の宣言
```

## 4.3 周回をまたぐ運用

```yaml
周回をまたぐ運用:
  
  パッチの目的:
    新規セッション（別チャット）で新規周回を始める際の
    世界の継続性を保証する。
  
  プレイヤーの作業:
    - 周回末で更新パッチを受け取る
    - 該当ファイルに手動で反映
    - 次回のセッションでは更新されたファイルが世界の前提
  
  整合性の維持:
    - パッチは「世界の自然な変化」として書く
    - 既存ファイルと矛盾しないように
    - 矛盾がある場合は、適用時にプレイヤーが調整
```

# 5. 違反兆候

```yaml
違反兆候:

  パッチの形骸化:
    - 機械的に「変化なし」とだけ書く
    - 重要な変化を見落とす
    - 周回中の出来事を反映しない
  
  対象の混乱:
    - キャラクターリストへのパッチに世界設定の内容を混ぜる
    - クエストリストへのパッチにNPC情報を混ぜる
    - 適用先が不明瞭
  
  差分形式の濫用:
    - 「ここを変更」「ここを削除」のような差分形式
    - 新規記述形式の方が適用しやすい
  
  アーカイブとの混同:
    - パッチに物語的散文を多用
    - アーカイブの内容をパッチに含める
    - 両者の役割を曖昧にする
  
  プレイヤーの選択の無視:
    - 「すべて適用してください」と強制
    - 部分採用の余地を残さない
    - 不採用の選択肢を提示しない
  
  権限の越境:
    - 「ファイルを更新しました」と虚偽報告
    - LLMがファイルを直接編集できるかのような記述
    - 詳細: 2O-opus_archivist.md の権限制約セクション
```
