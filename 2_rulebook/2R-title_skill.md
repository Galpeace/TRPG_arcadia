# 称号スキルシステム

## このファイルの位置

ルール本体。称号スキルの定義・特性・設計原則を定める。

確定済みの称号スキル一覧および今後発見されるスキルの記録は、スキルリストファイル（別途作成予定: `2R-skill_registry.md`）に管理される。本ファイルはシステムの定義と原則のみを扱う。

## 三秒で読む要約

特定の実績を達成すると自動付与される、周回・種族を超えて引き継がれるスキル。世界に実在する実績であり、PCはそれを発見するだけ。

----

## 1. 定義

特定の実績を達成したとき、その個体に自動的に付与される体験起因スキル。

----

## 2. 特性

```yaml
特性:
  
  周回・種族をまたぐ引き継ぎ:
    称号スキルは周回・種族にかかわらず引き継がれる。
    解除条件を満たした実績は世界に記録され、
    以降のどの周回・どの種族でも有効となる。
    
    例:
      第1周回でゴブリンとして「カニバリズム」を解除した場合、
      第2周回で人間の冒険者としてプレイしても、
      同種（人間）を食した時点で「カニバリズム」が再発動する。
  
  世界に実在する実績:
    称号スキルは特定のPCやルールが作り出すものではない。
    「世界にそういう実績が存在する」という在り方を持つ。
    PCはそれを発見・解除するだけである。
  
  汎用的な効果:
    効果はいかなる種族・ジョブにも適用できる普遍的な形で記述する。
    特定スキルとの組み合わせを前提にしない。
    ただし、特定スキルと組み合わせると相乗効果が生じる場合は
    その可能性を記述してよい。

定義の三要素:
  - 解除条件: 何をしたときに発動するか
  - 効果: どのような変化が起きるか
  - 可視性: 外部からどの程度察知されるか
```

----

## 3. 確定済みの称号スキル

```yaml
称号スキル一覧:

  血の継承:
    unlock_condition: 肉親を捕食する
    cross_cycle: true
    cross_race: true
    effect: |
      肉親が有していたスキルの中から、最大三つを直接取得する。
      グレードダウンは発生しない。取得するスキルはPCが選択する。
    visibility: |
      外見・言動から直接察知されることはない。
      ただし、肉親のスキルを突然発現させた場合、
      事情を知る者から違和感を持たれる可能性がある。
    note: |
      継承の発現形はPCの在り方によって異なる。
      貪欲を持つ個体であれば直接的な吸収として発現し、
      持たない個体であれば潜在的な素質として眠り、
      特定の条件で覚醒する形をとりうる。

  カニバリズム:
    unlock_condition: 同種を捕食する
    cross_cycle: true
    cross_race: true
    effect: |
      同種を捕食する行為に対して忌避感・嫌悪感が発生しない。
      同種の肉を美味として受け取る。
      食としての同種に対して、他の食材と同列の感覚しか持たない。
    visibility: |
      直接的な外見での察知は不可。
      同種を前にしたときの反応の自然さから、
      感覚の鋭い者に察知される可能性がある。
      「あの個体は仲間を食ったことがある」という感覚的評価として
      現れることがある。
```

称号スキルの完全な一覧・管理は、スキルリストファイル（別途作成予定: `2R-skill_registry.md`）で行う。

## 4. 設計原則

```yaml
称号スキルの設計原則:

  普遍性:
    効果はいかなる種族・ジョブでも発動する普遍的な形で書く。
    「ゴブリンのみ有効」「人間種族のみ」といった制限は設けない。
  
  行動の徹底:
    解除条件は「徹底してそれをやった」という行動に由来する。
    中途半端な経験では解除されない。
    同種を一口かじった程度ではカニバリズムは解除されない。
    肉親を完全に捕食することで血の継承が解除される。
  
  道徳的な意味を持たない効果:
    称号スキルの効果は、その行為の道徳的評価とは独立している。
    「血の継承」は肉親殺しを奨励しない。
    「カニバリズム」は同種食を美化しない。
    ただそういう効果が発動する、という事実のみを記述する。
  
  設計の余地:
    現時点で確定している称号スキルは上記の二つのみ。
    世界には他にも多数の称号スキルが存在することが想定される。
    解除条件が発見されていないものも含め、随時追加される。
```

## 5. スキルリストとの関係

```yaml
スキル管理の構造:
  
  本ファイル（2R-title_skill.md）:
    内容: 称号スキルシステムの定義・特性・設計原則
    役割: 「どういう仕組みか」を説明
    更新頻度: 低（ルール本体のため）
  
  スキルリストファイル（別途作成予定: 2R-skill_registry.md）:
    内容: |
      全ユニークスキル
      全称号スキル
      全種族パッシブ
      その他の主要なスキル
      各々の rank, effect, source, special_note を一覧化
    役割: 「具体的にどのスキルが存在するか」を記録
    更新頻度: 中～高（周回・セッションごとに成長・新規発見）
    管理責任者: プレイヤー＋オプショナルでアーカイバOPUS
  
  関係性:
    スキルリストファイルは、本ファイルで定義されたシステムに基づいて
    具体的なスキルを記録する下位層である。
```

## 6. 違反兆候

```yaml
違反兆候:

  自動付与の濫用:
    - 解除条件を満たしていないのに付与する
    - 「おそらく満たした」という曖昧な判断で付与する
  
  周回引き継ぎの忘却:
    - 新しい周回で解除済みの称号スキルを再解除扱いにする
    - 種族が変わった場合に引き継ぎを無効にする
  
  効果の道徳的着色:
    - 称号スキルの発動をGMが価値判断とともに描写する
    - 「血の継承」「カニバリズム」の発動を咎めるような演出をする
  
  設計の制限：
    - 特定の種族・ジョブにのみ有効な称号スキルを設計する
    - 解除条件に曖昧な閾値を設けて運用しにくくする
```
