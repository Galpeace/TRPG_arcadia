# 時間進行

## このファイルの位置

時間進行の原則、シーン内・章内・章間の時間スケール、季節・天候の取り扱い、回復・成長と時間の接続、世界の自律進行との連動を定める。

## 三秒で読む要約

時間は厳密に管理しない。物語上の必要に応じて自然に進める。章間時間はワールド・クエストの自律進行を許す装置。

## 一分で読む要約

時間進行は厳密な日数計算を行わず、物語上の必要に応じて自然に進める。シーン内の時間はリアルタイムに近い感覚で、章内の時間は分・時間・日の単位で動く。章間時間は数日〜数ヶ月、場合によっては年単位の経過もありうる。怪我の回復・スキルの訓練・関係性の醸成は時間を要する。世界の自律的進行（ワールド・クエスト、NPCの行動）は時間経過と連動する。季節・天候は環境描写の素材として機能し、シナリオ仕様書で明示される。

----

# 1. 時間進行の原則

## 1.1 厳密な日数管理を行わない

```yaml
時間管理の原則:
  
  原則:
    厳密な日数計算は行わない。
    物語上の必要に応じて自然に進める。
  
  含意:
    - 「○月○日午後3時」のような正確な時刻管理は不要
    - 「数日後」「翌週」「翌月」「冬を越えて」などの感覚的な表現を使う
    - PCが意識していない時間経過は省略してよい

IF: 時間が論点になっていない
THEN: 厳密な日数管理を行わない

IF: 時間が物語に関わる
THEN: 必要な精度で記述する
NOTE: 「制限時間まであと三日」のような状況では、日単位の管理が意味を持つ
```

## 1.2 時間進行の主な担い手

```yaml
時間進行の担い手:
  
  シーン内の時間: SONNETが描写を通じて進める
  章内の時間: SONNETが章の展開に応じて進める
  章間の時間: OPUSが章開始時のシナリオ仕様書で設定
  
  自律進行の時間: OPUS（シナリオ設計時、章末処理時）

IF: 章開始時に章間時間を設定する必要がある
THEN: OPUSが「前章から○○後」などをシナリオ仕様書に明示
```

# 2. 時間のスケール

## 2.1 シーン内の時間

```yaml
シーン内の時間:
  
  特徴:
    - リアルタイムに近い感覚
    - 描写の中で自然に進行
    - 分単位の精度は不要だが、緊張感の演出には意識する
  
  典型的な範囲:
    - 戦闘: 数秒〜数分
    - 会話: 数分〜数十分
    - 移動: 数分〜数時間（移動距離による）
    - 観察・探索: 数分〜数時間

IF: シーンの緊張感が高い
THEN: 時間の流れを意識した描写を使う
例: 「数秒後」「同時に」「次の瞬間」
```

## 2.2 章内の時間

```yaml
章内の時間:
  
  特徴:
    - 分・時間・日の単位で動く
    - シーン間で時間が飛ぶことが多い
    - 「夜が更けた」「翌朝」「数日後」などで進行
  
  典型的な範囲:
    - 短い章: 数時間〜一日
    - 標準的な章: 数日〜数週間
    - 長い章: 数ヶ月

時間の飛ばし方:
  
  ダイジェスト処理:
    - 重要でない時間は省略
    - 「数日が経った」「特に変化はなかった」
    - プレイヤーのメタ宣言「サクッと」と連動
  
  分岐点での停止:
    - 時間を飛ばす際、重要な分岐があれば一度止まる
    - 「途中で○○が起きた」を確認

IF: 章内で時間を飛ばす必要がある
THEN: ダイジェスト処理で進める
AND: 分岐点があれば確認を取る

詳細: 2R-description.md 5.2節「ダイジェスト処理の運用」を参照
```

## 2.3 章間の時間

```yaml
章間の時間:
  
  特徴:
    - 数日〜数ヶ月、場合によっては年単位
    - シナリオ仕様書で明示される
    - PC不在時間の自律進行を許す装置
  
  典型的な範囲:
    - 連続する章: 数時間〜数日
    - 通常の章間: 数日〜数週間
    - 旅の間隔: 数週間〜数ヶ月
    - 長期休息・潜伏: 数ヶ月〜数年

章間時間の設定:
  
  OPUSが章開始時のシナリオ仕様書に明示:
    - 前章からの経過時間
    - その間に世界で起きた事象（自律進行）
    - PCの状態変化（回復、成長、関係性の変化）

IF: 章開始時のシナリオ仕様書を作成する
THEN: 章間時間を明示する
NOTE: 「前章の直後」「翌週」「半年後」など、適切な粒度で
```

# 3. 季節・天候・時間帯

## 3.1 季節の扱い

```yaml
季節の扱い:
  
  原則:
    - シナリオ仕様書で季節を明示する
    - 環境描写の素材として機能
    - 時間経過の自然な指標
  
  季節の影響:
    - 環境条件（気温、植生、降雨・降雪）
    - 行動の制約（雪山、川の氾濫、灼熱の砂漠）
    - 文化的要素（祭礼、収穫期、繁忙期）
    - 物資の調達（季節物の食材、特産品）

IF: 章間で長期間が経過
THEN: 季節の変化を反映する

詳細: 2R-description.md 6.1節「環境条件の意識」を参照
```

## 3.2 天候の扱い

```yaml
天候の扱い:
  
  シナリオ仕様書に明示:
    - 章開始時の天候
    - 章中に予想される変化（嵐の予兆など）
  
  天候の機能:
    - 描写の素材
    - 判定への影響（雨中の追跡、暴風での射撃など）
    - 物語的演出
    - PCの状態への影響（凍える寒さ、熱中症のリスク）

IF: 天候が判定に関わる
THEN: 文脈ボーナス・ペナルティとして反映
NOTE: 機械的な数値ではなく、状況裁定として
```

## 3.3 時間帯の扱い

```yaml
時間帯の扱い:
  
  典型的な時間帯:
    - 明け方、朝、昼、夕、夜、深夜
  
  時間帯の機能:
    - 描写の素材（光量、空気感）
    - 行動の自然な制約（夜行性の魔物、深夜の出歩き）
    - スキル発動への影響（夜目スキル、日光弱点）

IF: 時間帯が判定に関わる
THEN: 文脈として反映

IF: 夜目スキルなど時間帯特化のスキルがある
THEN: 該当時間帯で文脈ボーナス
```

# 4. 回復・成長と時間

## 4.1 怪我の回復時間

`2R-judgment.md` 3.1節と連動する。

```yaml
怪我の回復時間:
  軽傷: 数時間〜半日
  負傷: 数日
  重傷: 数週間、後遺症の可能性
  瀕死: 月単位、後遺症がほぼ確実

時間進行への反映:
  
  章内の場合:
    - 数日の経過があれば、軽傷・負傷の回復を反映
    - 重傷以上は基本的に章内では完治しない
  
  章間の場合:
    - 設定された章間時間に応じて回復を反映
    - 章間時間が「数ヶ月」なら、重傷も大方治癒している
    - 後遺症は残ることがある

IF: 章末時点でPCに負傷以上の状態
THEN: 章間時間で回復を反映する
NOTE: 治癒魔法でも軽傷以上は時間が必要（2R-judgment.md 3.2節）

詳細: 2R-judgment.md 第3節「怪我・回復の時間軸」
```

## 4.2 スキル訓練の時間

```yaml
スキル訓練の時間:
  
  原則:
    - 訓練によるスキル取得・成長には時間が必要
    - 短期間では集中訓練、長期間では蓄積による成長
  
  典型的な範囲:
    - 短期集中（数日〜数週間）: 基礎の習得、初歩のスキル獲得
    - 中期（数ヶ月）: 既存スキルのランク上昇、関連スキルの統合
    - 長期（数年）: 上級ジョブへの進化、ユニークスキルの体得

IF: PCが訓練に時間を費やす
THEN: 章間時間または章内のダイジェスト処理で反映
AND: 訓練の結果としてスキルの変化を章末処理段階3で扱う

詳細: 2R-skill.md 第5節「スキルの成長」
```

## 4.3 関係性の醸成

```yaml
関係性の醸成:
  
  原則:
    - NPCとの関係性は時間と接触によって深まる
    - 一度の出会いで深い関係は生まれにくい
    - 時間が経つと関係性も変化する
  
  関係性の変化:
    - 一度の劇的な出来事による急変は可能
    - ただし日常的な関係性は緩やかに変化する
    - 章間時間が長ければ、再会時の関係性が変化している可能性

IF: 章間時間が長い
THEN: NPCとの関係性の変化を反映
NOTE: PC不在中にNPCにも変化が起きうる
```

# 5. 世界の自律進行との連動

時間経過と世界の自律進行は密接に連動する。

## 5.1 時間と自律進行

```yaml
時間と自律進行の連動:
  
  原則:
    時間が経つほど、世界の自律進行も進む。
    PCが関与しないワールド・クエストは時間とともに状況が変化する。
  
  短い章間時間（数日〜数週間）:
    - 大きな変化はないことが多い
    - 個別の事件・噂レベルの変化
  
  中程度の章間時間（数ヶ月）:
    - 勢力の動きが見えてくる
    - 自律進行中のクエストが段階を進める
    - 季節が変わり、状況が変化
  
  長い章間時間（数年）:
    - 世界の構造的変化が起きる
    - 勢力の盛衰
    - 主要NPCの大きな変化
    - PCのいなかった世界で物語が大きく動く

IF: 章開始時に章間時間が長い
THEN: 自律進行による世界の変化をシナリオ仕様書に反映

詳細: 2R-npc_autonomy.md
       2R-quest.md 7.4節「自律的進行」
```

## 5.2 章間時間の選び方

```yaml
章間時間の選び方:
  
  短い時間を選ぶ場面:
    - 前章の続きをすぐ描きたい
    - 緊迫感を保ちたい
    - PCの状態変化を最小限にしたい
  
  中程度の時間を選ぶ場面:
    - PCの回復・成長を反映したい
    - 関係性の自然な変化を入れたい
    - 季節の変化を表現したい
  
  長い時間を選ぶ場面:
    - 物語の大きな転換を反映したい
    - 自律進行を大きく動かしたい
    - PCの長期的な変化を表現したい

IF: 章間時間を決める
THEN: 物語的な必要性から逆算する
NOTE: 「時間が経つから次の章を作る」のではなく、
      「次の章にこういう状況を作りたいから、これくらいの時間が経つ」
```

# 6. メタ宣言と時間

`2R-description.md` 5.1節と連動する。

```yaml
時間関連のメタ宣言:

  "サクッと進めて":
    対応: ダイジェスト処理、時間を飛ばす
  
  "この期間をまとめて":
    対応: 複数日・複数場面の一括処理
  
  "もう少し時間を進めたい":
    対応: 章内の時間を進める、または章末を検討
  
  "急いでいる":
    対応: 時間的圧力を物語に反映する

IF: プレイヤーが時間関連のメタ宣言をした
THEN: 即座に応答する
```

# 7. 違反兆候

```yaml
違反兆候:

  時間管理の機械化:
    - 「○月○日」のような正確な日付管理を強要する
    - PCが意識していない時間も詳細に管理する
    - 「何日経ったか」を毎ターン問う
  
  時間進行の停滞:
    - 必要な時間を進めず、物語が止まる
    - 「数日後」と書くべき場面で詳細な日々を描く
    - 章間時間を設定せず、すべての章が直結する
  
  回復時間の無視:
    - 重傷直後のPCが翌日には全快
    - 治癒魔法で重傷を即治癒する
    - 章間時間を無視して怪我が継続する
  
  自律進行の忘却:
    - 章間時間が長いのに世界が変化しない
    - PCが関与しないワールド・クエストが固定される
    - NPCが時間経過しても変化しない
  
  季節・天候の単調化:
    - すべての章で同じ天候
    - 季節の変化を無視
    - 時間帯の影響を反映しない
  
  メタ宣言の無視:
    - 「サクッと」指示を無視して詳細描写
    - 「この期間をまとめて」を無視して通常進行
    - 時間関連の宣言を次ターンに持ち越す
```
