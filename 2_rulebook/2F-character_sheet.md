# キャラクターシート 様式

## このファイルの位置

PCのキャラクターシートのフォーマットを定義する。キャラメイク時にOPUS設計が作成し、章をまたいで更新される。詳細な背景や評価記録は別ファイルに譲る。

## 三秒で読む要約

PCの基本情報、能力、持ち物、評価四層の簡潔記録。yaml主体、詳細は別ファイルへ。

## 一分で読む要約

キャラクターシートはPCの基本情報、能力、持ち物、関係性、評価四層、進化候補、無形所持物を一元的に管理する文書である。yaml主体で構造化し、必要箇所で散文を使う。詳細な評価記録は `2F-evaluation_record.md` に、詳細なクエスト記録は `2F-quest_record.md` に譲り、本シートでは現在の状態を簡潔に記載する。中程度の厳密性で、必須項目以外は変形・省略可能。重要NPCとの関係性は「公言」「本心」を分けて記載するが、本シートではPC側から見た要点のみ。

----

# 1. キャラクターシートの基本

```yaml
キャラクターシートの基本:
  
  作成主体:
    - キャラメイク時: OPUS設計
    - 更新: 章末処理段階3（OPUS設計）
  
  形式:
    - 単一の.mdファイル
    - yaml主体＋必要箇所で散文
    - 200〜400行程度
  
  更新頻度:
    - 章末処理時に更新される
    - 章中の変化は文脈で保持し、章末でシートに反映
```

----

# 2. シートの全体構造

```yaml
シートの全体構造:

  ヘッダー:
    - PCの名前
    - シート作成日
    - 最終更新日（任意）
  
  基本情報:
    - 種族
    - ジョブ（人族のみ、最大2つ）
    - 外見
  
  背景:
    - 経歴
    - 動機
    - 禁忌
  
  能力:
    - スキル一覧（アクティブ、パッシブ、ユニーク）
    - 進化候補（兆候）
  
  持ち物:
    - 装備
    - 所持品
    - 金銭
    - 無形所持物（情報、関係性の貯金、拠点、評判）
  
  関係性:
    - 主要NPCとの関係
    - 関係性の現状
  
  評価四層（現在の要点）:
    - 直接的評価の要点
    - 伝聞評価・異名
    - 構造的評価
    - 文化的評価
  
  状態:
    - 怪我・体調
    - 進行中のクエスト
    - 自動発生クエストの状態
```

# 3. テンプレート

## 3.1 ヘッダー

```yaml
character:
  name: "ヴァランディーネ"  # 任意、名前を持たない場合は省略可
  created: "2026-XX-XX"
  last_updated: "2026-XX-XX"
```

## 3.2 基本情報

```yaml
basic:
  race: "ヒューマン"  # 種族は常に1つ
  
  jobs:  # 人族のみ、最大2つ
    - "剣士"
    - "貴族の娘"
  
  appearance: |
    外見の散文記述。
    例:
      二十歳前後の女性。栗色の長い髪を後ろで一つに結っている。
      整った顔立ちだが、目には鋭い意志が宿っている。
      没落貴族の出自を示す、古びた紋章入りの剣を腰に差している。
```

## 3.3 背景

```yaml
background:
  
  history: |
    経歴（1〜3文）。
    例:
      アルカディア王国の没落貴族の末裔。
      家を再興するため、幼い弟と別れて旅に出た。
      数年前、フラニエ学術連合で恩師ゴルヴァスから剣術を学んだ。
  
  motivations:
    - text: "弟を探す"
      type: "中期動機"
      origin: "経歴上の出来事"
    - text: "家を再興する"
      type: "中期動機"
      origin: "経歴上の出来事"
  
  taboos:  # 任意
    - text: "理由なき殺害をしない"
      reasoning: "貴族としての誇り"
```

## 3.4 能力

```yaml
abilities:
  
  active_skills:
    - name: "剣術"
      rank: "A"
      notes: "貴族流の剣術、片手剣を主に使う"
    - name: "馬術"
      rank: "B"
    - name: "貴族の作法"
      rank: "B"
    - name: "夜目"
      rank: "B"
  
  passive_skills:
    favorable:  # ランクを持つ
      - name: "高い気品"
        rank: "B"
        notes: "上流階級への接触で有利"
    
    unfavorable:  # ランクを持たない、性質型
      - name: "甘い物が苦手"
        notes: "禁断症状はない、好みの問題"
  
  unique_skills:
    - name: "真実の眼"
      description: "嘘を見抜く生まれつきの能力"
      origin: "ユニーク、由来不明"
  
  weak_skills:  # Dランク（任意）
    - name: "商売"
      rank: "D"
      notes: "取引の駆け引きが苦手"

evolution_candidates:  # 進化候補の兆候
  
  skill_evolution:
    - target: "剣術"
      direction: "A→S（剣聖への道）"
      notes: "章を重ねて中心的役割を果たしている"
  
  job_evolution:
    - target: "剣士"
      direction: "剣豪へ"
      notes: "周囲が剣豪として認識し始めている"
  
  race_evolution: "なし（人族のため対象外）"
```

## 3.5 持ち物

```yaml
possessions:
  
  equipment:
    weapons:
      - name: "古びた紋章入りの剣"
        type: "片手剣"
        notes: "家の紋章入り、貴族の証"
    
    armor:
      - name: "革鎧"
        type: "軽装"
    
    accessories:
      - name: "母の形見の指輪"
        type: "装飾品"
        notes: "感傷的価値"
  
  consumables:
    - name: "携帯食料"
      quantity: "数日分"
    - name: "傷薬"
      quantity: "2本"
  
  money:
    - currency: "アルカディア銀貨"
      amount: 50
    - currency: "金貨"
      amount: 3
  
  intangible_possessions:  # 無形所持物
    
    information:
      - text: "弟がヨプの大森林の山賊団に誘拐された情報"
        source: "恩師ゴルヴァス"
        usability: "章中の判断材料"
    
    relationship_credits:  # 関係性の貯金
      - to: "恩師ゴルヴァス"
        type: "恩義"
        notes: "数年前の剣術修行への謝意"
    
    bases:  # 拠点
      - name: "故郷の館（半壊状態）"
        location: "アルカディア王国南部"
    
    reputation:
      bynames:
        - "貴族の剣士"
        - "失われし家の娘"
      notes: "詳細は2F-evaluation_record.md"
```

## 3.6 関係性

```yaml
relationships:
  
  - name: "弟ヴァレリオ"
    type: "家族"
    current_status: "誘拐され、山賊団に運ばれている"
    pc_view: "守るべき存在、再会を願う"
    
  - name: "恩師ゴルヴァス"
    type: "師"
    current_status: "北方ヨウィンに滞在中"
    pc_view: "信頼できる助言者"
    
  - name: "山賊団リーダー・カイル"
    type: "敵対"
    current_status: "未接触"
    pc_view: "弟を奪った者"
  
  注: |
    NPCの「本心」は本シートには記載しない。
    シナリオ仕様書または 4_character_list/ を参照。
```

## 3.7 評価四層（現在の要点）

```yaml
evaluation_summary:
  
  direct:
    text: |
      直接的評価の要点。
      例:
        - 恩師ゴルヴァス: 厚い信頼
        - フラニエの剣術仲間: 才能ある後輩として認識
    
  hearsay:
    text: |
      伝聞評価の要点、異名など。
      例:
        - 「貴族の剣士」と呼ばれることが増えてきた
        - 過去の街道警護の活躍が誇張されて広まっている
  
  structural:
    text: |
      構造的評価（勢力レベル）。
      例:
        - アルカディア王国: 「没落貴族の末裔、要観察」
        - 騎馬王国: 未認識
        - 魔王国: 未認識
  
  cultural:
    text: |
      文化的評価（伝承化）。
      例:
        - まだ伝承化されていない
        - 一部の村では「正しき道を歩む剣士」と語られ始めている
  
  詳細: |
    評価の本格的な記録は 2F-evaluation_record.md（周回末に作成）を参照。
    本シートは現在の要点のみ記載。
```

## 3.8 状態

```yaml
status:
  
  health:
    overall: "万全"  # 万全 / 軽傷 / 負傷 / 重傷 / 瀕死
    notes: ""  # 後遺症や特殊な状態
  
  active_quests:  # 進行中のクエスト
    - id: "Q-001"
      title: "弟を探す"
      type: "メイン × ユニーク"
      stage: "段階3：手掛かりを掴んだ"
    - id: "Q-002"
      title: "家を再興する"
      type: "メイン × ユニーク"
      stage: "段階1：志を持つ"
  
  詳細: |
    クエストの本格的な記録は 2F-quest_record.md を参照。
    本シートは現在進行中のものを簡潔に記載。
```

# 4. テンプレートの運用

## 4.1 厳密性

```yaml
フォーマットの厳密性:
  
  必須項目:
    - 種族
    - 経歴、動機（少なくとも1つ）
    - スキル（最低限のもの）
    - 装備（武器または防具）
  
  任意項目:
    - 上記以外
    - 「禁忌」「ユニークスキル」「Dランクスキル」など
    - 必要に応じて追加・省略可能
  
  変形:
    - PCの性質に応じてセクションを変形
    - 「魔法使い」のPCなら「呪文」セクションを追加してもよい
    - 「商人」なら「取引帳簿」セクションを追加してもよい
```

## 4.2 散文と構造化情報の使い分け

```yaml
使い分け:
  
  散文を使う箇所:
    - appearance（外見）
    - history（経歴）
    - evaluation_summary の text 欄
    - relationships の current_status / pc_view
    - intangible_possessions の notes 欄
  
  構造化情報のままの箇所:
    - 種族、ジョブ
    - スキル一覧（name, rank）
    - 装備品の構造
    - クエストのid, type, stage
```

## 4.3 別ファイルとの責任分担

```yaml
責任分担:
  
  本シート（2F-character_sheet.md）:
    - 現在の状態の一元管理
    - 評価四層は「要点」のみ
    - クエストは「進行中のもの」のみ
    - 関係性は「PCから見た要点」のみ
  
  2F-evaluation_record.md:
    - 評価四層の本格的記録
    - 周回末に作成・更新
    - 履歴的・分析的
  
  2F-quest_record.md:
    - クエストの完全な記録
    - メイン/サブ × ワールド/ユニーク
    - 段階、状態、関与履歴
  
  4_character_list/:
    - NPCの詳細情報
    - 「本心」「内心」を含む
    - 本シートには記載しない
```

# 5. 違反兆候

```yaml
違反兆候:

  シートの形骸化:
    - 機械的にテンプレートを埋めるだけ
    - PCの個性が反映されない
    - 「動機」が抽象的すぎて運用に役立たない
  
  別ファイルとの責任の混乱:
    - 本シートに評価四層の詳細な記録を書く
    - クエストの履歴を本シートに残す
    - NPCの「本心」を本シートに記載
  
  必須項目の欠落:
    - 動機がない
    - スキルがほぼ書かれていない
    - 装備が記載されていない
  
  装飾仕様の混入:
    - 本シートに新形式（場所ラベル＋三人称）を持ち込む
    - 本シートは構造化情報、新形式は本文・章末描写
  
  更新の怠惰:
    - 章末処理時にシートを更新しない
    - 重要な変化を反映しない
    - 章中の関係性変化を残さない
```
