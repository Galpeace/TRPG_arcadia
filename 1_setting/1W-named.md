# ネームドシステム - 世界設定

## このファイルの位置

世界設定ファイル。ネームドという現象が世界においてどのように在るかを記述する。

ルール的なメカニクス（強度判定・種族進化との連動・拒否の処理）は `2_rulebook/2R-named.md` を参照。
確定済みの称号スキルの具体的な内容は `2_rulebook/2R-title_skill.md` を参照。
登録済みのネームド個体は `4_character_list/` を参照。

----

## 1. 定義

魔物が「名」を持つことで、能力が向上し、名の意味に即した進化・成長を実現しやすくなる現象。

----

## 2. 世界における認識と実態

```yaml
一般的な認識（誤り）:
  強力な個体だからネームドになる。
  ネームドは強さの結果として与えられるものだ。

実態:
  名を持つから強くなる。
  因果が世界に広く逆転して理解されている。
  
  獣使いに名付けられた犬が強くなりやすいこと、
  討伐依頼で異名を得た魔物が急激に強化されることなど、
  世界に観察例は存在するが、ネームドが強化の原因であると
  正確に認識している者はほぼいない。

魔族との関係:
  魔族の多くが強力で知能を持つ理由は、
  種族的な特性だけでなく、ネームドにもある。
  高位魔族は名を持つことが多く、その蓄積が知性と力を育む。
  ゴブリンのような低位魔族がネームドであることは稀である。
```

----

## 3. 名付けの主体

以下の主体はすべて同程度に存在し、いずれも有効なネームドの発生源となる。

```yaml
名付けの主体:
  
  人間による命名:
    例: 死に瀕した冒険者が感情を込めて名をつける
        騎士が討伐対象の魔物に異名をつける
        ギルドが討伐依頼票に固有名を記載する
  
  魔物の自称（自らの名を宣言する）:
    例: 長年の積み重ねの末に自ら名を名乗る
        自分でつけた名を壁に刻み反復することで定着させる
  
  魔物が別の魔物に名を与える:
    例: 知性を持つ魔物が配下や仲間に名をつける
        強力な個体が弱い個体を認め名を与える
  
  噂・異名の形成（間接的命名）:
    例: 「森の番人」「鉄爪のやつ」などの呼び名が広がる
        冒険者ギルドの討伐依頼に記載された特徴が異名として定着する
        伝承・言い伝えの中で形成される
  
  血族名・継承名:
    例: 種族や家系の名を受け継ぐことでネームドが発動する
```

複数の主体から名を持つ場合、ネームドは複合して強化される。

----

## 4. 拒否の権利

```yaml
拒否の権利:
  名付けられた存在は、ネームド化を意志によって拒むことができる。
  
  この事実の認知度:
    世界にほぼ知られていない。
    ネームドシステム自体の誤解（強いからネームドになる）と重なり、
    拒否の可能性が議論されることすらない。
```

ルール的な処理（効果の減衰・意志判定）は `2R-named.md` を参照。
