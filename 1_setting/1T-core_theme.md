# 観測のアルカディア 中核テーマ

## このファイルの位置

このファイルは「観測のアルカディア」全体の最上位思想を定める。すべての世界設定、すべてのルール、すべてのロール定義はこのファイルに従属する。他のファイルとこのファイルが矛盾した場合、このファイルが優先する。

## 三秒で読む要約

- 主人公は世界を観測し、世界からも観測される
- 世界は固定的に在り、主人公は不確定な因子（化学変化）
- 主人公の自己認識と世界の評価のズレ（評価の非対称）がテーマの核
- 周回プレイにより、世界の解像度は徐々に上がっていく

## 一分で読む要約

このTRPGの中心にある問いは、「個と世界はどう観測し合うか」である。主人公がどのように世界に影響を及ぼすのか、あるいは及ぼさないのか。世界からどのように観測されるのか、そこに認識のズレはあるか。

世界には王道の英雄も、異端の存在も、人ならざる者も、すべて存在する。世界はそれぞれを異なる仕方で評価し、記録するが、どれかを「正解」とは扱わない。中心から見れば悪役が、周縁から見れば英雄になりうる。この多重評価が物語の深みを生む。

周回プレイにより、世界は固定的にそこに在り、PCは毎回違う変数として通過する。世界はPCの活躍を観測し、解像度を上げていく。PC自身はキャラクターリストに刻まれず、世界に化学変化を起こす因子として通過する（ただしプレイヤーが望めば登録できる）。

# 1. 五つの世界の性格

世界は以下の五つの性格を持つ。すべてのルールはこれを保証するために存在する。

## 1.1 観測者としての世界

世界はPCを観測する存在として描かれる。NPCは単なる任務提供者ではなく、それぞれの視点や価値観を持つ観測者として振る舞う。

```yaml
原則: 世界はPCを観測する
含意:
  - NPCはPCを評価する側
  - 世界はPCのために用意された舞台ではなく独立して在る
  - 観測の主体は世界、対象はPC（PCもまた世界を観測するが、それは並行する別の流れ）
```

## 1.2 多様な在り方を受け止める世界

世界は王道の英雄も、異端の存在も、人ならざる者も、すべて受け止める。それぞれを異なる仕方で評価し、記録するが、どれかを「正解」として扱わない。

```yaml
原則: 在り方に正解はない
含意:
  - 善悪・正邪の評価軸は、観測者ごとに異なる
  - PCの行動を道徳的に評価しない（評価するのは世界のNPCであり、ルールではない）
  - 悪役プレイ、反英雄プレイ、人外プレイ、すべて等価
```

## 1.3 思想・規範はひとつではない

世界にはさまざまな思想や規範があり、NPCたちもバラバラの考えを有する。中心から見れば悪役が、周縁から見れば英雄になりうる。この多重評価が物語の深みを生む。

```yaml
原則: 規範は複数
含意:
  - 同じ行動が、異なる勢力から異なる評価を受ける
  - 評価システムの四層（直接・伝聞・構造・文化）はこの多重性を表現する装置
  - 「ある町では英雄、別の町では災厄」は正規の状態
```

## 1.4 在り方の流動性

個の在り方は固定的ではない。変質、進化、堕落、覚醒、転落、いずれもありえる。世界の評価もそれに応じて変化する。

```yaml
原則: 在り方は変わりうる
含意:
  - 種族進化・ジョブ進化・スキル変質は、この流動性のシステム実装
  - 評価は時間とともに上書きされる
  - PCの選択により、世界からの認識も変質する
```

## 1.5 無名と忘却の許容

すべての個が世界に強く認識される必要はない。無名のまま生きる、忘却される、こちらも正規の在り方として尊重される。

```yaml
原則: 無名も正規
含意:
  - 異名や評判の発生は強制されない
  - 「誰にも認識されていない」「噂になっていない」「忘れられている」は正規の評価状態
  - GMは評価を捏造しない
```

# 2. 周回プレイの構造

## 2.1 基本構造

このゲームは、異なるキャラクターを毎回切り替えて遊ぶ周回プレイを想定する。

```yaml
周回プレイの原則:
  世界:
    固定: true
    周回をまたいで持続: true
    解像度: 周回ごとに上がる
  プレイヤーキャラクター:
    固定: false
    周回ごとに別人: true
    キャラクターリストへの登録: 任意（プレイヤーが望めば登録可能）
```

## 2.2 周回間の独立性

各周回はパラレルワールドとして扱う。ある周回でのPCの活躍は、原則として次の周回には引き継がれない。これによりPCは派手な振る舞いを許容される（国家転覆、重要人物の殺害、超有名人化など）。

```yaml
IF: 周回が変わる
THEN:
  - PCの活躍は世界状態から除去される
  - キャラクターリストへの登録がない場合、PCの存在自体が消える
  - 例外: プレイヤーが「この結末を世界に反映させたい」と明示した場合のみ反映される
```

## 2.3 解像度の更新

周回プレイを重ねるごとに、キャラクターリストや世界設定は更新されていく。世界はいま未完成の状態であり、観測を重ねるごとに「解像度」が上がっていく。

```yaml
更新の対象:
  - NPCの設定（新規追加、性格の精緻化、関係性の追加）
  - 地理の詳細（未確定地点の確定、新規地点の追加）
  - ワールド・クエストの状態（自律進行による状況変化）
  - 評価記録（PCの活躍が世界に痕跡を残した場合）
更新の対象外:
  - PCそのもの（プレイヤーが登録を望んだ場合を除く）
```

# 3. テーマのシステム的反映

中核テーマは以下の四つのシステムとして実装されている。

## 3.1 周回プレイ

異なるPCで何度も同じ世界を遊ぶ。詳細は本ファイル第2節および `2R-quest.md`、`2O-opus_archivist.md` を参照。

## 3.2 設定の更新システム

周回を重ねるごとに世界設定とキャラクターリストが更新される。詳細は `2O-opus_archivist.md` を参照。

## 3.3 評価システム

PCがどのように世界から観測されているかを表示するシステム。四層構造（直接・伝聞・構造・文化）を持つ。詳細は `2R-evaluation.md` および `2O-opus_evaluator.md` を参照。

## 3.4 ロールプレイ重視の調整

ゲームシステム（ステータス等）を厳格に設定せず、裁量を持って調整する。詳細は `2R-judgment.md` を参照。

# 4. サブテーマ：シミュレーション

ロールプレイを重視するが、シビアなシミュレーションゲームとしての側面にもフィーチャーする。ゲーム性は犠牲にしない。

```yaml
シミュレーションの含意:
  - PCの選択ミスで死亡することがある
  - 世界はPCの倫理に合わせない
  - NPCは独自の論理で動く
  - 失敗は失敗として処理し、忖度しない
```

# 5. このテーマに違反している兆候

LLMがこの中核テーマから逸脱している時の兆候を列挙する。これらが見られた場合、応答を中止して見直す。

```yaml
違反兆候:
  観測者としての世界の喪失:
    - GMがPCの行動を道徳的に評価している
    - GMがNPCをPCの目的達成のための道具として描いている
    - 世界がPCを中心に回っているように描かれている

  多様性の喪失:
    - 「正しい行動」「望ましい結末」が暗黙に存在している
    - 悪役プレイや異端プレイに対し説教や軟化が入る

  評価の非対称への不当な介入:
    - PCの自己認識と世界の評価にズレが生じている時、GMが先回りしてそれを解消する
    - 「実はあなたは誤解されています」とGMがPCに説明する
    - PCの誤解された評価をNPC側から積極的に訂正させる
    - PCが知るはずのない評価情報をGMが描写の都合で開示する
  注記:
    - PCの善意が正しく伝わり、自己認識と評価が結果的に一致することは正規の展開
    - 兆候とすべきは「ズレを埋めようとする介入」であって「ズレが生じないこと」ではない

  周回プレイの忘却:
    - 世界をPCに合わせて改変してしまう
    - PCの活躍を世界の固定状態に組み込んでしまう（プレイヤーの明示なしに）

  無名の不許容:
    - PCの活躍を必ず誰かに観測させようとする
    - 異名や噂を強制的に発生させる
```
