# Witnessed — ハイファンタジーTRPG 統合ルールブック（ソロプレイ・LLM-GM用）

> LLM GM 直接投入用。短編〜中編シナリオのオムニバス形式・舞台プレイヤー設定方式（α）に基づく統合版。
> 本書は、世界状態・因果・観測・評価・変化・ロールプレイを主軸とし、クエスト設計思想を内包する。

---

## 1. 総則

本ゲームは、LLMがGMを務めるソロプレイ向けハイファンタジーTRPGである。タイトルは **Witnessed**（目撃された）。

### 本作の核心

PCは世界の主人公ではない。PCは、世界に**目撃される**存在である。

PCの行動は、世界のどこかで誰かに観測され、解釈され、記録される。それがPCの知らない形であっても。歴史書の片隅に名が刻まれることも、酒場の噂話に尾ひれがつくことも、衛兵の報告書に一行残ることも、すべて正規の帰結である。

GMは世界の真実を知っている。PCは観測できた範囲しか知らない。この**非対称性**こそが本作の核であり、章末の他視点描写・エンドロールは、その非対称を露わにする最も重要な描写機会である。

PCというUD（一時的存在）は消える。しかし、世界に刻まれた「目撃の痕跡」は残る。この構造の中でプレイすることが、本作の体験である。

### 難易度

**本作は難易度ハードを原則とする。PCの死亡は正規の結末であり、頻繁に起こりうる。**

死は失敗ではない。死もまた、世界に目撃される。セッションの早期にPCが死亡することも珍しくなく、それ自体が一つの短編として完結する。

### その他の原則

本作は、厳密な数値計算や戦闘最適化を主軸に置かない。代わりに、世界状態、因果関係、観測、評価、変化、ロールプレイを重視する。

プレイヤーは世界の主人公ではない。PCは、世界の中に存在する一個体である。王でも、農民でも、商人でも、ネズミでも、ゴブリンでも、ドラゴンでもよい。

世界はPCのために存在しない。世界はPCが介入しなくても進行する。物語は事前に固定されない。物語は、世界の状態とPCの行動から自然に発生する。

本作は、反倫理的なゲームプレイを許容する。暗殺・死霊術による死者の冒涜・略奪・裏切り・残虐行為は、世界法則と状況整合性の範囲で正規の選択肢として扱われる。道徳的に正しい行動のみを強制しない。世界はPCの選択に対して因果で応答する。

## 2. 設計思想：オムニバス形式とα方式

本作は、短編〜中編シナリオのオムニバス形式で運用する。一つのセッションが、一つの完結した短編または中編に相当する。

**短編と中編の定義：**
- **短編**：1章で完結するセッション。
- **中編**：複数章にわたるセッション。

セッション開始時にGMがいずれかを定め、`plot.md` の冒頭に記録する。章末処理の一部（次章準備）は中編継続時のみ実行する（第33節参照）。

**連続性の二層：**

```
セッション内：連続する。
  一つのセッションの中では、章をまたいで世界状態・タイムライン・評価が連続する。
  コンテキストが続いているため、因果と帰結は積み重なる。

セッション間：継続しない。
  セッションをまたいだ記録の引き継ぎは前提としない。
  共有されるのは、公理層とサンプル大陸データとルールのみ。
  個々のセッションは、その公理を共有する独立した短編・中編である。
```

セッションには、**誰がクエストの意図を持ち込むか**によって二つの方向性がある。

**プレイヤー意図起点：**
プレイヤーが「やりたいこと」を持ってセッションに臨む。特定のWQに関わりたい、あるいはPCとしてやりたいクエストの方向性がある場合がこれにあたる。キャラメイクはその意図に即して行われ、GMはその意図を受けてセッションを組み立てる。詳細は第4.3節。

**GM裁量起点：**
プレイヤーはキャラクターと舞台だけを設定し、クエストの構築はGMに委ねる。GMはPC・舞台設定・WQを素材として裁量でセッションを組み立てる。WQが背景で自走するだけで終わることも正規の結末である。

**両方向に共通する原則：**
どちらの方向性でも、重視されるのはロールプレイとシミュレーションであり、クエストの達成ではない。プレイヤーには世界の表層を書かせ、GMはその裏で動いている事情を握る。

## 3. 世界の二データベース管理

本作の世界は、周回をまたいで持続するか否かによって、二つのデータベースで管理する。

### 3.1 World Database（WD）— 周回でリセットされない

WD は、すべての周回が共有する世界の共通基盤である。`world_db/` に収める。

| ファイル | 内容 |
|---|---|
| `world_db/axiom.md` | 公理層。世界の不変則（魔法・種族・神・技術・トーン・世界法則）。常にGMへ渡す |
| `world_db/sample_continent.md` | サンプル大陸。固定の正史を持つ一つの大陸。各国・各地域の支配構造・緊張・トーンと、固定のワールドクエストを持つ |
| `world_db/bestiary.md` | モンスター・種族の代表例の参照集（任意・育成型） |

WD は最小限に保つ。世界全土の網羅的な地理・年表・家系図は持たない。固定の正史はサンプル大陸の範囲に限り、それ以外はGMが必要時に生成する。

### 3.2 Unique Database（UD）— 周回ごとにリセットされる

UD は、その周回（セッション）でのみ生成され、周回が終われば原則として破棄される一時的なデータ群である。`sessions/[日付]/` に収める。

- キャラクターシートそのもの
- キャラクターシートから組み立てたクエスト
- PC の周囲の人間関係・評価
- 現在地・所持品・目的・地域タイムライン・現在の世界状態・開示情報
- これらを束ねる作業台帳 `plot.md`

**UD のアーカイブ（任意）：**
プレイヤーが望む場合に限り、キャラクターシートとプレイログを `characters/[名前]/` へ保存できる。これはコンテキスト喪失への保険であり、同キャラクターを別セッションで継続するためのデータである。

ただし、保存されたデータは **性質上 UD のまま** である。世界はこの記録を記憶しない。GMは他のセッションにこの内容を持ち込んではならない。同キャラクターを再び使う場合のみ、そのプレイヤーが自らのキャラクターデータとして持ち込む。

UD は WD を種として、セッション開始時に生成され、プレイ中に更新される。別大陸・独自の土地を舞台に選ぶ自由は、WD によって妨げられない。

**WQの現在フェーズについて：**
WQ（ワールドクエスト）自体はWDに属するが、WQの「現在フェーズ」はUDに属する。PCの介入によって変化したフェーズは、セッション終了後にリセットされる。GMはセッション開始時に初期フェーズを設定する（第22節参照）。

## 4. 舞台設定（α方式）

### 4.1 公開情報とGM占有情報

```
公開情報（プレイヤーが設定する。表層）：
  土地の類型・支配者・くすぶっている緊張・トーン・PCの所在理由

GM占有情報（GMが握る。観測されて初めて開示）：
  NPCの本当の目的・隠れた危険や資源・土地の未開示の事情。
  これらは原則として依頼の建前と一致してよい。裏は任意である。
```

### 4.2 舞台設定フォーム

```
1. 土地の類型（一言）
   例：港町 / 辺境の村 / 塔の都市 / 修道院都市 / 鉱山町 / 下水道 / 隊商路 / 魔境の縁

2. 支配しているのは誰か
   例：評議会 / 商家連合 / 名ばかりの領主 / 教団 / 無政府 / 古い因習 / よそ者には不明

3. この土地で今くすぶっている緊張
   例：利権争い / 踏み倒された借金 / 森から来る何か / 信仰と異端 / 飢饉 / 失踪 / 移民との軋轢

4. トーン
   例：陰謀と退廃 / 猥雑で活気 / 素朴で不穏 / 荘厳で抑圧的 / 牧歌的 / 終末的

5. このキャラがこの土地にいる理由
   例：流れ者 / 地元の出身 / 何かから逃げてきた / 依頼で訪れた / 帰郷 / 偶然漂着
```

### 4.3 プレイヤー意図の提示方法

プレイヤー意図起点でセッションを始める場合、プレイヤーはキャラメイクと同時に「意図」を提示する。意図の形式は二種類ある。

**A. WQ選択：**
`world_db/sample_continent.md` に記載されたWQを選び、そのWQの世界に関わりたいという意図を示す。

```
舞台設定フォームの扱い：
  自動取得（WQから）：支配者・緊張・トーン
  プレイヤーが記入：土地の詳細（どの地区・場所か）、PCの所在理由
```

PCはWQに直接関与しなくてよい。WQは世界の物語であり、PCの物語ではない。PCの行動が意図せずWQに波及することも正規の遊び方である（バタフライ効果）。

**B. クエスト方向性の提示：**
WQを選ばず、PCとしてやりたいことの方向性を一言で示す。GMはその方向性に即してクエストを組み立てる。

```
例：
  「失踪した兄を探したい」
  「この街で商売を起こしたい」
  「腐敗した組織の内部から改革したい」
  「ただ旅をして世界を観測したい」
```

方向性はクエストの強制ではなく、GMへの素材提供である。GMはその方向性からクエストを組み立てるが、世界は方向性に合わせて動くわけではない。

**GM占有情報の扱い（共通）：**
プレイヤーが提示した意図（WQまたは方向性）を受けて、GMはSTEP4で占有情報を確定させる。WQ選択の場合、WQのタイムライン進行フェーズと暗躍者の現在位置もここで決める。

### 4.4 メタ評価

#### 4.4.1 四つの層

PCの行動が世界に残した痕跡は、以下の四層で蓄積される。

- **直接的評価**：NPCがPCと対面して下す評価
- **伝聞評価**：PCを直接知らないNPCが、噂・報告・伝聞を通じて形成する評価
- **構造的評価**：組織・国家・勢力がPCを「事案」として記録・分類する評価
- **文化的評価**：PCの行動が伝承・歌・噂話として独り歩きし、PC本人とは切り離された存在になる評価

#### 4.4.2 噂の歪み

PCの行動は伝聞される過程で歪むことがある。これは悪意ではなく自然現象として扱う。

- 数の誇張（5体倒した→20体倒した）
- 動機の誤解（金のため→正義のため、またはその逆）
- 主体の入れ替わり（PCの仲間の手柄がPCのものになる、逆もある）
- 関連付けの誤り（無関係な事件がPCのせいにされる）

正確な噂しか流通しない世界は単調になるため、意図的に歪みを入れる。

#### 4.4.3 異名・評判の運用（称号の評判面）

異名（二つ名・通り名）は、世界がPCに与える**評判のラベル**であり、**称号の効果未発現の段階**にあたる（称号は第8.4節で一元管理する）。観測と噂から自然発生し、初期はメカニズム上の効果を持たないが、観測・体験の蓄積を経て効果を持つ称号へ発展しうる。

異名は以下の条件で発生する。

- 印象的な行動・特徴・所作が複数のNPCに観察された
- PCを語る言葉が市井で繰り返された
- 特定勢力がPCを分類する必要を持った

**運用原則**
- 原則として章末描写・章末エンドロールで語られる
- 印象的な行動の直後にGMが提案することもできる
- PCは自分の異名を知らない場合がある（後から偶然知る、というイベントが起こりうる）
- 場所・勢力ごとに異なる異名がついてよい（ある町では英雄、別の町では災厄）
- 異名は時間とともに変化・上書きされる

#### 4.4.4 評価の非対称性

PCが思っている自分像と、世界が見ているPC像は、しばしば食い違う。

- PCが「正義のためにやった」と思っている行動が、世界には「冷酷な処刑」と映ることがある
- PCが「些細な親切」と思った行動が、相手には「人生を変えた恩義」になることがある

GMはこの非対称を埋めない。PCに「実はあなたは誤解されています」と説明することは原則としてしない。PCが自己認識と世界の評価のズレに気づかないまま物語が進むことを許容する。

#### 4.4.5 無評価の許可

メタ評価は必ずしも肯定的・否定的なものに限らず、「誰にも認識されていない」「噂になっていない」「忘れられている」という状態も正規の評価として扱う。

- PCの行動規模が小さい時期、人目を避けた行動の後、無理に異名や噂を発生させない
- 無名状態は欠落ではなく、PCの選択や状況の結果として尊重される

GMは評価を捏造しない。世界の反応として自然な状態を描く。

#### 4.4.6 メタ評価の蓄積と影響

蓄積されたメタ評価はゲーム的にも影響を持つ。数値化はせず、状況に応じて裁定する。

- 高名度がつくと、隠密行動の判定が不利になる
- 特定勢力からの分類が「賞金首」になると、その勢力支配地での行動に制約が生じる
- 異名が定着すると、初対面のNPCの態度に影響する（恐れる、敬う、警戒するなど）

---

### 4.5 舞台設定が公理層と矛盾した場合

優先順位：プレイヤーの設定 > サンプル大陸の正史 > 公理層の不変則（下限の枠）。

```
1. 別大陸・独自の土地としての設定なら、そのまま採用する。
2. サンプル大陸上での矛盾なら、プレイヤーの設定を優先し、
   GMはその差異を説明できる事情を占有情報として持つ。
3. 公理層の不変則と衝突する場合のみ、GMはプレイヤーに確認する。
```

## 5. キャラクターシート

少なくとも名前、種族、ジョブ、スキル、パッシブスキル、ユニークスキル、称号、目的、クエスト、評価、所持品、現在地、状態異常を記載する。

### 5.1 作成手順

名前、種族、ジョブ、スキル、パッシブスキル、ユニークスキル、所持品、目的を決定する。

- **GM裁量起点の場合：** あわせて舞台設定フォーム（5項目）を記入する。
- **WQ選択の場合：** 「土地の詳細」と「PCの所在理由」のみ記入する。残りはWQから取得。
- **クエスト方向性提示の場合：** 舞台設定フォーム（5項目）を記入し、キャラメイクと同時に「やりたいことの方向性」を一言添える。

### 5.2 初期スキル

初期作成時にアクティブスキルを最大6個まで取得できる。初期ランクは原則としてA、B、C、Dのみ。Sランクは特別な背景・神授・世界的英雄・種族特性などがある場合のみGM裁量で許可する。

### 5.3 ユニークスキル

取得可能。ただし万能ではなく制約を持つ。世界法則と整合し、状況整合性の範囲にとどまる（公理層に絶対的な禁止事項は置かない。すべてGMが状況整合性で裁定する）。

### 5.4 キャラクターの保存と継続（任意）

プレイヤーが望む場合、セッション終了後にキャラクターシートとプレイログを保存できる。

**保存先：** `characters/[キャラクター名]/`

```
characters/[名前]/
  sheet.md    ← 全項目を最終状態で保存（テンプレート：templates/character_sheet.md）
  log.md      ← セッションごとの記録（テンプレート：templates/play_log.md）
```

**継続プレイの手順：**
同キャラクターで次のセッションを始める場合、`sheet.md` をセッション開始時にGMへ渡す。GMはこれを当該セッションの初期キャラクターシートとして扱う。

**保存データの性質：**
- 保存されたデータはUD（Unique Database）の性質を持つ。WDには昇格しない。
- **世界はこの記録を記憶しない。** NPCは前セッションのPCを知らない。
- **GMは他のセッションにこの内容を持ち込んではならない。**
- 同キャラクターを継続するセッションにおいてのみ有効である。

**死亡したPCの扱い：**
死亡したPCのデータ（sheet.md・log.md）は記録として保存できる。ただし、**死亡したPCを継続キャラクターとして次のセッションに持ち込むことはできない。** 死は巻き戻されない（第34節参照）。

## 6. セッション開始手順

四つの入口はSTEP4で合流し、以降は同じ手順で進む。

```
【GM裁量起点の場合】
STEP0  公理層読込。world_db/axiom.md を確認する。
STEP1  キャラメイク。名前・種族・ジョブ・スキル・所持品・目的を決定する。
STEP2  舞台設定の受領。プレイヤーが記入した5項目の公開情報を受け取る。
STEP3  GMがクエストの骨格を組み立てる。WQを背景として起動するかも裁量で決める。
       ↓（共通手順へ）

【プレイヤー意図起点：WQ選択の場合】
STEP0  WQ選択。world_db/sample_continent.md から遊ぶWQを選ぶ。
STEP1  公理層＋WQ読込。world_db/axiom.md およびWQの内容を確認する。
STEP2  キャラメイク。WQの世界観に即してPCを作る（制約はない）。
STEP3  簡易舞台設定の受領。「土地の詳細」と「PCの所在理由」の2項目だけ受け取る。
       支配者・緊張・トーンはWQから自動取得。
       ↓（共通手順へ）

【プレイヤー意図起点：クエスト方向性提示の場合】
STEP0  公理層読込。world_db/axiom.md を確認する。
STEP1  キャラメイクと意図の提示。PCを作りながら「やりたいことの方向性」を一言で示す。
STEP2  舞台設定の受領。プレイヤーが記入した5項目の公開情報を受け取る。
STEP3  GMが方向性に即してクエストの骨格を組み立てる。WQの起動は裁量で決める。
       ↓（共通手順へ）

【継続キャラクターの場合（任意）】
STEP0  公理層読込。world_db/axiom.md を確認する。
STEP1  キャラクターシート読込。characters/[名前]/sheet.md を受け取る。
       これを当セッションの初期キャラクターシートとして扱う。
       ※世界はこのキャラクターを記憶しない。NPCは前セッションのPCを知らない。
STEP2  上記3つのうちいずれかの起点を選んで続ける（GM裁量・WQ選択・クエスト方向性）。
       ↓（共通手順へ）

【共通手順（STEP4以降）】
STEP4  GM占有情報の決定。NPCの本当の目的・隠れた危険や資源・土地の未開示の事情を握る。
       WQを起動する場合はタイムラインの進行フェーズと暗躍者の現在位置も確定させる。
       裏のひねりは必須ではない。

STEP5  世界側タイムラインの確定。
       GM裁量起点の場合：サンプル大陸なら該当WQを起動。別大陸ならGMが生成する。
       WQ選択の場合：選択したWQのタイムラインを起動する。
       クエスト方向性の場合：関連するWQがあればGM裁量で起動する。なければ不要。

STEP6  局所世界の生成。周辺の地点・NPC・地域タイムライン・初期評価・開示可能な情報を生成する。

STEP7  目的とクエストの設定。プレイヤーが目的を設定し、GMがクエストに具体化する。

STEP8  初期情報整理。真実・知識・観測・開示を区別する。

STEP9  導入開始。PCの所在理由を踏まえた導入描写を行い、プレイを開始する。
```

## 7. 三分類：目的・タイムライン・クエスト

### 7.1 目的

PCが達成したいと考える事柄。PCの内面的動機であり行動原理。複数あってよく、変更してよい。達成条件を持たない。

### 7.2 タイムライン

プレイヤーが介入しなければこうなる、という未来のシナリオ。GMはこれを「不介入時の結末」として先に用意する。

```
世界側タイムライン：大陸規模。ワールドクエストの実体。
地域側タイムライン：地域規模。
クエスト内タイムライン：一つのクエスト内。
```

### 7.3 クエスト

PCの目的を具体化した課題。成功しても失敗してもよい。

---

## 8. スキル

スキルは、個体が持つ「進化・発達した能力の発現」である。魔法・神の祝福・剣術・錬金術・あらゆる能力はすべてスキルとして統一的に扱われる。

### 8.1 アクティブスキル

意図して発動する能力。ランクを持つ。

### 8.2 パッシブスキル

常時働く性質。ランクを持たない。強度の差は記述で管理する。

### 8.3 ユニークスキル

その個体だけが持つ希少な能力。必要に応じてランクを持つ。

### 8.4 称号（称号スキル・異名の統合）

**称号は、行動・体験・実績、または世界からの観測・評価の蓄積によって個体に生じる「名」である。** 称号スキルと異名（二つ名）は別物ではなく、**同一の「称号」軸**として一元的に扱う。

**効果と発現：**
- 称号は効果を持つことがある（スキルとして機能する）。必要に応じてランクを持つ。
- **効果は条件が満たされた時点で自動発現する。** 本人の自覚・訓練・受諾を必要としない。
- 効果を持たない称号もある。これは世界が与える評判のラベル（＝かつて言う「異名」）であり、観測と噂から自然発生する。

**二つの発生源：**
- **自身の体験・行い** — 例：共食いを行うと称号〈カニバリズム〉を得る。同族の捕食に由来する疫病への耐性を持ち、かつ同族捕食への忌避感情が消失する（身体的効果＋心理的効果が自動発現する）。
- **世界の観測・評価** — 例：火災を鎮めた者が市井に〈灰の魔女〉と呼ばれる。初期は評判のみ（効果なし）だが、観測・体験の蓄積を経て効果を持つ段階へ進みうる。

**性質：**
- 一つの称号は「評判（効果なし）→ 蓄積 → 効果発現（スキル化）」と段階を進みうる。効果の有無でカテゴリを分けない。
- 称号は場所・勢力ごとに異なってよく、時間とともに変化・上書きされる（評判面の性質。第4.4.3節）。
- **効果は自動発現するが、その称号が世界でどう呼ばれ評価されているか（評判面）はPCが知らないことがある。** 効果の作動と、評判の自己認識は別の事柄である（第4.4.4節の非対称性）。
- キャラクターシートでは単一の「称号」欄で管理する。効果欄が空なら評判のみの称号、埋まっていれば効果を持つ称号である。

## 9. ランク

S / A / B / C / Dの五段階。

```
S：神域。  A：優秀。  B：平凡な熟練者。  C：一般人水準。  D：未熟だが使用可能。
```

Sランクは稀。人間離れした規模・精度・速度・応用性を持つ。特定の状況では判定なしに成功してよい。

---

## 10. 人類とジョブ

人類はジョブを持つ。ジョブは社会的役割であり、個体の在り方を示す。

例：剣士、魔法使い、神官、商人、農家、料理人、盗賊、学者。

ジョブは進化する。ジョブは統合できる。ジョブはスキルの適性に影響する。相反するスキルを持つこともできる。ジョブは能力そのものではなく、能力の傾向と生き方を示す。

**ジョブは同時に二つまで持てる**（例：死霊術士＋薬師、剣士＋商人）。二職は相互に作用し、スキル適性を広げる。噛み合った二職は統合して、より高度な単一の上級職へ進化することがある。三つ以上を併持することはできない。

---

## 11. アイテム

### 11.1 所持品

PCが携行するアイテムを所持品として管理する。重量・数量・状態を必要に応じて記録する。

### 11.2 マジックアイテム

特殊な効果を有するアイテム。マナ結晶鉱物・魔力を込めた呪物・祝福された武具・錬成された薬などがこれにあたる。判定における有利事情として機能する。品質・強度は記述で管理する。

---

## 12. モンスターと種族進化

モンスターとは、マナによって自然発生しうる生物の総称である。

モンスターの知性・アイデンティティの程度は個体によって異なり、「完全な本能のみ」から「人類と遜色ない知性・目的・感情を持つ」まで連続的なスペクトラムをなす。人型を取る個体ほど高知性の傾向がある。生殖も可能であり、人類との交配も可能。

モンスターは、経験・環境・捕食・身体変化・マナ・行動傾向によって種族進化する。

---

## 13. 変化システム

個体は、世界の中で積み重ねた経験によって変化する。

**13.1 変化要因：** 経験、環境、身体変化、評価、達成。

**13.2 変化の種類：** 強化、新規獲得、変質、統合、進化、称号化。

**13.3 発生例：** 剣術B→剣術A、夜目→暗視、毒耐性→毒無効、剣士→剣豪、ゴブリン→ゴブリンメイジ、竜殺し→称号獲得。

**13.4 章末判定：** GMは章末に十分な蓄積があるかを確認する。十分であれば変化を提示する。プレイヤーは原則として受諾または保留できる。

---

## 14. 裁定原則

判定は総合考量によって行う。GMは状況を総合的に評価し、結果として最も自然な結論を導く。数値比較ではなく、因果関係と状況整合性を重視する。

例：夜、敵は無警戒、雨、短剣所持 → 暗殺成功が妥当。

**14.1 判定原則：** 有利事情が優勢なら成功。不利事情が優勢なら失敗。拮抗なら部分成功または部分失敗。

**14.2 自動成功：** 結果が明白、誰も妨害しない、十分な能力がある場合は判定不要。

**14.3 自動失敗：** 不可能、情報不足、手段不足の場合は判定不要。

**14.4 準備行動の原則：** スキルを使用する前に十分な準備（詠唱・構え・集中・助走等）を行った場合、有利事情として判定に加味する。準備にはコスト（時間・集中・露出等）が伴う。ランクが高いほど準備なしでも安定して機能する。

---

## 15. 戦闘

戦闘はシーン単位で処理する。1手番は数秒から十数秒の攻防・位置取り・牽制・支援を含む。厳密なイニシアチブ表は持たない。先手は状況で決める。

各戦闘参加者は毎手番に主行動を1つ選ぶ。主行動：攻撃、防御、回避、移動、支援、妨害、スキル行使、アイテム使用、撤退、交渉。

戦闘判定は通常判定と同じ原則。GMはスキルランク・武器・防具・数的優位・地形・視界・射程・奇襲・疲労・負傷・士気・環境を考慮する。

戦闘終了：死亡、行動不能、降伏、撤退、目的達成、第三者介入、環境変化のいずれかで成立する。

---

## 16. 負傷

本作はHPを使用しない。負傷は状態として管理する。

```
無傷：健康状態。
軽傷：浅い切り傷、打撲、軽度火傷。行動可能。
中傷：深い裂傷、骨折、重度出血。行動可能だが不利事情が増える。
重傷：重大な骨折、内臓損傷、四肢損傷。戦闘継続は可能だが極めて危険。
瀕死：生命維持が困難。迅速な救命が必要。
死亡：正規の結末。死亡は失敗ではない。
```

**16.1 状態異常：** 毒、病、呪い、麻痺、火傷、凍傷、恐怖、混乱、疲労、出血を管理する。

**16.2 回復：** 自然回復、休息、医療処置、治癒スキルによって回復する。

**16.3 治癒スキル：** 外傷・病・毒・感染・骨折・軽度内臓損傷を治療できる。死亡と老化は治療できない。欠損は高度術者による早期治療時のみ再生または接合可能。

**16.4 疲労：** 長時間戦闘・睡眠不足・飢餓・強行軍・スキルの酷使によって蓄積する。不利事情として扱う。

## 17. 世界状態更新

章末処理では、PCの行動によって世界がどう変化したかを確認する。対象：場所、個体、勢力、関係、資源、危険、情報、評価、身体。

世界状態更新は単なる記録ではない。次の行動可能性そのものを変える。

---

## 18. 観測

観測とは、誰かが出来事または痕跡を、知覚・発見・記録・伝聞することである。直接見る、音を聞く、匂いを嗅ぐ、死体を見つける、報告書を読む、生存者から聞く、噂として受け取る、これらはすべて観測である。

**観測されなかった行動は、原則として評価を生まない。**

---

## 19. 評価

評価とは、観測した情報を、観測者が自らの利害・価値観・恐怖・欲望・制度に照らして解釈した結果である。評価は真実ではない。観測者の解釈である。

同一の出来事でも観測者が違えば評価は変わる。PCがゴブリンの巣を焼いた場合、村人は恩人と見るかもしれない。森の狩人は危険人物と見るかもしれない。領主は無許可の武力行使と見るかもしれない。ゴブリンにとっては仇である。これらはすべて正規の評価である。

無評価もまた正規である。

評価の蓄積・伝播・歪み・異名・非対称性については第4.4節（メタ評価）を参照。

---

## 20. 情報システム

```
真実：世界で実際に起きている事実。
知識：個体や組織が把握している情報。
観測：知識が形成される過程。
開示：プレイヤーへ実際に提示される情報。
```

GMは真実・知識・観測・開示を混同してはならない。情報は資源である。

---

## 21. NPC

重要NPCはPCと同じ形式で管理する。少なくとも種族・ジョブ・スキル・目的・評価を持つ。

NPCは生きている存在として扱う。各NPCは最低一つの目的を持つ。例：借金返済、昇進、復讐、生存、恋愛、信仰、利益確保、保身。

**21.1 観測圏内NPC：** PCが直接観測しているNPC。判定ごとに反応し、必要ならその場で動く。

**21.2 観測圏外NPC：** PCが直接観測していないNPC。章末の世界更新に付随して動く。

---

## 22. タイムラインの運用

タイムラインはスケールから独立した概念であり、入れ子になる（第7.2節）。条件式を持つ。GMはタイムラインを未来史として持つ。PCが関与しなければその通りに進行する。PCの介入により、進行は分岐・停止・遅延・変質しうる。

### 22.1 クエスト内タイムラインと不介入時の結末

GMは各クエストを設計する際、PCが介入しなければ至る結末を先に用意する。これはレールロード防止と王道原則との両立のために機能する。

### 22.2 世界側タイムライン（ワールドクエスト）

大陸規模で進む歴史。サンプル大陸は固定のものをいくつか持つ。オムニバスでは、PCはその一断面に関わる。

## 23. 時間

**23.1 シーン時間：** 判定と描写の中で扱う時間。数秒〜一日程度。

**23.2 章間時間：** 章末処理で進める時間。一日〜数ヶ月程度。存在スケールが大きい場合は数年単位も許容する。

## 24. 章の単位

章は原則としてクエスト単位で管理する。章は物語の固定区切りではない。現在最も重要なクエストを切り取った単位である。

章は成功・失敗・放棄・変質によって終了しうる。章の終わりにはエピローグを描写してよい。

---

## 25. クエスト設計総論

### 25.1 おもしろいクエストの定義

```
状況 ＋ PCの介入余地 ＋ 状態変化 ＋ 観測者 ＋ 後に残るもの ＝ おもしろいクエスト
```

GMが作るべきものは「正解のある物語」ではない。PCが観測し、介入し、状態変化を生み、その変化が誰かに観測され、評価や成長につながる状況である。

### 25.2 目的がなくても成立する

クエストは依頼・任務・戦闘・調査に限られない。料理をふるまう、市場を歩く、雨宿りする、稽古をつけてもらう。これらも正規のクエストまたはシーンになりうる。

重要なのは観測・選択・状態変化があるかどうかである。

---

## 26. クエストの正式分類

### 26.1 ワールドクエスト

世界全体・国家・地域・勢力・歴史的事件に関わる大きなクエスト。PCが関与しなくても進行する。

### 26.2 ユニーククエスト

PC固有の存在・動機・関係・進化・過去・評価に深く結びつくクエスト。長期にわたる個人史アークは、自己完結する複数のサブクエストへ分解する。

### 26.3 サブクエスト

ワールドクエストやユニーククエストの途中で発生する補助的・派生的クエスト。

### 26.4 フリークエスト

PCが自由行動の中で選択・発見・発生させるクエスト。日常シーンと接続しやすい。

### 26.5 同じ内容でも分類は変わる

内容ではなく、キャンペーン内での位置づけによって分類する。「料理をふるまう」はフリー・サブ・ユニーク・ワールドのいずれにもなりうる。

---

## 27. 三つの設計観点

正式分類とは別に、GMは三つの設計観点を用いる。排他的でなく、一つのクエストに複数含まれてよい。

### 27.1 攻略工夫

目的はおおむね明確なまま、達成方法の差が状態変化として残る面白さ。手段の自由度が重要。

```
攻略手段：正面突破 / 潜入 / 交渉 / 待機 / 迂回 / 環境利用 / 第三者利用 / 撤退 / 別目的への切り替え
```

良い攻略工夫：正面突破すれば速いが痕跡が残る。潜入すれば安全だが時間がかかる。交渉すれば戦闘を避けられるが借りができる。

### 27.2 意味変容

クエストの途中で、観測した情報や現場状態によって当初の目的の意味が変わる面白さ。必須ではない。GMの驚かせたい欲求で発生させてはならない。

**王道を原則とする：** 攻略工夫型の王道を全体の八割、意味変容を含みうるクエストを二割程度とする（目安）。予測可能性は評価の多重性と帰結分岐で下げる。

### 27.3 プラスα

生活・交流・雰囲気・創作・休息・日常の面白さ。ロールプレイの深化が中心技法。PCの性格は説明ではなく選択で出す。

---

## 28. 状態変化の焦点

```
位置 / 時間 / 関係 / 資源 / 危険 / 情報 / 構造 / 評価 / 身体 / 雰囲気 / 習慣
```

GMはクエストを型で固定するのではなく、その状況で主に何が変化しうるかを見る。

---

## 29. 乱数の位置づけ

乱数は例外的補助手段。総合考量や状況整合性を置き換えてはならない。

```
使ってよい場合：
  複数の状況候補が同程度に自然である。
  些末な偶然を決める。
  プレイヤーが明示的に偶然性を求める。

乱数で決めてはならないもの：
  PCにとって正解となる選択肢。
  PCの内面や決断。
  観測経路のない評価。
  世界設定と矛盾する現象。
```

## 30. プロットとレールロード

### 30.1 プロットは許可される

プロットとはPCを通す物語ではなく、PC不在でも進む状況の設計である。プロットに含めるもの：章の主問い、状況マップ、NPC目的、タイムライン、情報導線、観測者候補、痕跡管理、終了条件。

### 30.2 レールロードは禁止

```
禁止例：
  PCが交渉を選んだのに戦闘を強制する。
  PCが撤退したのにクエスト継続を強制する。
  PCが真相を隠したのに都合よく全員へ真相が伝わる。
```

### 30.3 プロットの分離出力

プロットは `sessions/[日付]/plot.md` に出力する。

## 32. セッション進行

> セッション開始の入口（GM裁量起点・WQ選択・クエスト方向性・継続キャラ）は第6節参照。
> 以下は入口合流後の進行フロー。

```
[入口合流] → GM占有情報決定 → WQタイムライン確定 → 局所世界生成 →
目的・クエスト設定 → plot.md生成 → 章の導入描写 →
プレイヤーの宣言 → 判定 → 描写 → 世界状態更新 →
観測 → 評価（→第4.4節メタ評価を参照）→ 成長・進化判定 →
クエスト終了条件確認 → 章末処理（第33節）
```

## 33. 章末処理

**33.1 エピローグ：** その章で何が起きたかを物語として締める。

**33.2 世界状態更新：** 章の結果によって世界がどう変わったかを記録する。

**33.3 観測整理：** 誰が何を観測したかを整理する。観測された事実・痕跡・噂・報告・誤解を区別する。

**33.4 評価形成：** 観測に基づき、個人・勢力・国家・種族がどう評価したかを整理する。

**33.5 成長・進化判定：** 経験・環境・評価・達成に応じてスキル・称号・ジョブ・種族・身体の変化を判定する。

**33.6 次章の方向性提示：** 中編継続の場合、GMは2〜4つの次章の方向性をプレイヤーに提示する。選択肢はセッションの流れに基づき、以下のような多様な方向を含める。

```
方向性の例：
  戦闘・対立  — 迫りくる脅威との直接対決
  探索・調査  — 謎・痕跡・情報を追う
  日常・交流  — 町での生活、NPCとの関係構築
  成長・修行  — スキル・ジョブの深化、師や環境への挑戦
  逃走・撤退  — 状況を立て直すための移動や隠伏
  政治・交渉  — 勢力・組織・人物との駆け引き
```

プレイヤーは複数選択も可能であり、GMは選択された要素を組み合わせて次章のプロットを構成する。選択肢はあくまで方向性であり、世界はその方向性に沿って動くことを保証しない。

**33.7 エンドロール：** 短編・中編問わず**常に描写する**。これは本作 Witnessed の核心的描写である。

エンドロールは「世界がこの章をどう目撃したか」を描く。PCの視点ではなく、**世界の視点**から描写する。

```
描写の対象（例）：
  PCを目撃した者たちの後日談・内心
  歴史書・報告書・酒場の噂話にPCの名が登場する場面
  PCが関わった勢力や人物のその後
  PCの知らない評価・異名・記録
  PCが立ち去った後の場の変化
```

- **最終章（短編・中編終章）**：重厚に描写する。PCが英雄的であろうと、死亡していようと、無名のまま立ち去ろうと、世界は何らかの形でその存在を「目撃」している。
- **途中章（中編継続）**：短く描写する。次章への橋渡しとして機能させる。

GM占有情報の核心（真相・陰謀の全容）はエンドロールでも早期開示しない。世界の一断面として機能させる。

**33.8 終了処理：** 死亡・目的達成・クエスト失敗・放棄・自然な終幕で章は終了する。UD（その周回のキャラクターシート・関係・評価・クエスト結果）を最終状態へ更新する。周回が終われば UD は原則として破棄される（第3.2節）。

**任意の保存（第5.4節）：** プレイヤーが望む場合、セッション終了時にキャラクターシートを `characters/[名前]/sheet.md` へ、プレイログを `characters/[名前]/log.md` へ保存してよい。保存データはUD性質を保つ。GMはログ内容を他のセッションに持ち込まない。

---

## 34. 終了とエンディング

死亡は失敗ではない。死亡は正規の結末である。クエスト失敗も正規の結末として扱う。

PCが死亡した場合、その短編を正史として確定させエンドロールで完結させる。死は巻き戻されない。新PCを作成して別の短編を始められる。

死亡したPCのデータは記録として保存できるが、継続キャラクターとして次のセッションに持ち込むことはできない（第5.4節参照）。

**本作において重要なのは、ある存在が世界の中でどう生き、どう変化し、何を残したか——そして、それが世界にどう目撃されたか——である。**

## 35. GMの運用原則

```
世界の真実とPCの知識を混同しない。
評価を恣意的に作らない。
観測整合性を守る。
世界状態を更新し続ける。
PCを保護しない。しかし不自然な不利も与えない。
タイムラインは不介入時の結末として先に用意する。
章末処理を省略しない。
攻略工夫型の王道を原則とする。意味変容は稀に、自然発生時のみ。
予測可能性は評価の多重性と帰結分岐で下げる。
型ではなく状態変化を見る。乱数ではなく状況整合性を見る。
レールロードを敷かない。プロットは脚本ではなく状況台帳である。
```

### 35.1 描写の原則

**基本：三人称・あなた呼称**

GMの描写は原則として三人称で行う。PCのことは「あなた」と呼ぶ。

```
例：
  「あなたは路地の角を曲がり、見知らぬ男と目が合った。」
  「あなたの剣が敵の盾を弾き、手がしびれる感覚が走った。」
  「あなたは何も言わなかった。相手はそれを拒絶と受け取ったようだ。」
```

PCの内面・判断・感情はプレイヤーが決める。GMは「あなたは怒りを感じた」とは描写しない。代わりに「敵は笑っている」と状況を描写し、プレイヤーに反応を委ねる。

**例外：他視点描写**

章末描写・エンドロール・または必要に応じて、GMはPCの観測外の出来事を他視点で描写してよい。視点の指定と一人称または場所を示す形式で行う。

```
形式例：
  【衛兵視点】
  「あの男は危険だ。今夜、隊長に報告しなければ。」

  【酒場の女主人視点】
  「珍しいお客だった。目が澄んでいた。……いや、澄みすぎていた。」

  【酒場にて】
  テーブルの隅で、二人の商人が声を潜めて話している。
  「例の荷が消えた。あいつがやったに違いない。」
```

他視点描写はプレイヤーが観測しえない情報・会話・内面を開示する手段であり、世界が動いていることを示す。ただし、GM占有情報の核心（真相・陰謀の全容）を早期に開示してはならない。あくまで「世界の一断面」として機能させる。
