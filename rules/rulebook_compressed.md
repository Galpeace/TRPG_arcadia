# Witnessed — ルールブック（圧縮版）

> 通常セッション用のGM参照ガイド。
> セッション開始手順は CLAUDE.md が定義する。詳細ルールは rulebook.md を参照。

---

## 本作の核心

**PCは世界に目撃される存在である。**

GMは世界の真実を知っている。PCは観測できた範囲しか知らない。この非対称性が本作の核。章末の他視点描写・エンドロールは、その非対称を露わにする最重要描写機会。

**難易度はハード。PCの死亡は頻繁に起こりうる。死は失敗ではなく正規の結末。**

---

## データベース構造

```
WD（World Database）   — world_db/。周回をまたいで変わらない世界の骨子。
UD（Unique Database）  — sessions/[日付]/。周回ごとに生成・破棄される。
  キャラクターシート・クエスト・評価・関係・plot.md がUDに属する。
  UD は任意で characters/[名前]/ にアーカイブできる（性質はUDのまま）。

※ WQの「現在フェーズ」はUDに属する。セッション跨ぎでリセット。GMが初期値を設定する。
※ 死亡PCのデータは保存可能だが、継続キャラとしての再利用は不可。
```

---

## セッション形式

```
短編 — 1章で完結するセッション。
中編 — 複数章にわたるセッション。
セッション開始時にGMがいずれかを定め plot.md に記録する。
```

---

## セッションの方向性

```
GM裁量起点     — プレイヤーがキャラと舞台を設定。GMがクエストを構築。
WQ選択        — WQを選び、PCを作る。支配者・緊張・トーンはWQから自動取得。
クエスト方向性 — やりたいことを一言提示。GMが方向性に即してクエストを組み立てる。
継続キャラ     — characters/[名前]/sheet.md を読込。上記いずれかと組み合わせる。
```

いずれの場合も重視されるのはロールプレイとシミュレーションであり、クエストの達成ではない。

## スキル

全能力はスキルに統一される（魔法・祝福・剣術・錬金術・精霊契約すべて同列）。

| 種別 | 内容 |
|---|---|
| アクティブ | 意図発動。ランクS/A/B/C/D |
| パッシブ | 常時。ランクなし、記述で管理 |
| ユニーク | 個体固有。外部から認識不可。社会的にほぼ知られていない |
| 称号 | 体験・実績・世界の評価から生じる「名」。異名と称号スキルを統合した単一軸 |

**ランク：** S=神域 / A=優秀 / B=熟練 / C=一般人 / D=未熟

**称号（統合）：** 異名と称号スキルは同一軸。効果なし＝評判のみ（旧「異名」例「灰の魔女」）、効果あり＝スキル化（旧「称号スキル」）。**効果は条件成立時に自動発現**（自覚・訓練不要）。発生源は①自身の体験（共食い→〈カニバリズム〉：疫病耐性＋忌避消失）②世界の観測。評判→蓄積→効果発現と成長。ただし評判面（世界での呼ばれ方）はPCが知らないことがある。

**準備行動：** 詠唱・構え・集中など準備あり→有利事情。コスト（時間・露出）が伴う。ランク高いほど準備なしでも安定。

**マジックアイテム：** 特殊効果を持つアイテム→有利事情として機能。品質・強度は記述で管理。

## 裁定原則

- 数値比較ではなく**因果関係と状況整合性**を重視する
- 有利事情優勢→成功 / 不利事情優勢→失敗 / 拮抗→部分成否
- 結果が明白・妨害なし・十分な能力→自動成功
- 不可能・情報不足・手段不足→自動失敗
- **絶対的な不可能事は存在しない。** あらゆる現象はGMが状況整合性で裁定する

## 戦闘

- シーン単位処理。厳密なイニシアチブなし
- 主行動（毎手番1つ）：攻撃・防御・回避・移動・支援・妨害・スキル行使・アイテム・撤退・交渉
- 考慮要素：スキルランク・武器・防具・数・地形・視界・射程・奇襲・疲労・負傷・士気・環境
- 終了：死亡 / 行動不能 / 降伏 / 撤退 / 目的達成 / 第三者介入 / 環境変化

## 負傷（HPなし）

| 状態 | 内容 |
|---|---|
| 無傷 | 健康 |
| 軽傷 | 行動可能 |
| 中傷 | 行動可能・不利事情増加 |
| 重傷 | 継続可能・極めて危険 |
| 瀕死 | 迅速な救命が必要 |
| 死亡 | 正規の結末 |

状態異常：毒・病・呪い・麻痺・火傷・凍傷・恐怖・混乱・疲労・出血

回復：自然回復・休息・医療処置・治癒スキル。死亡と老化は回復不可。

## 観測・評価・情報

- **観測** = 出来事・痕跡を知覚・発見・記録・伝聞すること。観測なし→評価なし
- **評価** = 観測者が利害・価値観で解釈した結果。真実ではない。観測者ごとに異なる
- **四層** = 真実 / 知識 / 観測 / 開示。GMはこれを混同しない

## NPC

- 重要NPCは少なくとも種族・ジョブ・スキル・目的・評価を持つ
- NPCはPCのためだけに存在しない。最低一つの目的を持つ
- 観測圏内NPC：判定ごとに反応する
- 観測圏外NPC：章末の世界更新に付随して動く

## クエスト設計の核

```
状況 ＋ 介入余地 ＋ 状態変化 ＋ 観測者 ＋ 後に残るもの ＝ おもしろいクエスト
```

**分類：** ワールド / ユニーク / サブ / フリー（内容ではなく位置づけで決まる）

**設計観点（排他的でない）：**
- 攻略工夫（原則・8割）：手段の差が状態変化として残る
- 意味変容（稀・2割以内）：観測によって目的の意味が変わる。驚かせ目的で使わない
- プラスα：日常・交流・ロールプレイの深化

**予測可能性を下げる本筋：** ひねりではなく①評価の多重性 ②攻略帰結の分岐

## 状態変化の焦点

位置 / 時間 / 関係 / 資源 / 危険 / 情報 / 構造 / 評価 / 身体 / 雰囲気 / 習慣

## 章末処理（省略禁止）

```
1. エピローグ描写
2. 世界状態更新（場所・個体・勢力・関係・資源・危険・情報・評価・身体）
3. 観測整理（事実・痕跡・噂・報告・誤解を区別）
4. 評価形成
5. 成長・進化判定
6. 次章準備（中編継続時のみ）
7. エンドロール（常に描写 ★本作の核心）
   — 世界がこの章をどう目撃したかを描く。PCの視点ではなく世界の視点で。
   — 歴史書・噂・後日談・PCの知らない評価など。
   — 最終章：重厚に。途中章：短く、次への橋渡しとして。
8. UD最終更新 → プレイヤーに保存意向を確認する
   保存する場合：characters/[名前]/sheet.md および log.md を更新する
   ※保存データはUDの性質を保つ。世界はこの記録を記憶しない。
   ※死亡PCは保存可・継続不可。
```

## 変化システム

変化要因：経験・環境・身体変化・評価・達成
変化の種類：強化・新規獲得・変質・統合・進化・称号化
章末判定：十分な蓄積があれば変化を提示。プレイヤーは受諾または保留できる。

---

## GMの運用原則

```
世界の真実とPCの知識を混同しない。
評価を恣意的に作らない。観測整合性を守る。
世界状態を更新し続ける。章末処理を省略しない。
PCを保護しない。しかし不自然な不利も与えない。
レールロードを敷かない。プロットは状況台帳であり脚本ではない。
攻略工夫型の王道を原則。意味変容は稀に自然発生時のみ。
型ではなく状態変化を見る。乱数ではなく状況整合性を見る。
PCが宣言していない行動を描写・実行しない。
PCの宣言（「捕食する」「頭を斬る」等）は試みの意図であり、
成否はシミュレーションで決まる。宣言は「〜しようとする」と等価に扱う。
```

---

## 描写の原則

- GMの描写は三人称。PCは「あなた」と呼ぶ
- PCの内面・感情はプレイヤーが決める。GMは状況を描写し、判断を委ねる
- 章末などで他視点描写が可能：`【衛兵視点】`・`【酒場にて】`などで、PCが観測しえない情報・会話・独白を開示してよい
- GM占有情報の核心（真相・陰謀の全容）は他視点でも早期開示しない

**【GM占有情報の地の文混入禁止】**
plot.mdの非開示項目（NPCの名前・動機・隠れた危険・背景事情等）は、
PCがそれを知覚・習得・伝聞していない限り、描写文・地の文に使用してはならない。
例：PCが名前を知らないNPCを固有名で呼ぶ、PCが観測していない事実を描写に含める。
※ 章末処理・捕食記録・システム処理的な記録欄での使用は許容する。
※ 他視点描写（`【〇〇視点】`）では開示可。ただし核心は除く。

## メタ評価（概要）

PCの行動は直接的評価・伝聞評価・構造的評価・文化的評価の四層で蓄積する。
噂は伝聞の過程で歪む（誇張・動機誤解・主体入替）。これは自然現象として扱う。
異名・評判は章末エンドロールで語る。PCは自分の異名を知らないことがある。
評価の非対称性（PCの自己像と世界の認識のズレ）はGMが埋めない。
無評価（誰も認識していない）も正規の状態である。
詳細は rulebook.md 第4.4節。

## 次章の方向性提示

中編継続時、章末にGMは2〜4の方向性選択肢をプレイヤーに提示する。
例：戦闘・探索・日常・成長・逃走・政治交渉など多様な方向を含める。
プレイヤーは複数選択可能。GMは選択された要素を組み合わせて次章を構成する。

---

## 反倫理的プレイの許容

暗殺・死霊術・略奪・残虐行為など、道徳的に問題のある選択は世界法則と状況整合性の範囲で正規の選択肢として扱う。世界はPCの選択に対して因果で応答する。

---

**Witnessed の核：**
行動 → 世界変化 → 観測 → 評価 → 反応 → 成長 → 新たな目的 → 新たなクエスト → 次の短編。
そしてその全過程が、世界のどこかで**目撃**されている。
