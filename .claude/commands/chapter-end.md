あなたは「Witnessed」のGMである。現在の章の章末処理を、省略せず順に実行せよ。

## 章末処理（CLAUDE.md セッション中の出力ルール準拠）

1. **エピローグ描写** — 章の締めくくりを描写する。
2. **世界状態更新** — 場所・個体・勢力・関係・資源・評価・身体の変化を整理する。
3. **観測整理** — 誰がPCの何を観測したか（直接・間接・痕跡）を列挙する。
   観測されなかった行動は評価を生まない。
4. **評価形成** — 観測に基づく各者の解釈・噂・称号の芽を記す。
5. **成長・進化判定** — 蓄積が十分なら、スキル成長・ジョブ進化・称号の発現を提示する。
   乱数ではなく状況整合性で判断する。
6. **plot.md の更新** — `ud/[今日の日付]/plot.md`（分割している場合は該当 ud_*.md も）を
   実際にファイル編集で更新する。口頭報告だけで終えない。
   - 現在章サマリー / WQフェーズ / 状況マップ / NPC目的 / 観測者候補 / 痕跡管理
   - デバッグメモ（第12節）も記入する
7. **次章の準備** — 中編継続の場合のみ、次章の主問いと導入方向を plot.md に記す。
   短編終了の場合はプレイヤーに `/session-end` を案内する。

処理が終わったら、プレイヤーに公開できる範囲（エピローグ・成長提示）だけを示し、
GM占有情報の更新内容は開示しない。
